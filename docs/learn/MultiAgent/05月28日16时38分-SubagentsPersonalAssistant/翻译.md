## Overview / 概览

The **supervisor pattern** is a multi-agent architecture where a central supervisor agent coordinates specialized worker agents. This approach excels when tasks require different types of expertise. Rather than building one agent that manages tool selection across domains, you create focused specialists coordinated by a supervisor who understands the overall workflow.

**Supervisor 模式**是一种多代理架构，由一个中央 supervisor 代理协调专门的 worker 代理。当任务需要不同类型的专业知识时，这种方法表现出色。你不需要构建一个跨领域管理工具选择的代理，而是创建由理解整体工作流的 supervisor 协调的专注专家。

In this tutorial, you'll build a personal assistant system: 本教程将构建个人助理系统：

* A **calendar agent** that handles scheduling, availability checking, and event management — **日历代理**：处理调度、可用性检查和事件管理
* An **email agent** that manages communication, drafts messages, and sends notifications — **邮件代理**：管理通信、起草消息和发送通知

We will also incorporate **human-in-the-loop review** to allow users to approve, edit, and reject actions.
还会加入**人在回路审查**，允许用户批准、编辑和拒绝操作。

### Why use a supervisor? / 为什么用 supervisor？

Multi-agent architectures allow you to partition tools across workers, each with their own individual prompts or instructions. Consider an agent with direct access to all calendar and email APIs: it must choose from many similar tools, understand exact formats for each API, and handle multiple domains simultaneously.

多代理架构允许你将工具分配给不同的 worker，每个 worker 有自己的提示词。一个直接访问所有日历和邮件 API 的代理需要从许多相似工具中选择、理解每个 API 的精确格式、同时处理多个领域。

## 1. Define tools / 定义工具

```python
from langchain.tools import tool

@tool
def create_calendar_event(
    title: str,
    start_time: str,       # ISO 格式: "2024-01-15T14:00:00"
    end_time: str,         # ISO 格式
    attendees: list[str],  # 邮箱地址
    location: str = ""
) -> str:
    """Create a calendar event. Requires exact ISO datetime format."""
    return f"Event created: {title} from {start_time} to {end_time}"

@tool
def send_email(
    to: list[str],
    subject: str,
    body: str,
    cc: list[str] = []
) -> str:
    """Send an email via email API."""
    return f"Email sent to {', '.join(to)} - Subject: {subject}"

@tool
def get_available_time_slots(
    attendees: list[str],
    date: str,               # ISO 格式: "2024-01-15"
    duration_minutes: int
) -> list[str]:
    """Check calendar availability for given attendees on a specific date."""
    return ["09:00", "14:00", "16:00"]
```

## 2. Create specialized sub-agents / 创建专门的子代理

### Calendar Agent / 日历代理

The calendar agent understands natural language scheduling requests and translates them into precise API calls.
日历代理理解自然语言调度请求并将其转换为精确的 API 调用。

```python
from langchain.agents import create_agent

CALENDAR_AGENT_PROMPT = (
    "You are a calendar scheduling assistant. "
    "Parse natural language scheduling requests into proper ISO datetime formats. "
    "Use get_available_time_slots to check availability when needed. "
    "Use create_calendar_event to schedule events. "
    "Always confirm what was scheduled in your final response."
)

calendar_agent = create_agent(
    model,
    tools=[create_calendar_event, get_available_time_slots],
    system_prompt=CALENDAR_AGENT_PROMPT,
)
```

代理将 "next Tuesday at 2pm" 解析为 ISO 格式 "2024-06-18T14:00:00"，计算结束时间，调用 `create_calendar_event`，返回自然语言确认。

### Email Agent / 邮件代理

```python
EMAIL_AGENT_PROMPT = (
    "You are an email assistant. "
    "Compose professional emails based on natural language requests. "
    "Extract recipient information and craft appropriate subject lines and body text. "
    "Use send_email to send the message. "
    "Always confirm what was sent in your final response."
)

email_agent = create_agent(
    model,
    tools=[send_email],
    system_prompt=EMAIL_AGENT_PROMPT,
)
```

## 3. Wrap sub-agents as tools / 将子代理包装为工具

Now wrap each sub-agent as a tool that the supervisor can invoke. This is the key architectural step.
将每个子代理包装为 supervisor 可以调用的工具。这是关键的架构步骤。

```python
@tool
def schedule_event(request: str) -> str:
    """Schedule calendar events using natural language.

    Use this when the user wants to create, modify, or check calendar appointments.
    Input: Natural language scheduling request
    """
    result = calendar_agent.invoke({"messages": [{"role": "user", "content": request}]})
    return result["messages"][-1].text

@tool
def manage_email(request: str) -> str:
    """Send emails using natural language.

    Use this when the user wants to send notifications, reminders, or any email.
    Input: Natural language email request
    """
    result = email_agent.invoke({"messages": [{"role": "user", "content": request}]})
    return result["messages"][-1].text
```

The supervisor will see high-level tools like "schedule_event", not low-level tools like "create_calendar_event".
Supervisor 看到的是高层工具如 "schedule_event"，而非底层工具如 "create_calendar_event"。

## 4. Create the supervisor agent / 创建 supervisor 代理

```python
SUPERVISOR_PROMPT = (
    "You are a helpful personal assistant. "
    "You can schedule calendar events and send emails. "
    "Break down user requests into appropriate tool calls and coordinate the results. "
    "When a request involves multiple actions, use multiple tools in sequence."
)

supervisor_agent = create_agent(
    model,
    tools=[schedule_event, manage_email],
    system_prompt=SUPERVISOR_PROMPT,
)
```

## 5. Use the supervisor / 使用 supervisor

### Example 1: Single-domain / 单领域

```
用户: "Schedule a team standup for tomorrow at 9am"
    │
    ▼ Supervisor
    ├── schedule_event("team standup tomorrow at 9am")
    │       └── Calendar Agent → get_available → create_event → 确认
    └── 最终回复: "The team standup is scheduled for tomorrow at 9:00 AM"
```

### Example 2: Multi-domain / 跨领域

```
用户: "Schedule a meeting with design team next Tuesday at 2pm,
       and send them an email reminder about reviewing mockups"
    │
    ▼ Supervisor (并行调用两个工具)
    ├── schedule_event("meeting with design team next Tuesday at 2pm")
    │       └── Calendar Agent → create_event → 确认
    ├── manage_email("send design team reminder about mockups")
    │       └── Email Agent → send_email → 确认
    └── 最终回复: 综合两个结果
```

## 6. Human-in-the-loop / 人在回路

Add review to both sub-agents for sensitive actions.
为两个子代理的敏感操作添加审查。

```python
from langchain.agents.middleware import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import InMemorySaver

calendar_agent = create_agent(
    model, tools=[create_calendar_event, get_available_time_slots],
    system_prompt=CALENDAR_AGENT_PROMPT,
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={"create_calendar_event": True},
            description_prefix="Calendar event pending approval",
        ),
    ],
)

email_agent = create_agent(
    model, tools=[send_email],
    system_prompt=EMAIL_AGENT_PROMPT,
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={"send_email": True},
            description_prefix="Outbound email pending approval",
        ),
    ],
)

# checkpointer 只加在顶层 supervisor
supervisor_agent = create_agent(
    model, tools=[schedule_event, manage_email],
    system_prompt=SUPERVISOR_PROMPT,
    checkpointer=InMemorySaver(),
)
```

### Approve / 批准

```python
from langgraph.types import Command

for step in supervisor_agent.stream(
    Command(resume={interrupt_id: {"decisions": [{"type": "approve"}]}}),
    config,
):
    ...
```

### Edit / 编辑

```python
edited_action = interrupt_.value["action_requests"][0].copy()
```

## 7. Advanced: Control information flow / 高级：控制信息流

### Pass context to sub-agents / 向子代理传递上下文

```python
from langchain.tools import tool, ToolRuntime

@tool
def schedule_event(request: str, runtime: ToolRuntime) -> str:
    """Schedule calendar events using natural language."""
    original_user_message = next(
        message for message in runtime.state["messages"]
        if message.type == "human"
    )
    prompt = (
        "You are assisting with the following user inquiry:\n\n"
        f"{original_user_message.text}\n\n"
        "You are tasked with the following sub-request:\n\n"
        f"{request}"
    )
    result = calendar_agent.invoke({"messages": [{"role": "user", "content": prompt}]})
    return result["messages"][-1].text
```

This allows sub-agents to see the full conversation context for resolving ambiguities.
这允许子代理看到完整对话上下文，以解决歧义。

## Key takeaways / 核心要点

* **分层抽象** — 每层有明确职责
* **域隔离** — 不同领域的工具和提示词分开
* **逐层测试** — 独立测试每个子代理后再集成
* **信息流控制** — 根据需要定制子代理收到的上下文

**When to use:** 适用场景
- 多个不同领域（日历、邮件、CRM、数据库）
- 每个领域有多个工具或复杂逻辑
- 需要集中式工作流控制

**When NOT to use:** 不适用场景
- 只有几个工具 → 用单个代理
- 代理需要直接与用户对话 → 用 handoffs
- 代理间需要对等协作 → 其他多代理模式
