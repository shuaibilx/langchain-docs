# Skills SQL Assistant — 流程详细总结

## 核心概念：渐进式披露 (Progressive Disclosure)

```
┌─────────────────────────────────────────────────────────┐
│                   系统提示词 (始终存在)                    │
│                                                         │
│  "You are a SQL query assistant."                       │
│                                                         │
│  ## Available Skills                                    │
│  - **sales_analytics**: 销售数据分析 schema 和业务逻辑     │
│  - **inventory_management**: 库存管理 schema 和业务逻辑   │
│                                                         │
│  "Use the load_skill tool when you need details."       │
│                                                         │
│  ↑ 只有 name + description，没有完整内容                   │
└─────────────────────────────────────────────────────────┘

                    用户提问后...

┌─────────────────────────────────────────────────────────┐
│                   ToolMessage (按需加载)                  │
│                                                         │
│  load_skill("sales_analytics") →                        │
│                                                         │
│  # Sales Analytics Schema                               │
│  ## Tables                                              │
│  ### customers: customer_id, name, email, ...           │
│  ### orders: order_id, customer_id, total_amount, ...   │
│  ## Business Logic                                      │
│  - High-value: total_amount > 1000                      │
│  - Revenue: status = 'completed'                        │
│  ## Example Query                                       │
│  SELECT ... FROM customers JOIN orders ...               │
│                                                         │
│  ↑ 完整内容，只在需要时加载                                │
└─────────────────────────────────────────────────────────┘
```

## 三层架构

```
Layer 1: Metadata (始终在系统提示词中)
┌──────────────────────────────────────┐
│ skill_name: "sales_analytics"        │
│ description: "销售数据分析 schema"     │
│                                      │
│ → 约 20 tokens                       │
│ → 帮助代理判断是否需要此技能           │
└──────────────────────────────────────┘
                │
                │ 代理判断需要 → tool_call: load_skill()
                ▼
Layer 2: Core Content (按需加载)
┌──────────────────────────────────────┐
│ 表结构 (customers, orders, items)    │
│ 业务逻辑 (活跃客户、收入计算)          │
│ 示例查询                              │
│                                      │
│ → 约 500-2000 tokens                 │
│ → 提供足够的上下文来完成任务           │
└──────────────────────────────────────┘
                │
                │ 可选：进一步探索
                ▼
Layer 3: Detailed Resources (按需深入)
┌──────────────────────────────────────┐
│ 复杂 JOIN 示例                        │
│ 边缘情况处理                          │
│ 性能优化建议                          │
│                                      │
│ → 按需加载，不消耗初始上下文           │
└──────────────────────────────────────┘
```

## 关键组件

### 1. Skill 数据结构

```python
class Skill(TypedDict):
    name: str        # 唯一标识
    description: str # 轻量描述（系统提示词中）
    content: str     # 完整内容（按需加载）
```

### 2. load_skill 工具

```python
@tool
def load_skill(skill_name: str) -> str:
    for skill in SKILLS:
        if skill["name"] == skill_name:
            return f"Loaded skill: {skill_name}\n\n{skill['content']}"
    return f"Skill not found. Available: {available}"
```

返回完整内容 → 变成 ToolMessage → 进入对话历史 → 代理可以引用。

### 3. SkillMiddleware

```python
class SkillMiddleware(AgentMiddleware):
    tools = [load_skill]  # 注册工具

    def __init__(self):
        # 构建轻量描述列表
        self.skills_prompt = "\n".join(
            f"- **{s['name']}**: {s['description']}" for s in SKILLS
        )

    def wrap_model_call(self, request, handler):
        # 注入到系统提示词
        addendum = f"\n## Available Skills\n{self.skills_prompt}\nUse load_skill when needed."
        return handler(request.override(system_message=...))
```

**两个职责：**
1. 将技能描述注入系统提示词（发现）
2. 注册 load_skill 工具（加载）

### 4. 带约束的高级版本

```
用户: "验证这个 SQL 查询"
    │
    ▼ Agent 调用 write_sql_query(query, vertical="sales_analytics")
    │
    ▼ 检查 runtime.state["skills_loaded"]
    │
    ├── "sales_analytics" in skills_loaded → ✅ 执行验证
    └── "sales_analytics" not in skills_loaded → ❌ "先加载技能"
```

通过 Command 更新 state：

```python
@tool
def load_skill(skill_name, runtime) -> Command:
    return Command(update={
        "messages": [ToolMessage(...)],
        "skills_loaded": [skill_name],  # 记录已加载
    })
```

## 与其他模式的关系

### Skills vs RAG

| 维度 | Skills (渐进式披露) | RAG |
|------|-------------------|-----|
| 检索单位 | 技能（完整的 schema + 逻辑） | 文档块（文本片段） |
| 检索方式 | 工具调用（精确匹配） | 嵌入相似度 / 关键词 |
| 内容结构 | 结构化（表、字段、规则） | 非结构化文本 |
| 适用场景 | 已知领域、结构化知识 | 大量非结构化文档 |

### Skills vs DeepAgent Skills

| 维度 | LangChain Skills (本教程) | DeepAgent Skills |
|------|--------------------------|-----------------|
| 实现方式 | 手动 @tool + AgentMiddleware | `skills=["./skills/"]` 自动加载 |
| 存储 | 内存中的 Python 字典 | 文件系统 SKILL.md |
| 发现 | 系统提示词列出描述 | frontmatter 描述匹配 |
| 加载 | load_skill 工具调用 | 自动渐进式披露 |
| 适用场景 | 需要精细控制 | 开箱即用 |

### Skills vs Subagents/Router

| 维度 | Skills | Subagents/Router |
|------|--------|-----------------|
| 代理数量 | 单个 | 多个 |
| 知识加载 | 注入到同一上下文 | 各代理独立上下文 |
| 并行性 | 不适用 | 可并行 |
| 适用场景 | 知识量可控 | 知识量大、需要隔离 |

## 执行流程图

```
用户: "找出上个月订单超过 $1000 的客户"
    │
    ▼ Agent 读取系统提示词
    │  看到: "Available Skills: sales_analytics, inventory_management"
    │  判断: 需要销售数据 → sales_analytics
    │
    ▼ tool_call: load_skill("sales_analytics")
    │
    ▼ ToolMessage 返回:
    │  - customers 表: customer_id, name, email, status, customer_tier
    │  - orders 表: order_id, customer_id, total_amount, status
    │  - 业务逻辑: high-value = total_amount > 1000
    │  - 业务逻辑: revenue = status = 'completed'
    │
    ▼ Agent 基于 schema 写 SQL:
    │  SELECT DISTINCT c.customer_id, c.name
    │  FROM customers c
    │  JOIN orders o ON c.customer_id = o.customer_id
    │  WHERE o.total_amount > 1000
    │    AND o.status = 'completed'
    │    AND o.order_date >= CURRENT_DATE - INTERVAL '1 month'
    │
    ▼ 返回正确 SQL（遵循业务逻辑）
```

## 设计原则

| 原则 | 说明 |
|------|------|
| 轻量描述 | description 只需 1-2 句，帮助代理判断相关性 |
| 完整内容 | content 包含所有需要的信息，避免多次加载 |
| 按需加载 | 不预加载，只在代理判断需要时才加载 |
| 工具驱动 | 通过工具调用加载，而非动态修改系统提示词 |
| 状态追踪 | 可选：用 state 记录已加载技能，实现约束 |
