# Learn 模块总览 — 流程详细总结

## 模块定位

Learn 是 LangChain 官方文档的**实战教学模块**，位于文档体系的第三层：

```
文档体系
├── Get started     ← 入门（安装、快速开始、理念）
├── Core components ← 核心组件（Models、Tools、Agents...）
├── Learn           ← 实战教程（本模块）
└── Reference       ← API 参考
```

## 一、Use Cases — 实战用例

### 1.1 Deep Agents（深度代理）

Deep Agents 提供**内置能力**：上下文管理、虚拟文件系统、通用代理需求。

#### Data analysis（数据分析）
- **目标**：构建数据分析师代理，将报告发送到 Slack
- **核心流程**：
  ```
  用户数据 → 代理分析 → 生成报告 → 发送 Slack
  ```
- **关键能力**：数据处理、报告生成、Slack 集成

#### Deep research（深度研究）
- **目标**：构建多步骤网络研究代理
- **核心流程**：
  ```
  研究问题 → 战略规划 → 子代理委派 → 信息收集 → 反思 → 综合报告
  ```
- **关键能力**：子代理委派（subagent delegation）、战略反思（strategic reflection）

### 1.2 LangChain（简单场景）

LangChain agent 实现适合**简单用例**，快速上手。

#### Semantic Search（语义搜索）
- **目标**：基于 PDF 构建语义搜索引擎
- **核心流程**：
  ```
  PDF 文档 → 文本分割 → 向量嵌入 → 向量存储 → 语义检索
  ```
- **关键组件**：DocumentLoader、TextSplitter、Embeddings、VectorStore

#### RAG Agent（RAG 代理）
- **目标**：创建检索增强生成代理
- **核心流程**：
  ```
  用户问题 → 检索器(retriever) → 获取相关文档 → LLM 生成回答
  ```
- **关键 API**：`create_retrieval_chain`、`create_stuff_documents_chain`

#### SQL Agent（SQL 代理）
- **目标**：构建与数据库交互的 SQL 代理，支持人工审核
- **核心流程**：
  ```
  自然语言问题 → 生成 SQL → 人工审核 → 执行查询 → 返回结果
  ```
- **关键特性**：Human-in-the-loop（人工审核）

#### Voice Agent（语音代理）
- **目标**：构建可语音交互的代理
- **核心流程**：
  ```
  语音输入 → STT(语音转文本) → Agent 处理 → TTS(文本转语音) → 语音输出
  ```

### 1.3 LangGraph（自定义场景）

LangGraph 提供**细粒度控制**，适合深度定制。

#### Custom RAG Agent
- **与 LangChain RAG 的区别**：使用 LangGraph 原语（StateGraph、节点、边）
- **优势**：可精确控制检索流程的每一步

#### Custom SQL Agent
- **与 LangChain SQL Agent 的区别**：直接在 LangGraph 中实现
- **优势**：最大灵活性，可自定义查询生成、执行、错误处理流程

## 二、Multi-agent — 多代理模式

四种经典多代理模式：

### 2.1 Subagents（子代理）— 个人助手
```
用户请求
    │
    ▼
主代理（协调者）
    ├── 子代理 A（日程管理）
    ├── 子代理 B（邮件处理）
    └── 子代理 C（信息检索）
```
- **模式**：委派（delegation）
- **特点**：主代理将任务分配给专业子代理

### 2.2 Handoffs（切换）— 客户支持
```
客户咨询
    │
    ▼
客服代理 ──handoff──→ 技术支持代理
    │                      │
    ▼                      ▼
  一般问题              技术问题
```
- **模式**：状态转换（state transition）
- **特点**：单代理在不同状态间平滑切换

### 2.3 Router（路由器）— 知识库
```
用户查询
    │
    ▼
路由器代理
    ├──→ 知识库 A（产品文档）
    ├──→ 知识库 B（技术文档）
    └──→ 知识库 C（FAQ）
```
- **模式**：路由（routing）
- **特点**：根据查询内容路由到专业代理

### 2.4 Skills（技能）— SQL 助手
```
用户请求
    │
    ▼
SQL 助手代理
    ├── 按需加载：数据库 Schema 技能
    ├── 按需加载：查询优化技能
    └── 按需加载：数据可视化技能
```
- **模式**：按需上下文加载（on-demand context loading）
- **特点**：渐进式加载专业技能，减少 token 消耗

## 三、Conceptual overviews — 概念指南

| 概念 | 核心问题 | 关键点 |
|------|---------|--------|
| **Memory** | 如何持久化交互？ | 线程内/线程间的记忆管理 |
| **Context engineering** | 如何给 AI 正确信息？ | 信息选择、工具配置、上下文窗口管理 |
| **Graph API** | 如何声明式构建图？ | StateGraph、节点定义、边的条件路由 |
| **Functional API** | 如何用函数构建代理？ | 单函数封装，简化开发 |
