# Learn 模块总览 — 类型转换举例

## A. 各用例数据生命周期转换

### 1. RAG Agent 数据流

**生命周期：** `用户问题: str` → `RAG 回答: str + 引用文档: list[Document]`

```
【阶段 1：检索】
输入:  question = "什么是 LangChain 的 Memory？"
       ↓ retriever.invoke(question)
输出:  docs = [Document(page_content="Memory 是...", metadata={...}), ...]

【阶段 2：组装上下文】
输入:  docs: list[Document]
       ↓ create_stuff_documents_chain
输出:  context = "Memory 是...\n\n持久化交互..."
       prompt = "根据以下上下文回答：\n{context}\n\n问题：{input}"

【阶段 3：生成】
输入:  prompt: str → ChatPromptValue → messages: list[BaseMessage]
       ↓ llm.invoke(messages)
输出:  answer = "Memory 是 LangGraph 中的持久化机制..."

【最终输出】
response = {
    "input": "什么是 LangChain 的 Memory？",
    "context": [Document(...), Document(...)],
    "answer": "Memory 是 LangGraph 中的持久化机制..."
}
```

### 2. SQL Agent 数据流

**生命周期：** `自然语言问题: str` → `查询结果: list[dict] + 自然语言解释: str`

```
【阶段 1：生成 SQL】
输入:  question = "上个月销售额最高的产品是什么？"
       ↓ llm.invoke(prompt_with_schema)
输出:  sql = "SELECT product_name, SUM(amount) FROM orders
              WHERE date >= '2026-05-01' GROUP BY product_name
              ORDER BY SUM(amount) DESC LIMIT 1"

【阶段 2：人工审核（可选）】
输入:  sql: str
       ↓ human_review
输出:  approved_sql: str  (可能被修改)

【阶段 3：执行查询】
输入:  approved_sql: str
       ↓ db.run(sql)
输出:  results = [{"product_name": "Widget A", "sum": 15000}]

【阶段 4：生成解释】
输入:  question: str + results: list[dict]
       ↓ llm.invoke(...)
输出:  answer = "上个月销售额最高的产品是 Widget A，总销售额为 15,000 元。"
```

### 3. Semantic Search 数据流

**生命周期：** `PDF 文件路径: str` → `搜索结果: list[Document]`

```
【阶段 1：加载文档】
输入:  file_path = "report.pdf"
       ↓ PyPDFLoader.load()
输出:  pages = [Document(page_content="第1页内容..."), ...]

【阶段 2：分割文本】
输入:  pages: list[Document]
       ↓ RecursiveCharacterTextSplitter(chunk_size=1000)
输出:  chunks = [Document(page_content="片段1..."), ...]
       # 10 页 PDF → ~50 个片段

【阶段 3：向量嵌入】
输入:  chunks: list[Document]
       ↓ embeddings.embed_documents([c.page_content for c in chunks])
输出:  vectors = [[0.012, -0.034, ...], [0.056, 0.078, ...], ...]
       # 每个片段 → 1536 维向量

【阶段 4：存储】
输入:  texts: list[str] + embeddings: list[list[float]] + metadatas: list[dict]
       ↓ vectorstore.add_texts()
输出:  ids = ["doc_001", "doc_002", ...]

【阶段 5：检索】
输入:  query = "机器学习的应用场景"
       ↓ vectorstore.similarity_search(query, k=5)
输出:  results = [Document(page_content="机器学习在...", metadata={...}), ...]
```

### 4. Deep Research Agent 数据流

**生命周期：** `研究问题: str` → `研究报告: str`

```
【阶段 1：战略规划】
输入:  question = "2026 年 AI Agent 的发展趋势"
       ↓ planner_agent.invoke()
输出:  plan = ["1. 搜索最新论文", "2. 分析行业报告", "3. 整理关键趋势"]

【阶段 2：子代理委派】
输入:  plan: list[str]
       ↓ subagent.invoke(task)
输出:  findings = [
         {"topic": "最新论文", "content": "..."},
         {"topic": "行业报告", "content": "..."},
         {"topic": "关键趋势", "content": "..."}
       ]

【阶段 3：反思】
输入:  findings: list[dict]
       ↓ reflector_agent.invoke()
输出:  gaps = ["需要补充：边缘计算领域的 AI Agent 应用"]
       refined_findings: list[dict]

【阶段 4：综合报告】
输入:  refined_findings: list[dict]
       ↓ synthesizer_agent.invoke()
输出:  report = "# 2026 年 AI Agent 发展趋势\n\n## 1. ..."
```

### 5. Multi-agent: Router 数据流

**生命周期：** `用户查询: str` → `路由结果: str + 来源代理: str`

```
【阶段 1：路由判断】
输入:  query = "如何配置 LangChain 的 API Key？"
       ↓ router.invoke()
输出:  route = "technical_docs"
       # 可选路由: product_docs | technical_docs | faq

【阶段 2：专业代理处理】
输入:  query: str + context: str
       ↓ technical_agent.invoke()
输出:  answer = "在 .env 文件中设置 LANGCHAIN_API_KEY=..."

【最终输出】
response = {
    "query": "如何配置 LangChain 的 API Key？",
    "route": "technical_docs",
    "answer": "在 .env 文件中设置 LANGCHAIN_API_KEY=...",
    "source": "技术文档代理"
}
```

## B. 各模式核心类型转换速查

### RAG 模式

| 步骤 | 转换 | 说明 |
|------|------|------|
| 检索 | `str` → `list[Document]` | 问题变成文档片段列表 |
| 组装 | `list[Document]` → `str` | 文档拼接成上下文字符串 |
| 生成 | `str` → `str` | 上下文+问题 → 回答 |

### SQL 模式

| 步骤 | 转换 | 说明 |
|------|------|------|
| 生成 SQL | `str` → `str` | 自然语言 → SQL 语句 |
| 执行 | `str` → `list[dict]` | SQL → 查询结果 |
| 解释 | `list[dict]` → `str` | 结构化数据 → 自然语言 |

### 向量搜索模式

| 步骤 | 转换 | 说明 |
|------|------|------|
| 分割 | `list[Document]` → `list[Document]` | 大文档 → 小片段 |
| 嵌入 | `list[str]` → `list[list[float]]` | 文本 → 向量 |
| 检索 | `str` → `list[Document]` | 查询 → 相似文档 |

### 多代理路由模式

| 步骤 | 转换 | 说明 |
|------|------|------|
| 路由 | `str` → `str` | 查询 → 路由目标名称 |
| 分发 | `str` → `AgentOutput` | 路由 → 专业代理输出 |
| 汇总 | `AgentOutput` → `str` | 代理输出 → 最终回答 |
