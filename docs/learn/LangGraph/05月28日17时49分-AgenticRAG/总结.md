# Agentic RAG with LangGraph — 流程详细总结

## 图结构

```
START
  │
  ▼
generate_query_or_respond
  │
  ├── 有 tool_calls ──→ retrieve ──→ grade_documents
  │                                      │
  │                              ┌───────┴───────┐
  │                              ▼               ▼
  │                        generate_answer  rewrite_question
  │                              │               │
  │                              ▼               ▼
  │                            END    ──→ generate_query_or_respond (循环)
  │
  └── 无 tool_calls ──→ END
```

## 四个节点详解

### Node 1: generate_query_or_respond

```python
def generate_query_or_respond(state: MessagesState):
    response = response_model.bind_tools([retriever_tool]).invoke(state["messages"])
    return {"messages": [response]}
```

| 输入 | 决策 | 输出 |
|------|------|------|
| 用户消息 | LLM 判断是否需要检索 | tool_call 或直接回复 |

**关键：** `bind_tools` 让 LLM 可以选择调用检索工具。

### Node 2: retrieve

```python
ToolNode([retriever_tool])
```

| 输入 | 操作 | 输出 |
|------|------|------|
| tool_call | 执行 similarity_search | ToolMessage (文档内容) |

**关键：** 使用 LangGraph 预构建的 `ToolNode`。

### Node 3: grade_documents (条件边)

```python
def grade_documents(state: MessagesState) -> Literal["generate_answer", "rewrite_question"]:
    response = grader_model.with_structured_output(GradeDocuments).invoke([...])
    return "generate_answer" if response.binary_score == "yes" else "rewrite_question"
```

| 输入 | 评估 | 输出 |
|------|------|------|
| 问题 + 检索结果 | LLM 判断相关性 | 路由到下一节点 |

**关键：** `with_structured_output(GradeDocuments)` 强制输出 "yes" 或 "no"。

### Node 4: rewrite_question

```python
def rewrite_question(state: MessagesState):
    response = response_model.invoke([{"role": "user", "content": REWRITE_PROMPT}])
    return {"messages": [HumanMessage(content=response.content)]}
```

| 输入 | 操作 | 输出 |
|------|------|------|
| 原始问题 | LLM 改写为更好的查询 | 新的 HumanMessage |

**关键：** 返回 `HumanMessage`，模拟用户重新提问。

## 两条路径

### 路径 A: 直接回复（简单问题）

```
"hello!" → generate_query_or_respond → 直接回复 → END
```

不需要检索，LLM 直接回答。

### 路径 B: 检索 + 回答（复杂问题）

```
"What about reward hacking?"
→ generate_query_or_respond (tool_call)
→ retrieve (检索文档)
→ grade_documents ("yes")
→ generate_answer
→ END
```

### 路径 C: 检索 + 不相关 + 重试

```
"What about reward hacking?"
→ generate_query_or_respond (tool_call)
→ retrieve (检索到不相关文档)
→ grade_documents ("no")
→ rewrite_question (改写查询)
→ generate_query_or_respond (重新检索)
→ retrieve (检索到相关文档)
→ grade_documents ("yes")
→ generate_answer
→ END
```

## 关键技术点

| 技术 | 用途 | 代码 |
|------|------|------|
| `bind_tools` | 让 LLM 可以调用工具 | `model.bind_tools([tool])` |
| `with_structured_output` | 强制 LLM 输出指定格式 | `model.with_structured_output(GradeDocuments)` |
| `ToolNode` | 预构建的工具执行节点 | `ToolNode([tool])` |
| `conditional_edges` | 条件路由 | `workflow.add_conditional_edges(...)` |
| `MessagesState` | 消息状态管理 | `StateGraph(MessagesState)` |

## 与 LangChain RAG Agent 的对比

| 维度 | LangChain RAG Agent | LangGraph Agentic RAG |
|------|--------------------|-----------------------|
| 实现方式 | `create_agent` + 工具 | StateGraph + 节点 + 边 |
| 文档评估 | 无（信任检索结果） | grade_documents 条件边 |
| 查询重写 | 无 | rewrite_question 节点 |
| 自循环 | 不支持 | rewrite → generate_query 循环 |
| 控制粒度 | 中 | 高 |
