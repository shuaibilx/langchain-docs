# Custom SQL Agent with LangGraph — 流程详细总结

## 图结构

```
START
  │
  ▼
list_tables (强制执行)
  │
  ▼
call_get_schema (tool_choice="any" 强制调用)
  │
  ▼
get_schema (ToolNode)
  │
  ▼
generate_query (LLM 自主决定)
  │
  ├── 有 tool_calls ──→ check_query (强制调用)
  │                          │
  │                          ▼
  │                     run_query (ToolNode)
  │                          │
  │                          ▼
  │                   generate_query (循环)
  │
  └── 无 tool_calls ──→ END
```

## 六个节点详解

### Node 1: list_tables — 强制执行

```python
def list_tables(state):
    tool_call = {"name": "sql_db_list_tables", "args": {}, "id": "abc123", "type": "tool_call"}
    tool_call_message = AIMessage(content="", tool_calls=[tool_call])
    tool_message = list_tables_tool.invoke(tool_call)
    response = AIMessage(f"Available tables: {tool_message.content}")
    return {"messages": [tool_call_message, tool_message, response]}
```

**关键：** 不经过 LLM，直接构造消息。确保第一步一定是列出表。

### Node 2: call_get_schema — 强制工具调用

```python
def call_get_schema(state):
    llm_with_tools = model.bind_tools([get_schema_tool], tool_choice="any")
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}
```

**关键：** `tool_choice="any"` 强制 LLM 必须调用工具。

### Node 3: get_schema — ToolNode

```python
get_schema_node = ToolNode([get_schema_tool], name="get_schema")
```

### Node 4: generate_query — LLM 自主决定

```python
def generate_query(state):
    system_message = {"role": "system", "content": "You are an agent... create SQL queries..."}
    llm_with_tools = model.bind_tools([run_query_tool])
    response = llm_with_tools.invoke([system_message] + state["messages"])
    return {"messages": [response]}
```

**关键：** 不强制工具调用。LLM 可能直接回答简单问题，也可能生成 SQL。

### Node 5: check_query — LLM 检查 SQL

```python
def check_query(state):
    tool_call = state["messages"][-1].tool_calls[0]  # 提取上一步的 SQL
    user_message = {"role": "user", "content": tool_call["args"]["query"]}
    llm_with_tools = model.bind_tools([run_query_tool], tool_choice="any")
    response = llm_with_tools.invoke([system_message, user_message])
    return {"messages": [response]}
```

**关键：** 从 tool_call 提取 SQL，让 LLM 检查后强制执行。

### Node 6: run_query — ToolNode

```python
run_query_node = ToolNode([run_query_tool], name="run_query")
```

## 节点设计模式

| 节点 | LLM 参与 | 工具调用 | 设计模式 |
|------|---------|---------|---------|
| list_tables | 否 | 直接构造 | 程序化 |
| call_get_schema | 是 | 强制 (any) | LLM + 强制工具 |
| get_schema | 否 | ToolNode | 工具执行 |
| generate_query | 是 | 自主决定 | LLM 自由 |
| check_query | 是 | 强制 (any) | LLM + 强制工具 |
| run_query | 否 | ToolNode | 工具执行 |

## 强制 vs 自主

```
强制执行（程序控制）          自主决定（LLM 控制）
┌─────────────────┐         ┌─────────────────┐
│ list_tables     │         │ generate_query  │
│ call_get_schema │         │                 │
│ check_query     │         │ LLM 决定:       │
│                 │         │ - 生成 SQL?     │
│ 代码决定顺序     │         │ - 直接回答?     │
└─────────────────┘         └─────────────────┘
```

## Human-in-the-loop 实现

### interrupt 函数

```python
response = interrupt([request])
# response = {"type": "accept"}           → 执行原始查询
# response = {"type": "edit", "args": {}} → 修改后执行
# response = {"type": "response", "args": "feedback"} → 返回用户反馈
```

### 与预构建代理的 HITL 对比

| 维度 | 预构建代理 HITL | LangGraph HITL |
|------|---------------|----------------|
| 中断方式 | HumanInTheLoopMiddleware | interrupt() 函数 |
| 恢复方式 | Command(resume={decisions}) | Command(resume={type, args}) |
| 粒度 | 工具级 | 函数级 |
| 灵活性 | 中 | 高 |

## 与预构建 SQL Agent 的对比

| 维度 | 预构建 (create_agent) | LangGraph 自定义 |
|------|---------------------|-----------------|
| 表列出 | LLM 自主决定 | 强制第一步 |
| Schema 获取 | LLM 自主决定 | 强制第二步 |
| 查询检查 | query_checker 工具（可选） | check_query 节点（强制） |
| 错误处理 | LLM 重试 | run_query → generate_query 循环 |
| 人在回路 | HumanInTheLoopMiddleware | interrupt 函数 |
| 控制程度 | 低（提示词约束） | 高（代码强制） |
