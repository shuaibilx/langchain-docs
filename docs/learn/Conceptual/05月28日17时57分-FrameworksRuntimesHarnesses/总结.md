# Frameworks, Runtimes, Harnesses — 总结

## 三层架构速查

```
┌─────────────────────────────────────────────────────────┐
│  Layer 3: Harness (工具套件)                             │
│  Deep Agents SDK / Claude Agent SDK / Manus             │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 预定义工具 + 提示词 + 子代理 + 文件系统 + 规划    │    │
│  │ 适用: 复杂多步骤任务、长时间运行代理              │    │
│  └─────────────────────────────────────────────────┘    │
│                        构建在 ↓                          │
├─────────────────────────────────────────────────────────┤
│  Layer 2: Runtime (运行时)                               │
│  LangGraph / Temporal / Inngest                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 持久执行 + 流式 + HITL + 持久化 + 低级控制       │    │
│  │ 适用: 生产环境、有状态工作流、复杂编排            │    │
│  └─────────────────────────────────────────────────┘    │
│                        构建在 ↓                          │
├─────────────────────────────────────────────────────────┤
│  Layer 1: Framework (框架)                               │
│  LangChain / Vercel AI SDK / CrewAI / OpenAI Agents SDK │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 抽象 + 集成 + 标准接口                           │    │
│  │ 适用: 快速启动、标准化构建                       │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

## 选型决策树

```
你需要什么？
│
├── 快速启动、标准化 → Framework (LangChain)
│
├── 低级控制、持久执行、生产部署 → Runtime (LangGraph)
│
└── 自主代理、复杂任务、开箱即用 → Harness (Deep Agents SDK)
```

## 关键区别

| 维度 | Framework | Runtime | Harness |
|------|-----------|---------|---------|
| 抽象级别 | 高 | 低 | 中 |
| 控制粒度 | 粗 | 细 | 中 |
| 上手难度 | 低 | 高 | 中 |
| 生产就绪 | 中 | 高 | 高 |
| 自主性 | 低 | 中 | 高 |
