# Memory Overview — 逐段翻译

> 原文：https://docs.langchain.com/oss/concepts/memory

---

Memory is a system that remembers information about previous interactions. For AI agents, memory is crucial because it lets them remember previous interactions, learn from feedback, and adapt to user preferences.

记忆是记住先前交互信息的系统。对 AI 代理来说，记忆至关重要，因为它让代理记住先前交互、从反馈中学习、适应用户偏好。

## Two types of memory / 两种记忆

| 类型 | 作用域 | 说明 |
|------|--------|------|
| **Short-term memory** | 线程内 | 维护会话内的消息历史 |
| **Long-term memory** | 跨线程 | 跨会话存储用户或应用级数据 |

## Short-term memory / 短期记忆

Short-term memory lets your application remember previous interactions within a single thread. LangGraph manages short-term memory as part of the agent's state, persisted via thread-scoped checkpoints.

短期记忆让应用在单个线程内记住先前交互。LangGraph 将短期记忆作为代理状态的一部分管理，通过线程范围的检查点持久化。

### Challenge: Long conversations / 挑战：长对话

* Full history may not fit in context window — 完整历史可能超出上下文窗口
* LLMs perform poorly over long contexts — LLM 在长上下文下表现差
* Stale content causes distraction — 过时内容导致分心
* Higher costs with more tokens — 更多 token 意味着更高成本

**Solutions:** 摘要、裁剪、选择性保留等技术。

## Long-term memory / 长期记忆

Long-term memory stores information across different conversations or sessions, saved within custom "namespaces."

长期记忆跨不同对话或会话存储信息，保存在自定义"命名空间"中。

### Three memory types (from human memory) / 三种记忆类型

| Memory Type | What is Stored | Human Example | Agent Example |
|-------------|---------------|---------------|---------------|
| **Semantic** | Facts | 学校学到的知识 | 用户偏好、事实 |
| **Episodic** | Experiences | 做过的事情 | 过去的代理操作 |
| **Procedural** | Instructions | 骑自行车的技能 | 系统提示词 |

### Semantic memory / 语义记忆

Semantic memory involves the retention of specific facts and concepts.

语义记忆涉及特定事实和概念的保留。

**Two management approaches: 两种管理方式：**

| Approach | Profile (档案) | Collection (集合) |
|----------|---------------|-------------------|
| 结构 | 单个 JSON 文档，持续更新 | 多个文档，持续扩展 |
| 优点 | 简单、完整上下文 | 不易丢失信息、更高召回率 |
| 缺点 | 大时容易出错 | 需要删除/更新逻辑、搜索复杂 |
| 更新 | 传入旧档案，生成新档案 | LLM 生成新对象或更新现有 |

### Episodic memory / 情景记忆

Episodic memory involves recalling past events or actions. Often implemented through few-shot example prompting.

情景记忆涉及回忆过去的事件或操作。通常通过少样本示例提示实现。

### Procedural memory / 程序性记忆

Procedural memory involves remembering the rules used to perform tasks. Agents can modify their own prompts through "reflection" or meta-prompting.

程序性记忆涉及记住执行任务的规则。代理可以通过"反思"或元提示修改自己的提示词。

### Writing memories / 写入记忆

| 方式 | In the hot path (热路径) | In the background (后台) |
|------|------------------------|------------------------|
| 时机 | 代理运行时实时写入 | 独立后台任务 |
| 优点 | 实时更新、透明 | 不增加主应用延迟、更专注 |
| 缺点 | 增加复杂度和延迟 | 频率控制、触发时机 |

### Memory storage / 记忆存储

LangGraph stores long-term memories as JSON documents in a store. Each memory is organized under a custom namespace and a distinct key.

LangGraph 将长期记忆作为 JSON 文档存储在 store 中。每个记忆在自定义 namespace 和唯一 key 下组织。

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore(index={"embed": embed, "dims": 2})
store.put(("user_id", "chitchat"), "a-memory", {"rules": ["likes short language"]})
item = store.get(("user_id", "chitchat"), "a-memory")
items = store.search(("user_id", "chitchat"), query="language preferences")
```
