# Memory Overview — 总结

## 两种记忆对比

```
Short-term Memory (短期记忆)        Long-term Memory (长期记忆)
┌─────────────────────────┐       ┌─────────────────────────┐
│ 作用域: 单个线程          │       │ 作用域: 跨线程            │
│ 存储: 消息历史 + 状态     │       │ 存储: JSON 文档           │
│ 持久化: Checkpointer      │       │ 持久化: Store             │
│ 更新: 每次调用            │       │ 更新: 按需                │
│                         │       │                         │
│ Thread A:               │       │ Namespace (user_id):    │
│   msg1 → msg2 → msg3    │       │   key1: {facts}         │
│                         │       │   key2: {preferences}   │
│ Thread B: (独立)         │       │                         │
│   msg1 → msg2           │       │ Thread A 可以读取         │
│                         │       │ Thread B 也可以读取       │
└─────────────────────────┘       └─────────────────────────┘
```

## 三种长期记忆类型

| 类型 | 存储内容 | 代理用途 | 实现方式 |
|------|---------|---------|---------|
| **Semantic** | 事实 | 用户偏好、知识 | Profile 或 Collection |
| **Episodic** | 经验 | 过去操作 | Few-shot 示例 |
| **Procedural** | 规则 | 系统提示词 | 反思/元提示 |

## Semantic Memory 两种管理方式

| 维度 | Profile (档案) | Collection (集合) |
|------|---------------|-------------------|
| 结构 | 单个 JSON | 多个文档 |
| 更新 | 整体替换 | 增删改 |
| 优点 | 简单完整 | 不易丢失 |
| 缺点 | 大时出错 | 搜索复杂 |
| 召回率 | 中 | 高 |

## 记忆写入方式

| 维度 | Hot Path (热路径) | Background (后台) |
|------|------------------|------------------|
| 时机 | 代理运行时 | 独立任务 |
| 延迟 | 增加 | 不影响 |
| 透明度 | 用户可见 | 用户不可见 |
| 示例 | ChatGPT save_memories | 定时摘要 |

## 存储 API

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore(index={"embed": embed, "dims": 2})

# 写入
store.put(("user_id", "app"), "key", {"data": "value"})

# 读取
item = store.get(("user_id", "app"), "key")

# 搜索
items = store.search(("user_id", "app"), query="preferences", filter={"type": "semantic"})
```

## 与上下文工程的关系

Memory 是上下文工程的一部分：
- **短期记忆** → 管理对话历史长度
- **长期记忆** → 跨会话持久化知识
- **Skills** → 按需加载领域知识
- **Summarization** → 压缩冗余信息
