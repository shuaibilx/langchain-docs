# Context Overview — 总结

## 三维分类矩阵

```
                ┌──────────────┬──────────────┐
                │   Runtime    │   Cross-     │
                │   (单次运行)  │  Conversation │
                │              │   (跨对话)    │
┌───────────────┼──────────────┼──────────────┤
│ Static        │ 用户元数据    │     —        │
│ (不可变)       │ 工具         │              │
│               │ DB 连接       │              │
│               │ context=     │              │
├───────────────┼──────────────┼──────────────┤
│ Dynamic       │ 对话历史      │ 用户画像      │
│ (可变)         │ 中间结果      │ 偏好设置      │
│               │ 工具输出      │ 历史交互      │
│               │ state        │ store        │
└───────────────┴──────────────┴──────────────┘
```

## 三种上下文的访问方式

| 类型 | 访问方式 | 代码 |
|------|---------|------|
| Static runtime | `context` 参数 | `runtime.context.user_name` |
| Dynamic runtime | State 对象 | `request.state.get("key")` |
| Dynamic cross-conversation | Store | `store.get(namespace, key)` |

## 关键区分

### Runtime context ≠ LLM context

| 概念 | 说明 |
|------|------|
| **Runtime context** | 代码运行所需的数据和依赖（依赖注入） |
| **LLM context** | 传入 LLM 提示词的数据 |
| **Context window** | LLM 能处理的最大 token 数 |

Runtime context 可以用来**优化** LLM context（如用用户 ID 获取偏好，再注入提示词）。

## 与 Memory 的关系

```
Context Engineering (上下文工程)
    │
    ├── Static Runtime Context
    │   └── 用户元数据、工具、连接（不变）
    │
    ├── Dynamic Runtime Context = Short-term Memory
    │   └── 对话历史、状态（单次运行内变化）
    │
    └── Dynamic Cross-conversation Context = Long-term Memory
        └── 用户画像、偏好（跨对话持久化）
```

## 使用场景速查

| 需求 | 使用哪种 |
|------|---------|
| 根据用户姓名个性化 | Static runtime context |
| 记住当前对话中的信息 | Dynamic runtime context (state) |
| 跨对话记住用户偏好 | Dynamic cross-conversation context (store) |
| 传入数据库连接 | Static runtime context |
| 工具间传递中间结果 | Dynamic runtime context (state) |
| 学习用户行为模式 | Dynamic cross-conversation context (store) |
