# Context Overview — 逐段翻译

> 原文：https://docs.langchain.com/oss/concepts/context

---

**Context engineering** is the practice of building dynamic systems that provide the right information and tools, in the right format, so that an AI application can accomplish a task.

**上下文工程**是构建动态系统的实践，以正确的格式提供正确的信息和工具，使 AI 应用能够完成任务。

## Two dimensions / 两个维度

### By mutability / 按可变性

| 类型 | 说明 | 示例 |
|------|------|------|
| **Static context** | 执行期间不变的数据 | 用户元数据、数据库连接、工具 |
| **Dynamic context** | 运行时演变的数据 | 对话历史、中间结果、工具调用结果 |

### By lifetime / 按生命周期

| 类型 | 说明 | 示例 |
|------|------|------|
| **Runtime context** | 单次运行范围的数据 | 当前对话状态 |
| **Cross-conversation context** | 跨多次对话持久化的数据 | 用户偏好、长期记忆 |

> **Tip:** Runtime context refers to local context (data and dependencies your code needs). It does NOT refer to the LLM context (data passed into the prompt) or the context window (max tokens).
>
> 运行时上下文指本地上下文（代码运行所需的数据和依赖）。它不指 LLM 上下文（传入提示词的数据）或上下文窗口（最大 token 数）。

---

## Three context types / 三种上下文类型

| Context type | Description | Mutability | Lifetime | Access method |
|-------------|-------------|------------|----------|---------------|
| **Static runtime context** | 启动时传入的不可变数据 | 静态 | 单次运行 | `context` 参数 |
| **Dynamic runtime context (state)** | 运行时演变的可变数据 | 动态 | 单次运行 | LangGraph state |
| **Dynamic cross-conversation context (store)** | 跨对话的持久数据 | 动态 | 跨对话 | LangGraph store |

---

## Static runtime context / 静态运行时上下文

Immutable data like user metadata, tools, and database connections passed at the start of a run via the `context` argument.

通过 `context` 参数在运行开始时传入的不可变数据，如用户元数据、工具和数据库连接。

### In an agent prompt / 在代理提示词中

```python
from dataclasses import dataclass
from langchain.agents import create_agent
from langchain.agents.middleware import dynamic_prompt, ModelRequest

@dataclass
class ContextSchema:
    user_name: str

@dynamic_prompt
def personalized_prompt(request: ModelRequest) -> str:
    user_name = request.runtime.context.user_name
    return f"You are a helpful assistant. Address the user as {user_name}."

agent = create_agent(
    model="claude-sonnet-4-6",
    tools=[get_weather],
    middleware=[personalized_prompt],
    context_schema=ContextSchema,
)

agent.invoke(
    {"messages": [...]},
    context=ContextSchema(user_name="John Smith"),
)
```

### In a workflow node / 在工作流节点中

```python
from langgraph.runtime import Runtime

def node(state: State, runtime: Runtime[ContextSchema]):
    user_name = runtime.context.user_name
    ...
```

### In a tool / 在工具中

```python
from langchain.tools import tool, ToolRuntime

@tool
def get_user_email(runtime: ToolRuntime[ContextSchema]) -> str:
    """Retrieve user information."""
    email = get_user_email_from_db(runtime.context.user_name)
    return email
```

## Dynamic runtime context / 动态运行时上下文

Mutable data that can evolve during a single run, managed through the LangGraph state object. Includes conversation history, intermediate results, and values derived from tools or LLM outputs. The state object acts as short-term memory during a run.

在单次运行中演变的可变数据，通过 LangGraph state 对象管理。包括对话历史、中间结果和工具/LLM 输出的值。State 对象在运行期间充当短期记忆。

### In an agent / 在代理中

```python
from langchain.agents import create_agent, AgentState
from langchain.agents.middleware import dynamic_prompt, ModelRequest

class CustomState(AgentState):
    user_name: str

@dynamic_prompt
def personalized_prompt(request: ModelRequest) -> str:
    user_name = request.state.get("user_name", "User")
    return f"You are a helpful assistant. User's name is {user_name}"

agent = create_agent(
    model="claude-sonnet-4-6",
    tools=[...],
    state_schema=CustomState,
    middleware=[personalized_prompt],
)

agent.invoke({"messages": "hi!", "user_name": "John Smith"})
```

### In a workflow / 在工作流中

```python
class CustomState(TypedDict):
    messages: list[AnyMessage]
    extra_field: int

def node(state: CustomState):
    messages = state["messages"]
    return {"extra_field": state["extra_field"] + 1}
```

---

## Dynamic cross-conversation context / 动态跨对话上下文

Persistent, mutable data that spans across multiple conversations or sessions, managed through the LangGraph store. Includes user profiles, preferences, and historical interactions. The store acts as long-term memory.

跨多个对话或会话的持久可变数据，通过 LangGraph store 管理。包括用户画像、偏好和历史交互。Store 充当长期记忆。
