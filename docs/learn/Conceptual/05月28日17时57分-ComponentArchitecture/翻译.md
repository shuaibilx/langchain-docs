# Component Architecture — 逐段翻译

> 原文：https://docs.langchain.com/oss/langchain/component-architecture

---

LangChain's power comes from how its components work together to create sophisticated AI applications.

LangChain 的强大之处在于各组件如何协同工作创建复杂的 AI 应用。

## Core component ecosystem / 核心组件生态

```
📥 Input Processing (输入处理)
    Text → Document Loaders → Text Splitters → Documents

🔢 Embedding & Storage (嵌入与存储)
    Documents → Embedding Models → Vectors → Vector Stores

🔍 Retrieval (检索)
    User Query → Embedding → Query Vector → Retriever → Vector Store → Relevant Context

🤖 Generation (生成)
    Chat Models ↔ Tools ↔ Tool Results + Relevant Context → AI Response

🎯 Orchestration (编排)
    Agents → Chat Models + Tools + Retrievers + Memory
```

### Five layers / 五层架构

1. **Input processing** — Transform raw data into structured documents — 将原始数据转为结构化文档
2. **Embedding & storage** — Convert text into searchable vector representations — 将文本转为可搜索的向量表示
3. **Retrieval** — Find relevant information based on user queries — 根据用户查询找到相关信息
4. **Generation** — Use AI models to create responses, optionally with tools — 用 AI 模型创建响应
5. **Orchestration** — Coordinate everything through agents and memory — 通过代理和记忆协调一切

## Component categories / 组件分类

| Category | Purpose | Key Components | Use Cases |
|----------|---------|----------------|-----------|
| Models | AI 推理和生成 | Chat models, LLMs, Embeddings | 文本生成、推理、语义理解 |
| Tools | 外部能力 | APIs, databases | 网络搜索、数据访问、计算 |
| Agents | 编排和推理 | ReAct agents, tool calling agents | 非确定性工作流、决策 |
| Memory | 上下文保持 | Message history, custom state | 对话、有状态交互 |
| Retrievers | 信息访问 | Vector retrievers, web retrievers | RAG、知识库搜索 |
| Document processing | 数据摄取 | Loaders, splitters | PDF 处理、网页抓取 |
| Vector Stores | 语义搜索 | Chroma, Pinecone, FAISS | 相似度搜索、嵌入存储 |

## Common patterns / 常见模式

### RAG

```
User Question + Retriever → Relevant Docs → Chat Model → Informed Response
```

### Agent with tools

```
User Request → Agent → Need tool? → Yes → Call Tool → Tool Result → Agent (循环)
                                     → No → Final Answer
```

### Multi-agent system

```
Complex Task → Supervisor → Specialist 1 + Specialist 2 → Results → Supervisor → Coordinated Response
```
