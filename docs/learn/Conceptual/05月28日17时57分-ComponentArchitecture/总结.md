# Component Architecture — 总结

## 五层架构

```
Layer 5: Orchestration (编排)
    Agents → Models + Tools + Retrievers + Memory
        │
Layer 4: Generation (生成)
    Chat Models ↔ Tools → AI Response
        │
Layer 3: Retrieval (检索)
    Query → Embedding → Retriever → Vector Store → Context
        │
Layer 2: Embedding & Storage (嵌入与存储)
    Documents → Embedding Models → Vectors → Vector Stores
        │
Layer 1: Input Processing (输入处理)
    Text → Loaders → Splitters → Documents
```

## 组件速查表

| 组件 | 作用 | 示例 |
|------|------|------|
| Chat Models | LLM 推理 | GPT, Claude, Gemini |
| Tools | 外部能力 | 搜索、数据库、API |
| Agents | 编排循环 | ReAct, tool calling |
| Memory | 状态保持 | 消息历史、检查点 |
| Retrievers | 信息检索 | 向量检索、网页检索 |
| Document Loaders | 数据摄取 | PDF、网页、CSV |
| Text Splitters | 文本分割 | RecursiveCharacterTextSplitter |
| Vector Stores | 向量存储 | Chroma, Pinecone, FAISS |
| Embeddings | 文本向量化 | OpenAI, HuggingFace |

## 三种常见模式

| 模式 | 流程 | 适用 |
|------|------|------|
| RAG | 问题 → 检索 → LLM + 上下文 → 回答 | 知识问答 |
| Agent + Tools | 请求 → Agent ↔ 工具循环 → 回答 | 需要外部能力 |
| Multi-agent | 任务 → Supervisor → 专家代理 → 综合 | 复杂多领域 |
