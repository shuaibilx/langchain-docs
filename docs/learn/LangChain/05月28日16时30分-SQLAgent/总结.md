# SQL Agent — 流程详细总结

## 架构总览

```
用户问题: "Which genre has the longest tracks?"
    │
    ▼
┌─────────────────────────────────────────────┐
│            SQL Agent (create_agent)          │
│                                             │
│  系统提示词: 你是 SQL 数据库交互代理           │
│                                             │
│  Step 1: sql_db_list_tables()               │
│          → "Album, Artist, Customer, ..."    │
│                                             │
│  Step 2: sql_db_schema("Track, Genre")       │
│          → CREATE TABLE + 3 行示例数据        │
│                                             │
│  Step 3: sql_db_query_checker("SELECT ...")  │
│          → LLM 复查 SQL，无错误              │
│                                             │
│  Step 4: sql_db_query("SELECT ...")          │
│          → [('Sci Fi & Fantasy', 2911783)]   │
│                                             │
│  Step 5: 生成最终回答                        │
└─────────────────────────────────────────────┘
```

## 四个工具

| 工具 | 输入 | 输出 | 作用 |
|------|------|------|------|
| `sql_db_list_tables` | 空 | 逗号分隔的表名 | 发现可用表 |
| `sql_db_schema` | 表名列表 | CREATE TABLE + 示例数据 | 了解表结构 |
| `sql_db_query_checker` | SQL 查询 | 检查后的 SQL | 执行前复查 |
| `sql_db_query` | SQL 查询 | 查询结果 | 执行查询 |

### 工具调用顺序（典型流程）

```
1. sql_db_list_tables()          ← 总是先调用
2. sql_db_schema("相关表")       ← 获取表结构
3. sql_db_query_checker("SQL")   ← 检查 SQL
4. sql_db_query("SQL")           ← 执行查询
5. (如有错误) → 回到 Step 3 重写 SQL
```

### sql_db_query_checker 检查的常见错误

| 错误类型 | 示例 |
|----------|------|
| NOT IN with NULL | `WHERE id NOT IN (SELECT id FROM t)` 当 t 有 NULL |
| UNION vs UNION ALL | 不需要去重时用了 UNION |
| BETWEEN 范围 | BETWEEN 是闭区间，不是开区间 |
| 数据类型不匹配 | 字符串列用数字比较 |
| 标识符引用 | 列名含空格未加引号 |
| 函数参数数量 | 错误的参数个数 |
| CAST 类型 | 类型转换错误 |
| JOIN 列 | JOIN 条件用了错误的列 |

## Human-in-the-loop / 人在回路

### 为什么需要？

SQL 代理生成的查询可能：
- 查询大量数据导致性能问题
- 意外暴露敏感数据
- 生成不正确的查询

### 实现方式

```python
middleware=[
    HumanInTheLoopMiddleware(
        interrupt_on={"sql_db_query": True},  # 只拦截执行工具
        description_prefix="Tool execution pending approval",
    ),
]
checkpointer=InMemorySaver()  # 必须：支持暂停和恢复
```

### 流程

```
用户问题
    │
    ▼
Agent: sql_db_list_tables()     ← 自动执行
Agent: sql_db_schema(...)       ← 自动执行
Agent: sql_db_query_checker(...) ← 自动执行
Agent: sql_db_query(...)        ← ⏸️ 暂停，等待人工审批
    │
    ▼ 人工查看 SQL
    │
    ├── approve → 继续执行
    └── reject  → 停止或要求重写
    │
    ▼
Agent: 生成最终回答
```

### 恢复执行

```python
from langgraph.types import Command

# 批准
agent.stream(
    Command(resume={"decisions": [{"type": "approve"}]}),
    config,
)

# 拒绝
agent.stream(
    Command(resume={"decisions": [{"type": "reject", "reason": "Query too broad"}]}),
    config,
)
```

## 关键设计

### 系统提示词策略

```
1. 角色定义: "You are an agent designed to interact with a SQL database"
2. 行为约束: "DO NOT make any DML statements"
3. 工作流程: "ALWAYS look at the tables first"
4. 结果限制: "limit your query to at most {top_k} results"
5. 质量保证: "You MUST double check your query before executing"
```

### 错误自愈

代理具备自动纠错能力：
1. 执行查询 → 返回错误
2. 读取错误信息
3. 重写 SQL
4. 再次检查 → 再次执行
5. 直到成功或达到重试上限

### 安全考量

| 措施 | 说明 |
|------|------|
| 只读权限 | 系统提示词禁止 DML 语句 |
| 结果限制 | top_k 限制返回行数 |
| 人在回路 | 关键查询需人工审批 |
| 最小权限 | 数据库连接使用只读账号 |
