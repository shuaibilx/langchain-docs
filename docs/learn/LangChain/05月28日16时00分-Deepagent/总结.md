# Deep Agent from Scratch — 流程详细总结

## 核心思想

`create_deep_agent` = `create_agent` + middleware 的预组装版本。本文档教你**手动拆解并重组**这个过程。

## 架构对比

```
create_deep_agent (一体化)          create_agent + middleware (手动组装)
┌───────────────────────┐          ┌───────────────────────┐
│ create_deep_agent(    │          │ create_agent(         │
│   model,              │          │   model,              │
│   tools,              │          │   tools=[],           │
│   backend,      ──────│──等价于──▶│   middleware=[         │
│   subagents,          │          │     FilesystemMW,     │
│   skills,             │          │     SummarizationMW,  │
│   memory,             │          │     SkillsMW,         │
│ )                     │          │     TodoListMW,       │
└───────────────────────┘          │     SubAgentMW,       │
                                   │   ]                   │
                                   │ )                     │
                                   └───────────────────────┘
```

## 5 步渐进式构建

### Step 1: 最小代理（只有模型）

```python
agent = create_agent("anthropic:claude-sonnet-4-6", tools=[])
```
- 能运行，但什么工具都没有
- 没有文件系统、不能执行代码

### Step 2: 添加沙箱后端

```python
middleware=[FilesystemMiddleware(backend=backend)]
```

**FilesystemMiddleware 添加的工具：**
| 工具 | 功能 |
|------|------|
| `read_file` | 读取文件 |
| `write_file` | 写入文件 |
| `edit_file` | 编辑文件 |
| `glob` | 文件模式匹配 |
| `grep` | 内容搜索 |
| `execute` | 运行 shell 命令（仅当 backend 实现沙箱协议时） |

**LangSmithSandbox 的作用：**
- 创建隔离的云端沙箱环境
- 代理可以在其中安装包、写脚本、运行代码
- 不会影响宿主机

### Step 3: 添加上下文管理

```python
SummarizationMiddleware(model=model, backend=backend)
```
- 当上下文窗口接近满时自动压缩历史
- 用同一个模型生成摘要
- 代理可以持续工作而不触及 token 限制

### Step 4: 添加技能

```python
SkillsMiddleware(backend=backend, sources=["./skills/"])
```
- **渐进式披露**：技能仅在当前任务匹配时才加载
- 不是把所有知识都塞进系统提示词
- 节省 token，提高相关性

**SKILL.md 的 frontmatter 触发机制：**
```yaml
---
name: pandas-patterns
description: Common pandas and matplotlib patterns...  # ← 用于匹配任务
---
```

### Step 5: 添加子代理

```python
TodoListMiddleware()
SubAgentMiddleware(backend=backend, subagents=[visualizer])
```

**TodoListMiddleware 的作用：**
- 给代理一个待办列表工具
- 代理可以规划多步骤任务
- 跟踪进度

**SubAgentMiddleware 的作用：**
- 添加 `task` 工具
- 主代理通过 `task()` 委派子任务
- 子代理在独立上下文窗口运行

## Middleware 组合模式

### 最小可用
```python
middleware=[FilesystemMiddleware(backend=backend)]
```
适合：简单文件操作，不需要长对话

### 标准分析
```python
middleware=[
    FilesystemMiddleware(backend=backend),
    SummarizationMiddleware(model=model, backend=backend),
]
```
适合：中等复杂度的分析任务

### 完整配置
```python
middleware=[
    FilesystemMiddleware(backend=backend),
    SummarizationMiddleware(model=model, backend=backend),
    SkillsMiddleware(backend=backend, sources=["./skills/"]),
    TodoListMiddleware(),
    SubAgentMiddleware(backend=backend, subagents=[visualizer]),
]
```
适合：复杂多步骤任务，需要子代理协作

## 与 create_deep_agent 的 API 对照

| create_deep_agent 参数 | 对应 middleware |
|----------------------|----------------|
| `backend=backend` | `FilesystemMiddleware(backend=backend)` |
| `tools=[...]` | `tools=[...]`（直接传入） |
| `subagents=[...]` | `SubAgentMiddleware(subagents=[...])` |
| `skills=["./skills/"]` | `SkillsMiddleware(sources=["./skills/"])` |
| `memory=["./AGENTS.md"]` | 系统提示词中注入（手动处理） |
| `checkpointer=cp` | `SummarizationMiddleware`（类似效果） |

## 关键设计概念

### Middleware 模式
每个 middleware 是一个独立的功能层：
- 可以单独添加或移除
- 不影响其他 middleware
- 组合使用时按顺序执行

### 渐进式披露 (Progressive Disclosure)
技能不是一次性全部加载到系统提示词：
1. 只加载 frontmatter 的 `name` 和 `description`
2. 当任务匹配时，才加载完整的 `SKILL.md` 内容
3. 节省上下文窗口空间

### 子代理隔离
子代理在独立的上下文窗口中运行：
- 不共享主代理的对话历史
- 可以并行执行
- 减少主代理的上下文压力
