# Semantic Search Engine — 流程详细总结

## 架构总览

```
原始 PDF 文档
    │
    ▼ load_pdf_pages()
Document 对象列表 (page_content + metadata)
    │
    ▼ RecursiveCharacterTextSplitter
分割后的 Document 块 (chunk_size=1000, overlap=200)
    │
    ▼ embeddings.embed_query()
数值向量 (1536 维)
    │
    ▼ vector_store.add_documents()
VectorStore (向量索引)
    │
    ├──── similarity_search(query) ──→ 相关 Document 列表
    ├──── similarity_search_with_score(query) ──→ (Document, score) 列表
    └──── as_retriever() ──→ Retriever (Runnable 接口)
                                  │
                                  ▼ 集成到 RAG 链
                            LLM 生成回答
```

## 四大核心抽象

### 1. Document — 文档对象

```python
Document(
    page_content="文本内容",
    metadata={"source": "文件名", "page": 0, "start_index": 3125},
    id="可选ID"
)
```

| 属性 | 类型 | 用途 |
|------|------|------|
| `page_content` | `str` | 文本内容 |
| `metadata` | `dict` | 来源、页码、索引位置等 |
| `id` | `str` | 唯一标识符 |

### 2. Embeddings — 嵌入模型

将文本转换为数值向量。相似语义的文本在向量空间中距离相近。

| 提供商 | 模型 | 特点 |
|--------|------|------|
| OpenAI | `text-embedding-3-large` | 商业级，1536维 |
| Google | `gemini-embedding-001` | Gemini 生态 |
| HuggingFace | `all-mpnet-base-v2` | 本地免费 |
| Ollama | `llama3` | 本地免费 |
| Fake | `DeterministicFakeEmbedding` | 测试用 |

**核心 API：**
```python
# 查询嵌入
vector = embeddings.embed_query("text")
# 批量嵌入
vectors = embeddings.embed_documents(["text1", "text2"])
```

### 3. VectorStore — 向量存储

存储向量并支持高效相似度搜索。

**核心操作：**

| 方法 | 作用 | 返回 |
|------|------|------|
| `add_documents(docs)` | 索引文档 | IDs |
| `similarity_search(query)` | 相似度搜索 | `[Document]` |
| `similarity_search_with_score(query)` | 带分数搜索 | `[(Document, float)]` |
| `similarity_search_by_vector(vector)` | 按向量搜索 | `[Document]` |
| `as_retriever()` | 转为 Retriever | `VectorStoreRetriever` |

**主流 VectorStore 对比：**

| VectorStore | 类型 | 持久化 | 适用场景 |
|-------------|------|--------|---------|
| InMemoryVectorStore | 内存 | 否 | 原型/测试 |
| Chroma | 本地文件 | 是 | 本地开发 |
| PGVector | PostgreSQL | 是 | 已有 PG 基础设施 |
| Pinecone | 云服务 | 是 | 生产环境 |
| Milvus | 分布式 | 是 | 大规模数据 |
| Qdrant | 本地/云 | 是 | 高性能搜索 |

### 4. Retriever — 检索器

将 VectorStore 封装为 Runnable 接口，可以集成到 LangChain 链中。

**三种搜索类型：**

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `similarity` | 纯相似度搜索 | 默认，大多数场景 |
| `mmr` | 最大边际相关性 | 需要多样性时 |
| `similarity_score_threshold` | 带阈值过滤 | 只要高置信度结果 |

**两种创建方式：**

```python
# 方式 1: 自定义函数
@chain
def retriever(query: str) -> List[Document]:
    return vector_store.similarity_search(query, k=1)

# 方式 2: as_retriever
retriever = vector_store.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 1},
)
```

## 文本分割策略

```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,      # 每块最大字符数
    chunk_overlap=200,    # 块间重叠字符数
    add_start_index=True  # 保留原始位置信息
)
```

| 参数 | 作用 | 推荐值 |
|------|------|--------|
| `chunk_size` | 块大小 | 500-1500 |
| `chunk_overlap` | 重叠大小 | chunk_size 的 10-20% |
| `add_start_index` | 保留位置 | True（方便溯源） |

**为什么需要 overlap？**
- 防止句子在边界被切断
- 确保语义完整性
- 提高检索准确率

## 完整数据流（以 Nike 10-K 为例）

```
nke-10k-2023.pdf (107 页)
    │
    ▼ pypdf 提取文本
107 个 Document 对象
    │
    ▼ RecursiveCharacterTextSplitter(chunk_size=1000, overlap=200)
516 个 Document 块
    │
    ▼ InMemoryVectorStore.add_documents()
516 个向量索引到内存
    │
    ▼ similarity_search("How many distribution centers does Nike have?")
返回: "NIKE has eight significant distribution centers" (page 4)
```

## 搜索结果示例

| 查询 | 返回内容 | 分数 | 页码 |
|------|---------|------|------|
| "How many distribution centers?" | "NIKE has eight significant distribution centers" | — | 4 |
| "When was Nike incorporated?" | "NIKE, Inc. was incorporated in 1967" | — | 3 |
| "What was Nike's revenue in 2023?" | "$51.2 billion in fiscal 2023" | 0.237 | 35 |
| "How were margins impacted?" | "gross margin decreased 250 basis points to 43.5%" | — | 36 |

## 与 RAG 的关系

```
                    Semantic Search (本文档)
                           │
                           ▼
┌──────────────────────────────────────────┐
│                 RAG 链                    │
│                                          │
│  用户问题                                 │
│      │                                   │
│      ▼                                   │
│  Retriever ──→ 相关文档                   │
│      │                                   │
│      ▼                                   │
│  Prompt = 问题 + 相关文档                  │
│      │                                   │
│      ▼                                   │
│  LLM ──→ 基于文档的回答                   │
└──────────────────────────────────────────┘
```

Semantic Search 是 RAG 的**检索组件**。本文档构建的是检索层，RAG 教程会在此基础上添加 LLM 生成层。
