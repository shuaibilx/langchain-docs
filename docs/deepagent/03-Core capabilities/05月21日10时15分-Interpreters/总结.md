# Interpreters - 总结

## 核心概念

解释器给代理一个内存中的 JavaScript 运行时（QuickJS），用于编写代码来组合工具、编排 subagent 和转换数据。与沙箱不同，解释器在代理循环内运行，不暴露主机系统。

> 实验性功能，需要 `langchain-quickjs>=0.1.0` + Python `>=3.11`

## 解释器 vs 沙箱

| | 解释器 | 沙箱 |
|---|--------|------|
| 运行位置 | 代理循环内 | 远程隔离环境 |
| 语言 | JavaScript/TypeScript | 任意 |
| 访问 | 默认无外部访问 | 完整 OS 访问 |
| 用途 | 组合工具、数据转换 | 运行命令、安装依赖 |

## 适用场景

| 场景 | 使用 |
|------|------|
| 一两个简单调用 | 普通工具调用 |
| 循环/分支/重试/聚合 | 解释器 |
| 多个选定工具调用 | 解释器 + PTC |
| 跨线程可复用辅助 | 解释器 + Interpreter Skills |
| Shell/包/OS 访问 | 沙箱 |

## 基础用法

```python
from deepagents import create_deep_agent
from langchain_quickjs import CodeInterpreterMiddleware

agent = create_deep_agent(
    model="openai:gpt-5.4",
    middleware=[CodeInterpreterMiddleware()],
)
# 代理获得 eval 工具，可运行 TypeScript 代码
```

## eval 工具

- 在持久上下文中运行 TypeScript
- 捕获 `console.log`
- 返回最后一个表达式的结果
- 跨轮次持久化状态（快照）

## 编程式工具调用（PTC）

PTC 在解释器中暴露代理工具，允许代码调用工具。

### 启用

```python
CodeInterpreterMiddleware(ptc=["task", "web_search"])
```

### 使用

```typescript
// 代理编写的代码
const result = await tools.webSearch({ query: "deepagents" });
```

### 模式

| 模式 | 说明 |
|------|------|
| 批处理 | 循环多个输入调用工具 |
| 并行 | `Promise.all` 独立调用 |
| 条件逻辑 | 根据结果选择调用 |
| 早期终止 | 满足条件后停止 |
| 数据过滤 | 只返回相关结果 |
| 递归编排 | 重复调用 task + 组合结果 |

### 限制

- PTC 调用不经过正常工具路径
- `interrupt_on` 审批不强制执行

## 递归语言模型

```
模型 → 解释器变量空间
       ├── 大型输入/工作笔记
       ├── 选择下一个片段
       ├── 调用 subagent
       ├── 存储结果
       ├── 重复...
       └── 拼接结果
       │
       ▼
   紧凑综合 → 模型
```

变量空间（解释器内）与代理上下文（模型处理的）分离。

## Interpreter Skills

- 向解释器暴露代码模块
- `module: index.ts` 在 SKILL.md frontmatter 中
- 代理使用 `await import("@/skills/<name>")` 导入
- 适合：排序、分组、评分、解析、验证、聚合

## 快照和时间旅行

```
轮次 1：代理运行代码 → 快照解释器状态
轮次 2：恢复快照 → 代理继续使用之前的变量
```

- 默认 `snapshot_between_turns=True`
- 只保留可序列化的值
- 配合 LangGraph 时间旅行使用

## 安全边界

| 能力 | 默认可用 |
|------|---------|
| JavaScript 执行 | ✅ |
| console.log 捕获 | ✅ |
| 代理工具（PTC） | ❌ 需显式允许 |
| 文件系统访问 | ❌ 需 PTC |
| 网络访问 | ❌ 需 PTC |
| Shell/包/OS | ❌ 需沙箱 |

## Middleware 选项

| 参数 | 默认 | 说明 |
|------|------|------|
| `memory_limit` | 64 MB | 堆内存限制 |
| `timeout` | 5.0s | 每次 eval 超时 |
| `max_ptc_calls` | 256 | 最大 PTC 调用次数 |
| `ptc` | None | PTC 允许列表 |
| `snapshot_between_turns` | True | 跨轮次快照 |
| `capture_console` | True | 捕获 console 输出 |
| `skills_backend` | None | Skill 模块 backend |

## 关键注意事项

1. **QuickJS 运行时** — 轻量级 JavaScript，非 Node.js
2. **PTC 允许列表** — 权限边界，只暴露需要的工具
3. **快照只保留数据** — 函数/类不可序列化
4. **interrupt_on 不强制** — PTC 调用跳过审批
5. **沙箱做重活** — 解释器做轻量组合
