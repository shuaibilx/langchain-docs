# Profiles - 总结

## 核心概念

Profiles 是 DeepAgent 的**按模型配置系统**——为不同提供商/模型打包不同的 harness 行为和模型构建参数，无需更改 `create_deep_agent` 调用点。

## 两种 Profile 类型

| 类型 | 用途 | 影响 |
|------|------|------|
| **HarnessProfile** | 提示组装、工具可见性、middleware、subagent | 影响 harness 行为 |
| **ProviderProfile** | 模型构建 kwargs、凭证检查 | 仅影响模型构建 |

## HarnessProfile 字段

| 字段 | 说明 |
|------|------|
| `base_system_prompt` | 替换基础系统提示 |
| `system_prompt_suffix` | 追加到提示末尾 |
| `tool_description_overrides` | 覆盖工具描述 |
| `excluded_tools` | 移除特定工具 |
| `excluded_middleware` | 移除特定 middleware |
| `extra_middleware` | 追加 middleware |
| `general_purpose_subagent` | 配置 GP subagent |

## 注册键格式

```
"openai"           → 提供商级，适用于所有 OpenAI 模型
"openai:gpt-5.4"   → 模型级，仅适用于 gpt-5.4
```

**合并规则**：模型级未设置的字段从提供商级继承

## 合并语义

| 字段 | 行为 |
|------|------|
| 提示文本 | 新值设置时获胜，否则继承 |
| 工具描述覆盖 | 按键合并 |
| 排除的工具/middleware | 集合并 |
| 额外 middleware | 按类合并 |
| GP subagent | 按字段合并 |

## ProviderProfile 字段

| 字段 | 说明 |
|------|------|
| `init_kwargs` | 静态参数（如 temperature） |
| `pre_init` | 构建前副作用（如凭证检查） |
| `init_kwargs_factory` | 运行时派生 kwargs |

## 配置文件加载

```yaml
# openai.yaml
base_system_prompt: You are helpful.
system_prompt_suffix: Respond briefly.
excluded_tools: [execute, grep]
excluded_middleware: [SummarizationMiddleware]
general_purpose_subagent:
  enabled: false
```

```python
register_harness_profile("openai", HarnessProfileConfig.from_dict(yaml.safe_load(f)))
```

## 插件发布

```toml
[project.entry-points."deepagents.harness_profiles"]
my_provider = "my_pkg.profiles:register_harness"
```

加载顺序：内置 → 插件 → 用户代码

## 关键注意事项

1. **无通配符** — 需要为每个提供商单独注册
2. **重新注册 = 合并** — 不替换，叠加
3. **excluded_middleware 限制** — FilesystemMiddleware、SubAgentMiddleware 不能排除
4. **GP subagent 禁用** — `enabled=False` + 无同步 subagent
5. **HarnessProfile vs HarnessProfileConfig** — 前者支持运行时，后者支持序列化
