# 模型 - 总结

## 核心概念

Deep Agents 兼容任何支持工具调用的 LangChain 聊天模型。模型以 `provider:model` 格式指定。

## 模型字符串格式

```
provider:model
│          │
│          └── 传递给提供商的模型标识符
└── LangChain 集成前缀
```

示例：
- `anthropic:claude-sonnet-4-6`
- `openai:gpt-5.4`
- `google_genai:gemini-3.1-pro-preview`
- `baseten:zai-org/GLM-5.1`（命名空间 ID）
- `fireworks:accounts/fireworks/models/glm-5p1`（部署路径）

## 两种模型配置方式

| 方式 | 适用场景 | 示例 |
|------|---------|------|
| 模型字符串 | 快速原型，默认参数 | `model="anthropic:claude-sonnet-4-6"` |
| 模型实例 | 需要自定义参数 | `model=init_chat_model(..., thinking_level="medium")` |

## Provider Profiles（两级注册）

```
Provider 级 ──→ Model 级（合并覆盖）
"openai"          "openai:gpt-5.4"
temperature=0     reasoning_effort="medium"
                  （继承 temperature=0）
```

- Provider 级：适用于该提供商所有模型
- Model 级：仅适用于特定模型，合并到 Provider 级之上
- **不适用于**预配置的模型实例（通过 init_chat_model 传递的）

## 运行时动态切换模型

通过 `@wrap_model_call` middleware + runtime context 实现：

```
用户选择 "openai:gpt-5.4"
    ↓
context=Context(model="openai:gpt-5.4")
    ↓
wrap_model_call middleware 拦截
    ↓
init_chat_model("openai:gpt-5.4") 创建新模型
    ↓
request.override(model=new_model)
    ↓
handler(request) 使用新模型
```

## Harness Profile vs Provider Profile

| 维度 | Harness Profile | Provider Profile |
|------|----------------|-----------------|
| 作用对象 | 代理行为（工具、middleware、系统提示） | 模型初始化参数（temperature 等） |
| 何时应用 | 模型构建后 | 模型构建时 |
| 适用条件 | 始终应用 | 仅 `provider:model` 字符串 |

| 维度 | 最佳模型 | 分数 |
|------|---------|------|
| 总体 | openrouter:z-ai/glm-5.1 | 89% |

| 摘要 | 多个模型并列 | 100% |

## 关键注意事项

1. **`provider:model` 是推荐格式** — 提供商前缀决定使用哪个 LangChain 集成
2. **Provider Profile 只对字符串生效** — 传模型实例时不应用
3. **两级 Profile 可共存** — Model 级合并到 Provider 级之上
4. **运行时切换用 middleware** — 不要重建代理，用 `@wrap_model_call`
5. **评估通过 ≠ 实际任务表现** — 评估套件测试基本操作，复杂任务需要更强模型
