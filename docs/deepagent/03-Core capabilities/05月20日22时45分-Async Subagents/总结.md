# Async Subagents - 总结

## 核心概念

异步 subagent 允许主管代理启动立即返回的后台任务，主管继续与用户交互，subagent 并发工作。主管可随时检查进度、发送后续指令或取消任务。

> 预览功能，需要 `deepagents>=0.5.0`

## 同步 vs 异步对比

| 维度 | 同步 | 异步 |
|------|------|------|
| 执行模型 | 阻塞 | 非阻塞（立即返回任务 ID） |
| 并发 | 并行但阻塞 | 并行且非阻塞 |
| 任务中更新 | 不可能 | `update_async_task` |
| 取消 | 不可能 | `cancel_async_task` |
| 有状态性 | 无状态 | 有状态（跨交互维护状态） |
| 适用场景 | 快速任务 | 长时间运行、可并行化、需要中途引导 |

## 架构图

```
用户 ←→ 主管代理
            │
            ├── start_async_task → Researcher（独立线程）
            ├── start_async_task → Coder（独立线程）
            │
            ├── check_async_task → 获取状态/结果
            ├── update_async_task → 发送后续指令
            ├── cancel_async_task → 停止任务
            └── list_async_tasks → 列出所有任务
```

## AsyncSubAgent 配置

```python
AsyncSubAgent(
    name="researcher",           # 必需：唯一标识符
    description="...",           # 必需：描述
    graph_id="researcher",       # 必需：图 ID
    url="https://...",           # 可选：无则 ASGI，有则 HTTP
    headers={"Authorization": "..."},  # 可选：自定义认证头
)
```

## 五个异步工具

| 工具 | 用途 | 返回 |
|------|------|------|
| `start_async_task` | 启动后台任务 | 任务 ID（立即） |
| `check_async_task` | 获取状态和结果 | 状态 + 结果 |
| `update_async_task` | 发送新指令 | 确认 + 更新状态 |
| `cancel_async_task` | 停止任务 | 确认 |
| `list_async_tasks` | 列出所有任务 | 任务摘要 |

## 生命周期

```
启动 → 返回 task_id → 主管继续
                         │
用户询问进度 → check(task_id) → 返回状态
                         │
任务完成 → check(task_id) → 返回结果
```

## 状态管理

- 任务元数据存储在专用状态通道 `async_tasks`
- 与消息历史分离（压缩时不会丢失任务 ID）
- 每个任务记录：task_id、agent_name、thread_id、run_id、status、timestamps

## 传输方式

| 传输 | 配置 | 特点 |
|------|------|------|
| **ASGI** | 省略 `url` | 进程内，零延迟，推荐默认 |
| **HTTP** | 设置 `url` | 网络调用，支持远程部署 |

## 部署拓扑

| 拓扑 | 说明 | 适用场景 |
|------|------|---------|
| **单一部署** | 所有代理共同部署（ASGI） | 推荐起点 |
| **拆分部署** | 主管 + subagent 分服务器（HTTP） | 不同计算配置 |
| **混合部署** | 部分 ASGI + 部分 HTTP | 灵活组合 |


## 最佳实践

| 实践 | 说明 |
|------|------|
| 调整工作池 | `langgraph dev --n-jobs-per-worker 10`（主管 + subagent 数量） |
| 清晰描述 | 主管据此选择 subagent |
| 线程 ID 追踪 | LangSmith 中关联主管和 subagent 追踪 |
| 启动后不轮询 | 返回控制给用户 |
| 使用完整 task_id | 不截断 |

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| 启动后立即轮询 | 系统提示强化"启动后返回控制" |
| 报告过期状态 | 总是先调用 `check` 再报告 |
| task_id 查找失败 | 使用完整 ID，不截断 |
| 启动排队 | 增加 `--n-jobs-per-worker` |
