# Memory - 总结

## 核心概念

Memory 是 DeepAgent 的持久记忆系统。代理以文件形式读写记忆，通过 Backend 控制存储位置。记忆是文件系统支持的：代理读写记忆如同读写文件。

## 短期 vs 长期记忆

| 类型 | 范围 | 机制 | 说明 |
|------|------|------|------|
| **短期记忆** | 单次对话 | 检查点（自动） | 对话历史、草稿文件 |
| **长期记忆** | 跨对话 | Store Backend（手动配置） | 用户偏好、代理知识、技能 |

## 记忆工作流程

```
1. 配置：memory=["/memories/AGENTS.md"] + backend 路由
2. 读取：启动时加载到系统提示 / 对话中按需读取
3. 更新（可选）：对话中 edit_file / 后台整合
```

## 三种范围化模式

| 模式 | Namespace | 用例 |
|------|-----------|------|
| **代理范围** | `(assistant_id,)` | 所有用户共享，代理自我改进 |
| **用户范围** | `(user_id,)` | 每个用户隔离，个人偏好 |
| **组织范围** | `(org_id,)` | 组织策略，通常只读 |

### 代理范围记忆

```python
backend=CompositeBackend(
    default=StateBackend(),
    routes={
        "/memories/": StoreBackend(
            namespace=lambda rt: (rt.server_info.assistant_id,),
        ),
    },
)
```

### 用户范围记忆

```python
backend=CompositeBackend(
    default=StateBackend(),
    routes={
        "/memories/": StoreBackend(
            namespace=lambda rt: (rt.server_info.user.identity,),
        ),
    },
)
```

### 组织级记忆

```python
routes={
    "/policies/": StoreBackend(
        namespace=lambda rt: (rt.context.org_id,),
    ),
}
```

## 六个记忆维度

| 维度 | 选项 |
|------|------|
| 持续时间 | 短期（单对话）/ 长期（跨对话） |
| 信息类型 | 情景性（经历）/ 程序性（技能）/ 语义性（事实） |
| 范围 | 用户 / 代理 / 组织 |
| 更新策略 | 对话中 / 后台整合 |
| 检索 | 加载到提示 / 按需 |
| 权限 | 读写 / 只读 |

## 情景性记忆

- 存储过去经历的完整上下文
- 使用 checkpointer 持久化对话
- 通过 `search_past_conversations` 工具搜索历史

```python
@tool
async def search_past_conversations(query: str, runtime: ToolRuntime) -> str:
    threads = await client.threads.search(metadata={"user_id": user_id}, limit=5)
    ...
```

## 后台整合

| 方式 | 优点 | 缺点 |
|------|------|------|
| **热路径**（对话中） | 立即可用 | 增加延迟 |
| **后台**（对话间） | 无延迟，跨对话综合 | 下次对话才可用 |

### 整合代理

```python
# consolidation_agent.py
agent = create_deep_agent(
    model="google_genai:gemini-3.5-flash",
    system_prompt="Review recent conversations and update memory.",
    tools=[search_recent_conversations],
)
```

### Cron 计划

```python
await client.crons.create(
    assistant_id="consolidation_agent",
    schedule="0 */6 * * *",  # 每 6 小时
    input={"messages": [{"role": "user", "content": "Consolidate recent memories."}]},
)
```

> cron 间隔必须与回溯窗口匹配

## 只读 vs 可写记忆

| 权限 | 用例 | 实现 |
|------|------|------|
| **读写** | 用户偏好、代理自我改进 | 代理通过 edit_file 更新 |
| **只读** | 组织策略、合规规则 | permissions 拒绝写入 / policy hooks |

## 安全考虑

- 默认用户范围 `(user_id)`，除非有共享理由
- 共享策略用只读记忆（应用代码填充）
- 敏感路径写入前添加 HITL 验证

## 并发写入

- 同一文件并发写入 → 后写入获胜冲突
- 用户范围很少见（一次一个对话）
- 代理/组织范围 → 后台整合或按主题分文件

## 多代理部署

```python
StoreBackend(
    namespace=lambda rt: (
        rt.server_info.assistant_id,  # 按代理隔离
        rt.server_info.user.identity, # 按用户隔离
    ),
)
```

## 关键注意事项

1. **记忆 = 文件** — 代理读写记忆如同读写文件
2. **Backend 控制存储** — CompositeBackend + 路由
3. **namespace 决定范围** — assistant_id / user_id / org_id
4. **按需加载** — Skills 只在需要时加载完整内容
5. **cron 必须同步** — 间隔 = 回溯窗口
