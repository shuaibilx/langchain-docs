# Backends - 总结

## 核心概念

Backend 是 DeepAgent 文件系统工具的可插拔存储层。所有 `ls/read/write/edit/glob/grep` 操作都通过 backend 执行。

## 7 种内置 Backend

| Backend | 范围 | 持久性 | 适用场景 |
|---------|------|--------|---------|
| **StateBackend**（默认） | 线程 | 检查点内 | 草稿板、中间结果 |
| **FilesystemBackend** | 主机磁盘 | 永久 | 本地开发、CI/CD |
| **LocalShellBackend** | 主机磁盘+shell | 永久 | 本地开发 CLI |
| **StoreBackend** | 跨线程 | LangGraph Store | 长期记忆、跨对话 |
| **ContextHubBackend** | 跨线程 | LangSmith Hub | Hub 原生持久化 |
| **Sandbox** | 隔离容器 | 沙箱生命周期 | 生产代码执行 |
| **CompositeBackend** | 混合路由 | 取决于路由 | 灵活组合 |

## 架构图

```
文件系统工具 (ls/read/write/edit/glob/grep)
         │
         ▼
     Backend 层
    ┌────┼────┬────────┬──────────┬────────┬──────────┐
    ▼    ▼    ▼        ▼          ▼        ▼          ▼
  State Disk  Store  ContextHub Sandbox LocalShell Composite
                                              │         │
                                         +execute    Router
                                                    ┌──┼──┐
                                                State Disk Store
```

## CompositeBackend 路由模式

```python
CompositeBackend(
    default=StateBackend(),           # 默认：线程范围
    routes={
        "/memories/": StoreBackend(   # /memories/ → 跨线程持久化
            namespace=lambda rt: (rt.server_info.user.identity,)
        ),
        "/workspace/": FilesystemBackend(  # /workspace/ → 真实磁盘
            root_dir="/path/to/project", virtual_mode=True
        ),
    },
)
```

路由规则：更长前缀优先。`ls/glob/grep` 聚合结果并保留原始路径。

## Namespace Factory

控制 StoreBackend 数据隔离位置：

```python
namespace=lambda rt: (rt.server_info.user.identity,)     # 按用户
namespace=lambda rt: (rt.server_info.assistant_id,)       # 按助手
namespace=lambda rt: (rt.execution_info.thread_id,)       # 按线程
namespace=lambda rt: (rt.server_info.user.identity, "fs") # 组合
```

Runtime 提供：`rt.context`（用户上下文）、`rt.server_info`（服务器元数据）、`rt.execution_info`（执行信息）

## 安全等级

```
最安全 ←──────────────────────────────────────→ 最危险

StateBackend → StoreBackend → FilesystemBackend → LocalShellBackend
  线程范围       跨线程           磁盘读写           磁盘+shell执行
  无execute      无execute        virtual_mode       任意命令
```

## 权限系统

```python
permissions=[
    FilesystemPermission(
        operations=["write"],
        paths=["/policies/**"],
        mode="deny",
    ),
]
```

- 在调用 backend 之前评估
- 首次匹配获胜

## 策略 Hooks（自定义验证）

两种方式：

1. **子类化**：继承 FilesystemBackend，覆盖 write/edit
2. **通用包装器**：实现 BackendProtocol，包装任意 backend

用途：速率限制、审计日志、内容检查

## 工厂模式迁移（0.5.0 弃用）

| 之前 | 之后 |
|------|------|
| `backend=lambda rt: StateBackend(rt)` | `backend=StateBackend()` |
| `ctx.runtime.context.user_id` | `rt.server_info.user.identity` |

## BackendProtocol 必需方法

| 方法 | 功能 |
|------|------|
| `ls(path)` | 列出目录，返回 FileInfo 条目 |
| `read(file_path, offset, limit)` | 读取文件内容 |
| `grep(pattern, path, glob)` | 搜索文件内容 |
| `glob(pattern, path)` | 通配符搜索 |
| `write(file_path, content)` | 创建文件（仅创建） |
| `edit(file_path, old, new, replace_all)` | 精确字符串替换 |

## 关键注意事项

1. **默认 StateBackend** — 线程范围，检查点持久化
2. **FilesystemBackend 必须 virtual_mode=True** — 否则无安全保护
3. **LocalShellBackend 仅限受信环境** — 代理可执行任意命令
4. **CompositeBackend 是推荐模式** — 内部数据用 StateBackend，项目文件用 FilesystemBackend
5. **namespace 在 v0.5.0 必需** — 新代码始终显式设置
6. **factory 模式已弃用** — 直接传递实例
