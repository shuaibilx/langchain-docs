# Event Streaming - 总结

## 核心概念

Event Streaming 是 DeepAgent 的**实时输出系统**——流式传输 subagent、消息、工具调用和最终输出。Deep Agents 在 LangGraph 流式传输之上添加了 `stream.subagents` 投影。

## 流式传输 API

```python
stream = agent.stream_events(input, version="v3")
```

## 五种投影

| 投影 | 说明 |
|------|------|
| `stream.messages` | 顶层消息 |
| `stream.tool_calls` | 协调器工具调用 |
| `stream.subagents` | Subagent 流 |
| `stream.values` | 最终值 |
| `stream.interleave(...)` | 交错多个投影 |

## Subagent 流字段

| 字段 | 说明 |
|------|------|
| `name` | Subagent 名称 |
| `messages` | Subagent 消息 |
| `tool_calls` | Subagent 工具调用 |
| `subagents` | 嵌套 subagent |
| `output` | 最终状态/完成信号 |
| `path` | 命名空间路径 |
| `status` | started/completed/failed/interrupted |

## 使用模式

### 1. 追踪生命周期

```python
for subagent in stream.subagents:
    print(f"{subagent.name}: {subagent.status}")
    _ = subagent.output  # 等待完成
```

### 2. 流式传输消息

```python
# 协调器消息
for message in stream.messages:
    print(message.text)

# Subagent 消息
for subagent in stream.subagents:
    for message in subagent.messages:
        print(message.text)
```

### 3. 流式传输工具调用

```python
for call in stream.tool_calls:
    print(call.tool_name, call.input)
    for delta in call.output_deltas:
        print(delta, end="")
```

### 4. 递归嵌套

```python
for subagent in stream.subagents:
    for nested in subagent.subagents:
        print(nested.name, nested.status)
```

## 并发消费

| 方式 | 代码 |
|------|------|
| **异步** | `asyncio.gather(consume_coordinator(), consume_subagents())` |
| **同步** | `stream.interleave("messages", "subagents")` |
| **原始事件** | 迭代 `stream`，用 `namespace` 标识来源 |

## Subagent vs Subgraph

| 投影 | 内容 | 适用场景 |
|------|------|---------|
| `stream.subgraphs` | 图执行结构 | 调试 |
| `stream.subagents` | 产品级任务委派 | 面向用户的 UI |

## 关键注意事项

1. **轻量级投影** — 只在访问时打开流
2. **version="v3"** — 必须指定
3. **snake_case** — Python 使用 `tool_calls` 不是 `toolCalls`
4. **namespace 标识来源** — 空 = 协调器，非空 = subagent
5. **interleave** — 同步代码的并发消费方案
