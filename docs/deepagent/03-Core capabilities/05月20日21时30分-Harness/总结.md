# Harness 能力 - 总结

## 核心概念

Harness 是 DeepAgent 的"骨架"，由 8 大能力 + 2 个辅助系统组成，使构建长时间运行的代理更容易。

## 8 大能力

| 能力 | 核心机制 | 作用 |
|------|---------|------|
| **Planning** | `write_todos` 工具 | 维护结构化任务列表，跟踪 pending/in_progress/completed 状态 |
| **Virtual Filesystem** | `ls/read/write/edit/glob/grep/execute` | 可插拔 backend 支持的文件系统操作 |
| **Permissions** | 声明式规则（首次匹配获胜） | 控制代理可读写哪些文件/目录 |
| **Subagents** | `task` 工具 | 上下文隔离、并行执行、专业化子任务 |
| **Context Management** | 压缩 + 卸载 + 隔离 | 突破上下文窗口限制，自动摘要 |
| **Code Execution** | `execute`（沙箱）/ `eval`（解释器） | 运行 shell 命令或 JavaScript |
| **HITL** | `interrupt_on` 参数 | 指定工具调用前暂停，需人工批准 |
| **Profiles** | `HarnessProfile` | 每个提供商/模型的声明式配置包 |

## 2 个辅助系统

| 系统 | 加载方式 | 内容 | 用例 |
|------|---------|------|------|
| **Skills** | 渐进式披露（按需加载） | SKILL.md + 脚本 + 模板 | 领域工作流、专业知识 |
| **Memory** | 始终加载 | AGENTS.md 文件 | 编码风格、偏好、约定 |

## 文件系统工具

| 工具 | 功能 |
|------|------|
| `ls` | 列出目录及元数据 |
| `read_file` | 读取文件，支持 offset/limit/多模态 |
| `write_file` | 创建新文件 |
| `edit_file` | 精确字符串替换 |
| `glob` | 通配符搜索 |
| `grep` | 内容搜索（多种输出模式） |
| `execute` | 运行 shell 命令（仅沙箱） |

## 权限规则

- 首次匹配获胜，无匹配则默认允许
- 规则包含：`operations`、`paths`（glob）、`mode`（allow/deny）
- 不适用于沙箱 backend（沙箱通过 execute 可执行任意命令）

## 子代理特性

- 上下文隔离：繁重工作不污染主代理上下文
- 并行执行：多个子代理可同时运行
- 无状态：只返回单次结果，不能多轮对话
- 默认提供 `general-purpose` 子代理

## 代码执行两种方式

| 方式 | 环境 | 适用场景 |
|------|------|---------|
| Sandbox `execute` | 隔离容器 | 安装依赖、运行测试、调 CLI |
| Interpreter `eval` | QuickJS 运行时 | 循环、批处理、数据转换 |

## 配置禁用方式

- 隐藏文件系统工具 → `excluded_tools`（推荐）
- 禁用子代理 → harness profile + 不传同步 subagents
- 不要用 `excluded_middleware` 移除核心 middleware（会被拒绝）

## 关键注意事项

1. Harness 是组合式设计，8 个能力可独立配置
2. Permissions 默认允许，需显式拒绝敏感路径
3. Skills 渐进加载节省 Token，Memory 始终加载保证一致性
4. 子代理无状态，只能单次返回结果
5. `edit_file` 是精确替换，不是全量重写
