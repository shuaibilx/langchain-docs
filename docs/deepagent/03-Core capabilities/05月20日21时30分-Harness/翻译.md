# Harness 能力

代理 harness 是几种不同能力的组合，使构建长时间运行的代理更容易：

* [规划能力](#规划能力)
* [虚拟文件系统](#虚拟文件系统访问)
* [文件系统权限](#文件系统权限)
* [任务委派（子代理）](#任务委派子代理)
* [上下文和 token 管理](#上下文管理)
* [代码执行](#代码执行)
* [人机交互](#人机交互)
* [Harness profiles](#harness-profiles)

除了这些能力外，Deep Agents 还使用 [Skills](#skills) 和 [Memory](#memory) 来提供额外的上下文和指令。

## 规划能力

harness 提供了一个 `write_todos` 工具，代理可以用它来维护结构化的任务列表。

**特性：**

* 跟踪多个任务及其状态（`'pending'`、`'in_progress'`、`'completed'`）
* 持久化在代理状态中
* 帮助代理组织复杂的多步骤工作
* 适用于长时间运行的任务和规划

## 虚拟文件系统访问

harness 提供了一个可配置的虚拟文件系统，可以由不同的可插拔 backend 支持。
backend 支持以下文件系统操作：

| 工具 | 描述 |
|------|------|
| `ls` | 列出目录中的文件及元数据（大小、修改时间） |
| `read_file` | 读取文件内容及行号，支持大文件的 offset/limit。还支持为非文本文件（图像、视频、音频和文档）返回多模态内容块。 |
| `write_file` | 创建新文件 |
| `edit_file` | 在文件中执行精确字符串替换（支持全局替换模式） |
| `glob` | 查找匹配模式的文件（例如 `**/*.py`） |
| `grep` | 搜索文件内容，支持多种输出模式（仅文件、带上下文的内容或计数） |
| `execute` | 在环境中运行 shell 命令（仅限[沙箱 backend](/oss/python/deepagents/sandboxes)） |

**支持的多模态文件扩展名：**

| 类型 | 扩展名 |
|------|--------|
| 图像 | `.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`、`.heic`、`.heif` |
| 视频 | `.mp4`、`.mpeg`、`.mov`、`.avi`、`.flv`、`.mpg`、`.webm`、`.wmv`、`.3gpp` |
| 音频 | `.wav`、`.mp3`、`.aiff`、`.aac`、`.ogg`、`.flac` |
| 文件 | `.pdf`、`.ppt`、`.pptx` |

**不使用默认文件系统工具运行：**

要对模型隐藏上述文件系统工具，使用 `excluded_tools` 注册一个 [harness profile](#harness-profiles)：

```python
from deepagents import HarnessProfile, register_harness_profile

register_harness_profile(
    "anthropic:claude-sonnet-4-6",
    HarnessProfile(
        excluded_tools=frozenset(
            {"ls", "read_file", "write_file", "edit_file", "glob", "grep"}
        ),
    ),
)
```

移除 `FilesystemMiddleware` 本身（通过 `excluded_middleware`）会被有意拒绝——使用 `excluded_tools` 仅隐藏模型可见的工具表面，同时保留 middleware。要移除 `task` 工具，请参阅[不使用子代理运行](#不使用子代理运行)。

虚拟文件系统被其他几个 harness 能力使用，如 skills、memory、代码执行和上下文管理。你也可以在为 Deep Agents 构建自定义工具和 middleware 时使用文件系统。

更多信息请参阅 [backends](/oss/python/deepagents/backends)。

## 文件系统权限

harness 支持声明式权限规则，控制代理可以读取或写入哪些文件和目录。权限适用于上面列出的内置文件系统工具，按声明顺序评估，采用首次匹配语义。

**工作原理：**

* 创建代理时传递规则列表到 `permissions=`
* 每条规则指定 `operations`（`"read"`、`"write"`）、`paths`（glob 模式）和 `mode`（`"allow"` 或 `"deny"`）
* 第一条匹配的规则获胜。如果没有规则匹配，则允许操作。

**用途：**

* 将代理限制在特定目录（例如 `/workspace/`）
* 保护敏感文件（例如 `.env`、凭据）
* 给子代理比父代理更窄的访问权限

权限不适用于[沙箱 backend](/oss/python/deepagents/sandboxes)，沙箱支持通过 `execute` 工具执行任意命令。对于自定义验证逻辑，请使用 [backend policy hooks](/oss/python/deepagents/backends#add-policy-hooks)。

完整的规则结构、示例和子代理继承，请参阅 [Permissions](/oss/python/deepagents/permissions)。

## 任务委派（子代理）

harness 允许主代理创建临时的"子代理"用于隔离的多步骤任务。

**用途：**

* **上下文隔离** — 子代理的工作不会混乱主代理的上下文
* **并行执行** — 多个子代理可以同时运行
* **专业化** — 子代理可以有不同的工具/配置
* **Token 效率** — 大型子任务上下文被压缩为单个结果

**工作原理：**

* 主代理有一个 `task` 工具
* 调用时，它创建一个带有自己上下文的新代理实例
* 子代理自主执行直到完成
* 返回单个最终报告给主代理
* 可以使用[默认的 `general-purpose` 子代理](/oss/python/deepagents/subagents#default-subagent)（默认启用）或添加[自定义子代理](/oss/python/deepagents/subagents#custom-subagents)
* 子代理是无状态的（不能发送多条消息回来）

**不使用子代理运行（没有 `task` 工具）：**

要运行没有 `task` 工具的代理，请参阅[不使用子代理运行](/oss/python/deepagents/subagents#running-without-subagents)。不要尝试通过 `excluded_middleware` 移除 `SubAgentMiddleware`——这会被有意拒绝。相反，通过 [harness profile](#harness-profiles) 禁用自动添加的子代理，并且不通过 `subagents=` 传递同步子代理。异步子代理不受影响。

## 上下文管理

harness 管理上下文，使 deep agents 能够在 token 限制内处理长时间运行的任务，同时保留所需的信息。

**工作原理：**

* **输入上下文** — 系统提示、memory、skills 和工具提示塑造代理启动时的知识
* **压缩** — 内置的卸载和摘要功能在任务进行中保持上下文在窗口限制内
* **隔离** — 子代理隔离繁重工作并仅返回结果（参见[任务委派](#任务委派子代理)）
* **长期记忆** — 通过虚拟文件系统跨线程持久化存储

**用途：**

* 使超出单个上下文窗口的多步骤任务成为可能
* 无需手动修剪即可保持最相关的信息在范围内
* 通过自动摘要和卸载减少 token 使用

配置详情请参阅 [Context engineering](/oss/python/deepagents/context-engineering)。

## 代码执行

Deep Agents 以两种方式支持代码执行：

* [沙箱 backend](/oss/python/deepagents/sandboxes) 暴露一个 `execute` 工具，用于在隔离环境中运行 shell 命令。
* [Interpreters](/oss/python/deepagents/interpreters) 添加一个 `eval` 工具，在作用域化的 QuickJS 运行时中运行 JavaScript。

当代理需要安装依赖、运行测试、调用 CLI 或使用操作系统文件系统时，使用沙箱 backend。沙箱 backend 实现 `SandboxBackendProtocolV2`；检测到时，harness 会将 `execute` 工具添加到代理的可用工具中。

当代理需要一个轻量级可编程层用于循环、批处理、确定性数据转换或程序化工具调用时，使用 interpreters。Interpreters 不提供 shell 访问、包安装或文件系统/网络访问。

沙箱设置、提供商和文件传输 API，请参阅 [Sandboxes](/oss/python/deepagents/sandboxes)。QuickJS 运行时和程序化工具调用，请参阅 [Interpreters](/oss/python/deepagents/interpreters)。

## 人机交互

harness 可以在指定的工具调用处暂停代理执行，以允许人工批准或修改。此功能通过 `interrupt_on` 参数选择启用。

**配置：**

* 使用工具名称到中断配置的映射传递 `interrupt_on` 到 `create_deep_agent`
* 示例：`interrupt_on={"edit_file": True}` 在每次编辑前暂停
* 你可以在提示时提供批准消息或修改工具输入

**用途：**

* 破坏性操作的安全门
* 昂贵 API 调用前的用户验证
* 交互式调试和指导

## Skills

harness 支持 skills，为你的 deep agent 提供专门的工作流和领域知识。

**工作原理：**

* Skills 遵循 [Agent Skills 标准](https://agentskills.io/)
* 每个 skill 是一个包含 `SKILL.md` 文件的目录，文件中有指令和元数据
* Skills 可以包含额外的脚本、参考文档、模板和其他资源
* Skills 使用渐进式披露——它们仅在代理确定它们对当前任务有用时才加载
* 代理在启动时读取每个 `SKILL.md` 文件的 frontmatter，然后在需要时查看完整的 skill 内容

**用途：**

* 通过仅在需要时加载相关 skills 来减少 token 使用
* 将能力捆绑成带有额外上下文的更大操作
* 提供专业知识而不会混乱系统提示
* 实现模块化、可重用的代理能力

更多信息请参阅 [Skills](/oss/python/deepagents/skills)。

## Memory

harness 支持持久化 memory 文件，为你的 deep agent 跨对话提供额外上下文。这些文件通常包含通用编码风格、偏好、约定和指南，帮助代理理解如何处理你的代码库并遵循你的偏好。

**工作原理：**

* 使用 [`AGENTS.md` 文件](https://agents.md/) 提供持久化上下文
* Memory 文件始终加载（不像 skills 使用渐进式披露）
* 创建代理时传递一个或多个文件路径到 `memory` 参数
* 文件存储在代理的 backend 中（StateBackend、StoreBackend 或 FilesystemBackend）
* 代理可以根据你的交互、反馈和识别的模式更新 memory

**用途：**

* 提供不需要每次对话重新指定的持久化上下文
* 适用于存储用户偏好、项目指南或领域知识
* 始终对代理可用，确保一致的行为

配置详情和示例请参阅 [Memory](/oss/python/deepagents/customization#memory)。

## Harness Profiles

harness 可以在选择特定提供商或模型时应用声明式配置包（`HarnessProfile`）。Profiles 在模型构建后调整运行时行为，无需每个代理的设置代码。

**工作原理：**

* 在提供商名称（`"openai"`）或 `provider:model` 键（`"openai:gpt-5.4"`）下注册 profile
* `create_deep_agent` 在解析模型时查找并应用 profile
* 提供商级和模型级 profiles 在解析时合并

**用途：**

* 在一个地方打包每个提供商或每个模型的默认值（系统提示调整、工具覆盖、middleware）
* 切换模型时保持 `create_deep_agent` 调用点不变
* 通过 entry points 将可重用 profiles 作为插件发布

完整的字段列表、合并语义和插件打包，请参阅 [Profiles](/oss/python/deepagents/profiles)。
