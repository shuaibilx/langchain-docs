# Permissions - 总结

## 核心概念

Permissions 是声明式文件系统访问控制。通过 `FilesystemPermission` 规则列表配置，控制代理内置文件系统工具的读写权限。

> 需要 `deepagents>=0.5.2`

## 适用范围

| 覆盖 | 不覆盖 |
|------|--------|
| `ls`、`read_file`、`glob`、`grep` | 自定义工具 |
| `write_file`、`edit_file` | MCP 工具 |
| | 沙箱 backend 的 `execute` 工具 |

## 规则结构

```python
FilesystemPermission(
    operations=["read", "write"],  # read 或 write
    paths=["/workspace/**"],       # glob 模式
    mode="allow",                  # allow 或 deny
)
```

| 字段 | 值 | 说明 |
|------|-----|------|
| `operations` | `["read"]` | 覆盖 ls、read_file、glob、grep |
| | `["write"]` | 覆盖 write_file、edit_file |
| | `["read", "write"]` | 覆盖所有 |
| `paths` | glob 模式 | `**` 递归，`{a,b}` 交替 |
| `mode` | `"allow"` / `"deny"` | 默认 `"allow"` |

## 评估规则

```
首次匹配获胜（first-match-wins）
├── 规则 1 匹配？→ 使用规则 1 的 mode
├── 规则 2 匹配？→ 使用规则 2 的 mode
├── ...
└── 无规则匹配？→ 允许（宽容默认）
```

**关键：更具体的规则必须放在更广泛的规则之前。**

## 常见模式

### 只读代理

```python
permissions=[
    FilesystemPermission(operations=["write"], paths=["/**"], mode="deny"),
]
```

### 隔离到工作区

```python
permissions=[
    FilesystemPermission(operations=["read", "write"], paths=["/workspace/**"], mode="allow"),
    FilesystemPermission(operations=["read", "write"], paths=["/**"], mode="deny"),
]
```

### 保护特定文件

```python
permissions=[
    FilesystemPermission(operations=["read", "write"], paths=["/workspace/.env"], mode="deny"),
    FilesystemPermission(operations=["read", "write"], paths=["/workspace/**"], mode="allow"),
    FilesystemPermission(operations=["read", "write"], paths=["/**"], mode="deny"),
]
```

### 只读记忆

```python
permissions=[
    FilesystemPermission(operations=["write"], paths=["/memories/**", "/policies/**"], mode="deny"),
]
```

## Subagent 权限

- 默认继承父代理权限
- 设置 `permissions` 时**完全替换**父代理规则

```python
subagents=[{
    "name": "auditor",
    "permissions": [
        FilesystemPermission(operations=["write"], paths=["/**"], mode="deny"),
        FilesystemPermission(operations=["read"], paths=["/workspace/**"], mode="allow"),
        FilesystemPermission(operations=["read"], paths=["/**"], mode="deny"),
    ],
}]
```

## CompositeBackend 限制

当默认 backend 是沙箱时：
- 权限路径必须限定在已知路由前缀下
- 包含路由外路径 → `NotImplementedError`
- 原因：沙箱支持 `execute` 工具，路径限制无法阻止 shell 命令

```python
# ✅ 有效：权限限定在 /memories/ 路由
permissions=[FilesystemPermission(operations=["write"], paths=["/memories/**"], mode="deny")]

# ❌ 引发错误：/workspace/** 命中沙箱默认
permissions=[FilesystemPermission(operations=["write"], paths=["/workspace/**"], mode="deny")]
```

## Permissions vs Backend Policy Hooks

| 机制 | 适用场景 |
|------|---------|
| **Permissions** | 基于路径的允许/拒绝规则 |
| **Backend Policy Hooks** | 自定义验证逻辑（速率限制、审计、内容检查）或控制自定义工具 |

## 关键注意事项

1. **首次匹配获胜** — 规则顺序决定结果
2. **宽容默认** — 无规则匹配时允许操作
3. **Subagent 完全替换** — 不是合并，是替换
4. **沙箱限制** — 权限路径必须在路由前缀下
5. **仅内置工具** — 自定义和 MCP 工具不受覆盖
