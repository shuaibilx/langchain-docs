# Skills - 总结

## 核心概念

Skills 是可复用的代理能力，遵循 [Agent Skills 规范](https://agentskills.io/specification)。每个 skill 是一个目录，包含 `SKILL.md`（指令+元数据）和可选的脚本/文档/资产。

## 渐进式披露

```
启动时：读取所有 SKILL.md 的 frontmatter（名称、描述）
         ↓
收到提示：检查是否有 skill 描述匹配任务
         ↓
匹配成功：读取完整 SKILL.md + 支持文件
         ↓
执行：遵循 skill 指令
```

## SKILL.md Frontmatter 前置元数据字段

| 字段              | 必需  | 说明                          |
| --------------- | --- | --------------------------- |
| `name`          | ✅   | Skill 唯一名称                  |
| `description`   | ✅   | 描述（最大 1024 字符）              |
| `module`        | ❌   | 可导入模块路径（interpreter skills） |
| `license`       | ❌   | 许可证                         |
| `compatibility` | ❌   | 兼容性要求                       |
| `metadata`      | ❌   | 自定义元数据                      |
| `allowed-tools` | ❌   | 允许的工具                       |

> SKILL.md 最大 10 MB

## 三种 Backend 配置

| Backend               | Skills 来源                          | 说明                      |
| --------------------- | ---------------------------------- | ----------------------- |
| **StateBackend**      | `invoke(files={...})`              | 需要 `create_file_data()` |
| **StoreBackend**      | `store.put(namespace, key, value)` | 跨线程持久化                  |
| **FilesystemBackend** | 磁盘目录                               | 相对于 `root_dir`          |

## 源优先级

```python
skills=["/skills/user/", "/skills/project/"]
# 同名 skill：后面的覆盖前面的（最后获胜）
```

## Subagent Skills 继承

| 类型           | 继承     | 说明              |
| ------------ | ------ | --------------- |
| GP subagent  | ✅ 自动继承 | 主代理的 skills     |
| 自定义 subagent | ❌ 不继承  | 需要显式配置 `skills` |

> Skill 状态完全隔离：主代理 ↔ subagent 互不可见

## Skills vs Memory

|     | Skills     | Memory      |
| --- | ---------- | ----------- |
| 目的  | 按需能力       | 持久上下文       |
| 加载  | 仅在相关时      | 始终加载        |
| 格式  | `SKILL.md` | `AGENTS.md` |
| 分层  | 最后获胜       | 合并          |
| 适用  | 任务特定、可能很大  | 始终相关        |

## 代码执行方式

### Interpreter Skills

- 暴露可导入函数给解释器
- `module: index.ts` 在 frontmatter 中
- 代理使用 `await import("@/skills/<name>")` 导入
- 适合：可复用、确定性、太详细的逻辑

### 沙箱脚本执行

- Skill 脚本在沙箱中执行
- 需要 `SkillSandboxSyncMiddleware` 上传脚本
- 适合：需要依赖安装、测试运行、CLI 调用

## 关键注意事项

1. **渐进式披露** — 启动时只读 frontmatter，需要时才加载完整内容
2. **描述决定匹配** — 清晰具体的描述导致更好的匹配
3. **最后获胜** — 同名 skill 后面的源覆盖前面的
4. **状态隔离** — 主代理和 subagent 的 skills 互不可见
5. **10 MB 限制** — SKILL.md 文件超过此限制被跳过
