# Streaming - 总结

## 核心概念

Streaming 是 DeepAgent 的**传统流式传输系统**（v2 格式）——基于 LangGraph 的 `stream_mode` 和 `subgraphs` 参数，从主 agent 和 subagent 流式传输进度、token、工具调用和自定义更新。与 Event Streaming（v3 类型化投影 API）不同，v2 使用 chunk 分支模式。

## v2 vs v3 对比

| 特性 | Streaming (v2) | Event Streaming (v3) |
|------|---------------|---------------------|
| API | `stream_mode` + `subgraphs=True` | `stream.subagents` 投影 |
| 格式 | `StreamPart` dict（type/ns/data） | 类型化投影迭代器 |
| 消费方式 | 在 chunk type 上分支 | 每个投影独立迭代 |
| 推荐 | 旧版，仍可用 | 新应用推荐 |

## 启用方式

```python
for chunk in agent.stream(
    input,
    stream_mode="updates",  # 或 "messages"、"custom"、组合
    subgraphs=True,          # 必须启用
    version="v2",            # 推荐格式
):
```

## Namespace 标识

| Namespace | 来源 |
|-----------|------|
| `()` （空） | 主 agent |
| `("tools:abc123",)` | Subagent |
| `("tools:abc123", "model_request:def456")` | Subagent 内部节点 |

## 五种 Stream Mode

| Mode | 内容 | 用途 |
|------|------|------|
| `updates` | 每步完成时的状态更新 | 跟踪进度、生命周期 |
| `messages` | LLM token 流 | 实时显示生成内容 |
| `custom` | 用户定义信号 | 自定义进度事件 |
| `values` | 最终状态值 | 获取完整结果 |
| 组合模式 | `["updates", "messages", "custom"]` | 完整画面 |

## 四种流式传输能力

### 1. Subagent 进度（updates）

```python
for chunk in agent.stream(input, stream_mode="updates", subgraphs=True, version="v2"):
    if chunk["ns"]:
        # Subagent 事件
        print(f"[subagent: {chunk['ns']}]")
    else:
        # 主 agent 事件
        print("[main agent]")
```

### 2. LLM Token（messages）

```python
for chunk in agent.stream(input, stream_mode="messages", subgraphs=True, version="v2"):
    token, metadata = chunk["data"]
    is_subagent = any(s.startswith("tools:") for s in chunk["ns"])
    if token.content:
        print(token.content, end="", flush=True)
```

### 3. 工具调用（messages）

```python
if token.tool_call_chunks:
    for tc in token.tool_call_chunks:
        if tc.get("name"):
            print(f"Tool call: {tc['name']}")
        if tc.get("args"):
            print(tc["args"], end="", flush=True)
```

### 4. 自定义更新（custom）

```python
# 工具内部
from langgraph.config import get_stream_writer

writer = get_stream_writer()
writer({"status": "progress", "percent": 50})

# 消费端
for chunk in agent.stream(input, stream_mode="custom", subgraphs=True, version="v2"):
    print(chunk["data"])
```

## 生命周期跟踪模式

```
阶段 1: 检测启动
  → 主 agent model_request 中的 task 工具调用

阶段 2: 检测运行
  → tools:UUID namespace 的事件

阶段 3: 检测完成
  → 主 agent tools 节点的 tool 消息
```

## v2 格式结构

```python
chunk = {
    "type": "updates" | "messages" | "custom",
    "ns": (),  # 空 = 主 agent, ("tools:<id>",) = subagent
    "data": {...},  # 有效载荷
}
```

## 关键注意事项

1. **subgraphs=True** — 必须启用才能接收 subagent 事件
2. **version="v2"** — 推荐格式，消除嵌套 tuple 解包
3. **namespace 标识来源** — 空 = 主 agent，"tools:" 前缀 = subagent
4. **多模式组合** — `stream_mode=["updates", "messages", "custom"]`
5. **get_stream_writer** — 工具内部发出自定义事件
6. **v2 vs v1** — v2 统一格式，v1 需要 `(namespace, (mode, data))` 解包
7. **LangGraph >= 1.1** — v2 格式要求
