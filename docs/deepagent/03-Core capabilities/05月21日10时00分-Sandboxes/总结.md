# Sandboxes - 总结

## 核心概念

沙箱是隔离的执行环境，作为 DeepAgent 的 backend。与其他 backend 不同，沙箱除了文件系统工具外还提供 `execute` 工具，用于运行 shell 命令。

## 沙箱提供的能力

| 工具 | 说明 |
|------|------|
| `ls`、`read_file`、`write_file`、`edit_file`、`glob`、`grep` | 标准文件系统工具 |
| `execute` | 运行任意 shell 命令（沙箱独有） |

## 架构图

```
代理（主机/服务器）
├── LLM → 工具调用
│         ↓
│   backend.execute("python script.py")
│         ↓
└── 沙箱（隔离环境）
    ├── 文件系统
    ├── Shell
    └── 依赖
```

## 两种集成模式

| 模式 | 代理位置 | 优点 | 缺点 |
|------|---------|------|------|
| **代理在沙箱中** | 沙箱内 | 与本地开发镜像 | API 密钥在沙箱内，需重建镜像 |
| **沙箱作为工具** | 主机上 | 即时更新，密钥在外 | 网络延迟 |

## 两种生命周期范围

| 范围 | 说明 | 适用场景 |
|------|------|---------|
| **线程范围** | 每个对话一个沙箱 | 默认，隔离性好 |
| **助手范围** | 同一助手共享一个沙箱 | 文件/包跨对话持久化 |

### 线程范围

```python
thread_id = config["configurable"]["thread_id"]
sandbox = await client.find_one(labels={"thread_id": thread_id})
# 或创建新沙箱
```

### 助手范围

```python
assistant_id = config["configurable"]["assistant_id"]
sandbox = await client.find_one(labels={"assistant_id": assistant_id})
# 注意：需要 TTL 或清理逻辑
```

## execute() 方法

- 提供商必须实现的唯一方法
- 运行 shell 命令，返回 stdout/stderr + 退出码
- BaseSandbox 基类基于 execute() 构建所有文件系统操作
- 大输出自动保存到文件，代理用 read_file 增量读取

## 两种文件访问平面

| 平面 | 调用者 | 方法 | 用途 |
|------|--------|------|------|
| **代理工具** | LLM | `execute()` 内的 read/write | 代理执行任务 |
| **文件传输 API** | 应用代码 | `upload_files()` / `download_files()` | 种子/检索文件 |

### 文件传输 API

```python
# 种子：代理运行前
backend.upload_files([("/src/app.py", b"print('hello')")])

# 检索：代理完成后
results = backend.download_files(["/output/result.csv"])
```

## 安全考虑

### 沙箱不能防止的

| 风险 | 说明 |
|------|------|
| **上下文注入** | 攻击者通过输入控制代理在沙箱内执行命令 |
| **网络窃取** | 代理可通过 HTTP/DNS 发送数据（除非阻止网络） |

### 密钥处理

| 方式 | 推荐 | 说明 |
|------|------|------|
| 密钥在沙箱外的工具中 | ✅ 推荐 | 代理调用工具但看不到密钥 |
| 网络代理注入凭证 | ⚠️ 部分支持 | 代理发出普通请求，代理附加凭证 |
| 密钥注入沙箱 | ❌ 不推荐 | 可被上下文注入攻击窃取 |

### 最佳实践

1. **永远不在沙箱内放密钥**
2. 不需要时阻止网络访问
3. 审查沙箱输出后再操作
4. 使用 middleware 过滤敏感输出
5. 沙箱内一切视为不受信任

## 关键注意事项

1. **沙箱 = backend** — 实现 BackendProtocol + execute()
2. **BaseSandbox 基类** — 基于 execute() 构建所有文件操作
3. **线程范围默认** — 每个对话独立沙箱
4. **TTL 必需** — 防止资源泄漏
5. **密钥在外** — 永远不放密钥在沙箱内
