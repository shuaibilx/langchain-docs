# Human-in-the-Loop - 总结

## 核心概念

人机交互（HITL）允许在敏感工具执行前暂停代理，等待人工审批。通过 `interrupt_on` 参数配置，使用 LangGraph 的 interrupt 机制实现。

## 工作流程

```
代理调用工具 → interrupt_on 匹配？
├── 否 → 直接执行
└── 是 → 暂停，返回中断
         │
         ├── approve → 用原始参数执行
         ├── edit → 修改参数后执行
         ├── reject → 跳过执行
         └── respond → 返回人工消息作为结果
```

## interrupt_on 配置

```python
interrupt_on = {
    "delete_file": True,                                    # 默认四种决策
    "read_file": False,                                     # 禁用中断
    "send_email": {"allowed_decisions": ["approve", "reject"]},  # 自定义
}
```

| 值 | 效果 |
|----|------|
| `True` | 启用中断，允许 approve/edit/reject/respond |
| `False` | 禁用中断 |
| `{"allowed_decisions": [...]}` | 自定义允许的决策 |

## 四种决策类型

| 决策 | 用途 | 说明 |
|------|------|------|
| `approve` | 批准执行 | 使用代理提议的原始参数 |
| `edit` | 修改后执行 | 可修改工具参数 |
| `reject` | 拒绝执行 | 完全跳过工具调用 |
| `respond` | 返回消息 | 跳过执行，直接返回人工消息 |

## 处理中断

```python
# 1. 调用代理
result = agent.invoke(input, config=config, version="v2")

# 2. 检查中断
if result.interrupts:
    interrupt_value = result.interrupts[0].value
    action_requests = interrupt_value["action_requests"]
    review_configs = interrupt_value["review_configs"]

    # 3. 构建决策
    decisions = [{"type": "approve"}]

    # 4. 恢复执行
    result = agent.invoke(
        Command(resume={"decisions": decisions}),
        config=config,  # 必须使用相同 config
        version="v2",
    )
```

## 多工具调用

- 多个需要审批的工具被批量放在一个中断中
- decisions 列表必须与 action_requests 顺序匹配

## 编辑工具参数

```python
decisions = [{
    "type": "edit",
    "edited_action": {
        "name": action_request["name"],  # 必须包含工具名称
        "args": {"to": "team@company.com", ...}
    }
}]
```

## Subagent 中断

### 两种方式

| 方式 | 说明 |
|------|------|
| `interrupt_on` 配置 | 每个 subagent 可覆盖主代理设置 |
| `interrupt()` 原语 | 工具内直接调用，暂停等待审批 |

### Subagent 覆盖配置

```python
subagents=[{
    "name": "file-manager",
    "interrupt_on": {
        "delete_file": True,
        "read_file": True,  # 与主代理不同
    }
}]
```

### interrupt() 原语

```python
from langgraph.types import interrupt

@tool
def request_approval(action: str) -> str:
    approval = interrupt({"type": "approval_request", "action": action})
    if approval.get("approved"):
        return "Approved"
    return "Rejected"
```

## 必需组件

| 组件 | 用途 |
|------|------|
| `checkpointer` | 持久化中断和恢复之间的代理状态 |
| `thread_id` | 恢复时必须使用相同的 thread_id |
| `version="v2"` | invoke 调用时必须指定 |

## 风险分级配置

```
高风险（approve + edit + reject）：delete_file, send_email
中风险（approve + reject）：write_file
低风险（无中断）：read_file, list_files
```

## 关键注意事项

1. **checkpointer 必需** — 没有它 HITL 无法工作
2. **相同 config 恢复** — thread_id 必须一致
3. **决策顺序匹配** — decisions 与 action_requests 一一对应
4. **Subagent 可覆盖** — 子代理的 interrupt_on 优先于主代理
5. **interrupt() 原语** — 工具内可直接调用，更灵活
