# Subagents - 总结

## 核心概念

Subagent 是深度代理委派工作的机制。主代理通过 `task` 工具将任务委派给 subagent，subagent 在独立上下文中执行，只返回最终结果。这解决了**上下文膨胀问题**。

## 架构图

```
主代理 ──task()──→ Subagent（独立上下文）
                      │
                      ├── 工具调用 1
                      ├── 工具调用 2
                      ├── ...
                      └── 工具调用 N
                      │
                      ▼
                 压缩结果 ──→ 返回主代理
```

## 两种 Subagent 类型

| 类型 | 定义方式 | 适用场景 |
|------|---------|---------|
| **SubAgent（字典）** | `dict` 配置 | 大多数用例，简单配置 |
| **CompiledSubAgent** | 预编译 LangGraph 图 | 复杂工作流，需要自定义图逻辑 |

## SubAgent 字典字段

| 字段 | 必需 | 继承 | 说明 |
|------|------|------|------|
| `name` | ✅ | - | 唯一标识符，用于 `task()` 调用 |
| `description` | ✅ | - | 主代理据此决定何时委派 |
| `system_prompt` | ✅ | ❌ | 独立指令，不继承主代理 |
| `tools` | ❌ | ✅ | 默认继承主代理，指定时完全覆盖 |
| `model` | ❌ | ✅ | 默认继承主代理 |
| `middleware` | ❌ | ❌ | 不继承主代理 |
| `interrupt_on` | ❌ | ✅ | 默认继承，subagent 值覆盖 |
| `skills` | ❌ | ❌ | 不继承（仅 GP subagent 继承） |
| `response_format` | ❌ | - | 结构化输出 schema |
| `permissions` | ❌ | ✅ | 默认继承，设置时完全替换 |

## General-Purpose Subagent

- **自动添加**：除非用户自定义同名 subagent
- **默认能力**：与主代理相同（系统提示、工具、模型、skills）
- **覆盖方式**：传递 `name="general-purpose"` 的自定义 subagent
- **禁用方式**：`general_purpose_subagent=GeneralPurposeSubagentProfile(enabled=False)`

## 上下文传播规则

```
父代理上下文
├── runtime.context ──→ 自动传播到所有 subagent
├── system_prompt ───→ 不继承（各自独立）
├── tools ──────────→ 默认继承（可覆盖）
├── model ──────────→ 默认继承（可覆盖）
├── skills ─────────→ GP subagent 继承，自定义不继承
└── permissions ────→ 默认继承（可完全替换）
```

## 结构化输出

```python
subagent = {
    "response_format": PydanticModel,  # 父代理收到 JSON
}
# 需要 deepagents>=0.5.3
```

## 最佳实践

| 实践 | 说明 |
|------|------|
| 清晰描述 | 主代理据此选择 subagent |
| 详细系统提示 | 包含工具使用指导和输出格式 |
| 最小工具集 | 只给需要的工具 |
| 按任务选模型 | 不同模型擅长不同任务 |
| 返回简洁结果 | 限制字数，避免上下文膨胀 |
| 大数据写文件 | subagent 写文件，主代理按需读取 |

## 流式传输与追踪

- 代理名称在元数据中以 `lc_agent_name` 可用
- 可用于区分主代理和 subagent 的运行
- LangSmith 中可视化追踪

## 常见模式

### 多个专门 subagent

```
主代理
├── data-collector（收集数据）
├── data-analyzer（分析数据）
└── report-writer（撰写报告）
```

### 识别调用来源

```python
agent_name = runtime.config.get("metadata", {}).get("lc_agent_name")
```

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| subagent 未被调用 | 使描述更具体；指示主代理委派 |
| 上下文仍膨胀 | 指示 subagent 返回简洁结果；大数据写文件 |
| 选错 subagent | 在描述中清晰区分各 subagent |
