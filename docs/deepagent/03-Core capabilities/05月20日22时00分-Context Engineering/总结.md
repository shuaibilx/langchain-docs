# 上下文工程 - 总结

## 核心概念

上下文工程 = 以正确的格式提供正确的信息和工具，使代理可靠完成任务。

## 5 种上下文类型

| 类型 | 内容 | 范围 | 机制 |
|------|------|------|------|
| **输入上下文** | 系统提示、Memory、Skills、工具提示 | 静态，每次运行 | 启动时组装 |
| **运行时上下文** | user_id、API keys、feature flags | 每次运行 | `context` 参数 |
| **上下文压缩** | 卸载 + 摘要 | 自动触发 | 内置机制 |
| **上下文隔离** | 子代理返回压缩结果 | 每个子代理 | `task` 工具 |
| **长期记忆** | 跨对话持久化信息 | 跨线程 | CompositeBackend |

## 输入上下文组装顺序（完整系统提示）

```
1. 自定义 system_prompt
2. 基础代理提示（内置）
3. 待办列表提示（write_todos 指令）
4. Memory 提示（AGENTS.md + 使用指南）
5. Skills 提示（位置 + frontmatter + 用法）
6. 虚拟文件系统提示（ls/read/write/edit/glob/grep/execute）
7. 子代理提示（task 工具用法）
8. 用户 middleware 提示
9. 人机交互提示（interrupt_on）
```

## 输入上下文四大来源

| 来源 | 加载方式 | 用途 |
|------|---------|------|
| **系统提示** | 静态，每次运行 | 定义角色、行为、知识 |
| **Memory** | 始终加载 | 项目约定、用户偏好、关键指南 |
| **Skills** | 渐进式披露（按需） | 专门工作流、领域知识 |
| **工具提示** | 自动生成 | 工具 schema + 描述 + middleware 指令 |

## 运行时上下文

- 使用 `context_schema` 定义形状（dataclass 或 TypedDict）
- 通过 `context=` 参数传递给 `invoke`
- 在工具中通过 `runtime.context` 读取
- **自动传播到所有子代理**
- 不自动出现在模型提示中（需要工具/middleware 读取并添加）

## 上下文压缩两种机制

| 机制 | 触发条件 | 做什么 |
|------|---------|--------|
| **卸载** | 工具输入/结果 > 20,000 tokens | 存储到文件系统，替换为引用 |
| **摘要** | 上下文 > 85% max_input_tokens | LLM 生成结构化摘要替换历史 |

### 卸载细节

- **输入超限**：文件写入/编辑的工具调用被截断为文件指针
- **结果超限**：结果卸载到 backend，替换为路径 + 前 10 行预览

### 摘要细节

- 触发：85% max_input_tokens
- 保留：10% tokens 作为最近上下文
- 回退：无 profile → 170,000-token 触发 / 保留 6 条消息
- 溢出：ContextOverflowError → 立即摘要 + 重试
- 双重保留：摘要在上下文中，原文写入文件系统
- 可选摘要工具：`create_summarization_tool_middleware` 让代理主动触发

## 子代理上下文隔离

```
主代理 → task(subagent, description) → 子代理（独立上下文）
                                         ↓
                                    100+ 工具调用
                                         ↓
                                    压缩结果 ← 只有这个返回主代理
```

最佳实践：
1. 委派复杂任务
2. 子代理响应限制 500 字以内
3. 大数据写文件，主代理按需读取

## 长期记忆

- 默认文件系统：工作记忆限定在单个线程
- 长期记忆：`CompositeBackend` + `StoreBackend` 跨线程持久化
- 路径 `/memories/` 路由到 LangGraph Store
- 代理按需创建文件，无需预填充

## 最佳实践

1. Memory 最小化，Skills 聚焦
2. 子代理处理繁重工作
3. 子代理输出配置简洁
4. 大输出写文件
5. 记录 Memory 结构
6. 运行时上下文传递工具所需数据
