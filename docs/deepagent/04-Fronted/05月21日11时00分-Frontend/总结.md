# Frontend - 总结

## 核心概念

Frontend 是 DeepAgent 的**前端 UI 系统**——使用 `useStream` hook 构建实时可视化 deep agent 工作流的前端，包括 subagent 流式传输、待办事项列表和 sandbox IDE 体验。

## 四大前端模式

| 模式 | 核心功能 | 关键 API |
|------|---------|---------|
| **Overview** | 架构总览 | `useStream`、`stream.subagents`、`stream.values.todos` |
| **Subagent Streaming** | Subagent 卡片、进度跟踪 | `filterSubagentMessages`、`getSubagentsByMessage` |
| **Todo List** | 实时待办事项列表 | `stream.values.todos` |
| **Sandbox** | IDE 风格 UI | sandbox API、三面板布局 |

## useStream 核心配置

```ts
const stream = useStream<typeof agent>({
  apiUrl: "http://localhost:2024",
  assistantId: "agent",
  filterSubagentMessages: true,  // 过滤 subagent 消息
  threadId,                       // 持久化 thread
  onThreadId: updateThreadId,     // thread ID 回调
});
```

## Subagent Streaming

### 关键配置

```ts
filterSubagentMessages: true  // 必须设置！
```

### SubagentStreamInterface

| 属性 | 说明 |
|------|------|
| `id` | 唯一标识符 |
| `status` | `pending` → `running` → `complete` / `error` |
| `messages` | 实时消息流 |
| `result` | 最终输出（仅 complete 时） |
| `toolCall.args.description` | 任务描述 |
| `toolCall.args.subagent_type` | 专家类型 |
| `startedAt` / `completedAt` | 计时 |

### 核心 API

```ts
// 按消息获取 subagent
const subagents = stream.getSubagentsByMessage(msg.id);

// 获取所有 subagent
const all = [...stream.subagents.values()];
```

### UI 组件

```
MessageWithSubagents
├── 协调器消息
├── SubagentProgress（进度条）
└── SubagentCard × N
    ├── StatusIcon（○/◉/✓/✕）
    ├── StatusBadge
    ├── 任务描述
    ├── 流式内容/最终结果
    └── 计时信息
```

### SynthesisIndicator

所有 subagent 完成后显示"综合结果中..."指示器。

## Todo List

### Todo 接口

```ts
interface Todo {
  status: "pending" | "in_progress" | "completed";
  content: string;
}
```

### 访问方式

```ts
const todos = stream.values?.todos ?? [];
```

### UI 组件

```
TodoList
├── ProgressBar（进度条 + 百分比）
└── TodoItem × N
    ├── 状态图标（○/◉/✓）
    ├── 颜色编码
    └── 完成时删除线
```

### 自定义状态

`stream.values` 可以暴露 agent 定义的**任何自定义状态**，不仅仅是 `todos`。

## Sandbox IDE

### 三层架构

```
1. Deep Agent + Sandbox Backend
   → 自动获得 read_file/write_file/edit_file/execute

2. FastAPI API Server（langgraph.json http.app）
   → /api/sandbox/:threadId/tree
   → /api/sandbox/:threadId/file

3. IDE Frontend（三面板）
   → 文件树 | 代码/Diff | 聊天
```

### Sandbox 生命周期

| 范围 | 说明 | 适用场景 |
|------|------|---------|
| **Thread-scoped** | 每个 thread 独立 sandbox | 推荐，生产环境 |
| **Agent-scoped** | 同一 assistant 共享 | 持久项目 |
| **User-scoped** | 每个用户独立 | 多用户 |
| **Session-scoped** | 客户端生成 ID | 演示/原型 |

### 实时文件同步

监控 `write_file`、`edit_file`、`execute` 的 ToolMessage → 刷新对应文件。

### 三面板布局

| 面板 | 宽度 | 用途 |
|------|------|------|
| 文件树 | 208px | 浏览 + 更改指示器 |
| 代码/Diff | flex | 内容/统一 diff |
| 聊天 | 320px | Agent 交互 |

### langgraph.json 配置

```json
{
  "graphs": { "coding_agent": "./src/agents/my_agent.py:agent" },
  "http": { "app": "./src/api/server.py:app" }
}
```

## 关键最佳实践

1. **filterSubagentMessages: true** — 始终设置，否则消息交错不可读
2. **显示任务描述** — `toolCall.args.description` 告诉用户每个 subagent 做什么
3. **可折叠卡片** — 5+ subagent 时自动折叠已完成的
4. **显示计时数据** — 帮助用户理解性能
5. **递归限制** — Deep agent 需要更高限制，从 100 开始
6. **每个 subagent 独立处理错误** — 一个失败不应崩溃整个 UI
7. **Thread-scoped sandbox** — 生产环境推荐
8. **sessionStorage 持久化 threadId** — 页面重载时重连
9. **实时同步文件** — 不要等到 run 完成
10. **过滤 node_modules** — 文件树中排除
