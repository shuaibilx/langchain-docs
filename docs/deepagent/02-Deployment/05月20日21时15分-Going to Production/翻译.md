# 投入生产

> 使用持久化记忆、沙箱、弹性 middleware 和部署选项将 deep agent 投入生产

本指南涵盖将 deep agent 从本地原型转移到生产部署的考虑因素。它涵盖范围记忆、配置执行环境、添加防护栏和连接前端。

## 概述

代理使用来自记忆和执行环境的信息来完成任务。在生产环境中，有几个原语决定信息如何共享和访问：

* **Thread（线程）**：单个对话。消息历史和临时文件默认限定在线程范围内，不会跨线程传递。
* **User（用户）**：与代理交互的人。记忆和文件可以对用户私有或跨用户共享。身份和授权来自你的[认证层](/langsmith/auth)。
* **Assistant（助手）**：配置的代理实例。记忆和文件可以绑定到一个助手或跨所有助手共享。

本页涵盖：

* **[LangSmith 部署](#langsmith-deployments)**：带认证、webhooks 和 cron 的托管基础设施
* **[生产考虑因素](#production-considerations)**：调用、多租户、认证、凭据、异步和持久性
* **[记忆](#memory)**：跨对话持久化信息
* **[执行环境](#execution-environment)**：文件存储和代码执行
* **[防护栏](#guardrails)**：速率限制、错误处理和数据隐私
* **[前端](#frontend)**：将 UI 连接到已部署的代理

## LangSmith 部署

将 Deep Agent 投入生产的推荐路径是 [Managed Deep Agents](/langsmith/deploy-managed-deep-agent)，这是一个 API 优先的托管运行时，用于在 LangSmith 中创建、运行和操作 deep agents。Managed Deep Agents 目前处于私有预览阶段（[加入等待列表](https://www.langchain.com/langsmith-managed-deep-agents-waitlist)）。

对于需要自定义应用代码、自定义路由、高级认证或完整 Agent Server API 的团队，你可以直接配置 [LangSmith Deployment](/langsmith/deployment)。

两种路径都会配置你的代理所需的基础设施：[threads](/langsmith/use-threads)、[runs](/langsmith/runs)、store 和 checkpointer，因此你不需要自己设置这些。传统的 LangSmith Deployment 还提供开箱即用的[认证](/langsmith/auth)、[webhooks](/langsmith/use-webhooks)、[cron jobs](/langsmith/cron-jobs) 和[可观测性](/langsmith/observability)，并可以通过 [MCP](/langsmith/server-mcp) 或 [A2A](/langsmith/server-a2a) 暴露你的代理。

所有代码片段使用以下 `langgraph.json`：

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./agent.py:agent"
  },
  "env": ".env"
}
```

`langgraph.json` 是告诉 LangGraph 平台如何构建和运行应用的配置文件。它位于项目根目录，本地开发（`langgraph dev`）和生产部署都需要。关键字段：

| 字段 | 描述 |
|------|------|
| `dependencies` | 要安装的包。`["."]` 将当前目录作为包安装（从 `requirements.txt`、`pyproject.toml` 或 `package.json` 读取）。 |
| `graphs` | 将图 ID 映射到代码位置。每个条目是 `"<id>": "./<file>:<variable>"`，其中 `<id>` 是通过 API 调用图的名称，`<variable>` 是从 `<file>` 导出的编译图或构造函数。 |
| `env` | 包含环境变量（API keys、secrets）的 `.env` 文件路径。这些在构建时设置，在运行时可用。 |

## 生产考虑因素

### 调用代理

在生产环境中，每次调用应携带两个运行级参数：

* **`thread_id`**（通过 `config={"configurable": {"thread_id": ...}}` 传递）：对话的稳定标识符。[checkpointer](#durability) 使用它来持久化和恢复消息历史，因此后续轮次继续同一对话。生成新的 `thread_id` 开始新对话。
* **`context`**：你的工具和 middleware 在调用时读取的每次运行数据，例如 `user_id`、API keys、feature flags 或会话元数据。使用 `context_schema` 定义形状，通过 `runtime.context` 访问。

两者独立且几乎总是一起传递：

```python
from dataclasses import dataclass
from deepagents import create_deep_agent
from langchain_core.utils.uuid import uuid7

@dataclass
class Context:
    user_id: str

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    context_schema=Context,
)

# 开始对话
config = {"configurable": {"thread_id": str(uuid7())}}
agent.invoke(
    {"messages": [{"role": "user", "content": "Plan a 3-day trip to Tokyo"}]},
    config=config,
    context=Context(user_id="user-123"),
)

# 同一对话的后续：复用相同的 thread_id
agent.invoke(
    {"messages": [{"role": "user", "content": "Make it 5 days instead"}]},
    config=config,
    context=Context(user_id="user-123"),
)
```

使用 LangGraph SDK 部署时，SDK 为你管理线程：

```python
from langgraph_sdk import get_client

client = get_client(url="<DEPLOYMENT_URL>", api_key="<LANGSMITH_API_KEY>")

thread = await client.threads.create()
async for chunk in client.runs.stream(
    thread["thread_id"],
    "agent",
    input={"messages": [{"role": "user", "content": "Plan a 3-day trip to Tokyo"}]},
    context={"user_id": "user-123"},
    stream_mode="updates",
):
    print(chunk.data)
```

> `thread_id` 限定*对话*（消息历史、检查点）。`context` 携带你的工具和 middleware 读取的*每次运行*数据。它们独立：更改一个不影响另一个，你可以传递任一个或两个。

### 多租户

当你的代理服务多个用户时，你需要处理三个问题：验证每个用户是谁、控制他们可以访问什么、以及管理代理代表他们行事时使用的凭据。

#### 用户身份和访问控制

[LangSmith Deployments](/langsmith/deployment) 支持[自定义认证](/langsmith/custom-auth)来建立用户身份，以及[授权处理器](/langsmith/auth)来控制对线程、助手和 store 命名空间等资源的访问。授权处理器在认证成功后运行，可以：

* 用所有权元数据标记资源（例如 `owner: user_id`）
* 返回过滤器，使用户只能看到自己的资源
* 对未授权操作拒绝访问（HTTP 403）

#### 团队访问控制（RBAC）

LangSmith 的[基于角色的访问控制](/langsmith/rbac)管理团队中谁可以部署、配置和监控代理。这与上面的最终用户授权是分开的。

| 角色 | 访问权限 |
|------|---------|
| Workspace Admin | 完全权限，包括设置和成员管理 |
| Workspace Editor | 创建和修改资源，但不能删除 runs 或管理成员 |
| Workspace Viewer | 只读访问 |

#### 最终用户凭据

当你的代理需要代表用户调用外部 API（例如读取他们的 GitHub repos、发送 Slack 消息、查询数据仓库）时，你需要一种方式将用户的凭据传递给代理而不硬编码它们。

**通过 Agent Auth 的 OAuth。** [Agent Auth](/langsmith/agent-auth) 提供托管的 OAuth 2.0 流程。配置 OAuth 提供商，代理可以请求限定于每个用户的 token。首次使用时，代理[中断](/oss/python/langgraph/interrupts)执行并呈现 OAuth 同意 URL。用户认证后，代理使用有效 token 恢复。Token 自动存储和刷新。

```python
from langchain_auth import Client
from langchain.tools import tool, ToolRuntime

auth_client = Client()

@tool
async def github_action(runtime: ToolRuntime):
    """Perform an action on behalf of the user via GitHub."""
    auth_result = await auth_client.authenticate(
        provider="github",
        scopes=["repo", "read:org"],
        user_id=runtime.server_info.user.identity,
    )
    # Use auth_result.token for GitHub API calls
```

**沙箱凭据注入。** 如果你的代理在[沙箱](#sandboxes)内运行调用外部 API 的代码，[沙箱认证代理](/langsmith/sandbox-auth-proxy)可以自动将凭据注入出站请求，因此沙箱代码永远不会接收原始 API keys。

**工作区 secrets。** 对于跨所有用户共享的 API keys（例如组织的 LLM 提供商 keys、搜索 API keys），将它们存储为 LangSmith 中的[工作区 secrets](/langsmith/set-up-hierarchy#configure-workspace-settings)。

### 异步

基于 LLM 的应用是重度 I/O 密集型的：调用语言模型、数据库和外部服务。异步编程让这些操作并发运行而不是阻塞，提高吞吐量和响应性。

> LangChain 遵循在异步方法名前加 `a` 的约定（例如 `ainvoke`、`abefore_agent`、`astream`）。

构建生产应用时：

* **创建异步工具。** LangChain 在单独线程中运行同步工具以避免阻塞，但原生异步完全避免线程开销。
* **使用异步 middleware 方法。** 自定义 [middleware](/oss/python/langchain/middleware/custom) 应实现异步钩子（例如 `abefore_agent` 而非 `before_agent`）。
* **使用异步管理外部资源生命周期。** 创建[沙箱](#sandboxes)或连接到 [MCP servers](/oss/python/langchain/mcp) 涉及网络调用，应该被 await。

### 持久性

Deep Agents 运行在 LangGraph 上，LangGraph 开箱即用地提供[持久执行](/oss/python/langgraph/durable-execution)。[持久化](/oss/python/langgraph/persistence)层在每一步检查点状态，因此被故障、超时或[人机交互](/oss/python/langgraph/interrupts)暂停中断的 run 从最后记录的状态恢复，无需重新处理先前的步骤。对于生成许多子代理的长时间运行 deep agents，这意味着运行中失败不会丢失已完成的工作。

检查点还支持：

* **无限期[中断](/oss/python/langgraph/interrupts)。** 人机交互工作流可以暂停数分钟或数天，并在中断处精确恢复。
* **[时间旅行](/oss/python/langgraph/use-time-travel)。** 每个检查点步骤都是你可以回退的快照，让你在出问题时从早期状态重放。
* **安全处理敏感操作。** 对于涉及付款或其他不可逆操作的工作流，检查点提供审计跟踪和恢复点。

> [LangSmith Deployments](/langsmith/deployment) 自动配置持久 checkpointer。如果你自托管，请参阅[持久化](/oss/python/langgraph/persistence)了解设置说明。

## 记忆

没有记忆，每个对话都从零开始。记忆让你的代理跨对话保留信息（用户偏好、学习的指令、过去的经验），使其能够随时间个性化行为。

### 范围

记忆始终跨对话持久化。主要问题是它如何跨用户和助手边界范围化。正确的范围取决于谁应该查看和修改数据：

| 范围 | 命名空间 | 用例 | 示例 |
|------|---------|------|------|
| **User**（推荐默认） | `(user_id)` | 每用户偏好和上下文 | "我偏好简洁响应" |
| **Assistant** | `(assistant_id)` | 一个助手的共享指令 | "帖子限制 280 字符" |
| **Global** | `(org_id)` | 所有用户和助手的只读策略 | "永远不要泄露内部定价" |

> 共享记忆（助手、用户或组织范围）是提示注入的载体。如果一个用户可以写入另一个用户对话读取的记忆，恶意用户可以将指令注入该共享状态。在适当时强制执行只读访问。使用 [permissions](/oss/python/deepagents/permissions) 声明性地拒绝写入共享路径，或使用 [backend policy hooks](/oss/python/deepagents/backends#add-policy-hooks) 进行自定义验证逻辑。

### 配置

在 Deep Agents 中，记忆作为文件存储在虚拟文件系统中。默认情况下，文件限定在单个线程（对话）范围内，不跨线程共享。要跨线程共享记忆，将路径如 `/memories/` 路由到写入 LangGraph [Store](/langsmith/custom-store) 的 [StoreBackend](https://reference.langchain.com/python/deepagents/backends/store/StoreBackend)。使用 [CompositeBackend](https://reference.langchain.com/python/deepagents/backends/composite/CompositeBackend) 为代理提供线程范围的临时空间和跨线程的[长期记忆](/oss/python/deepagents/memory)。

**User 范围（推荐）：**
```python
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(
                namespace=lambda rt: (
                    rt.server_info.assistant_id,
                    rt.server_info.user.identity,
                ),
            ),
        },
    ),
    system_prompt="""You have persistent memory at /memories/.
    Read /memories/instructions.txt at the start of each conversation.""",
)
```

**Assistant 范围：**
```python
backend=CompositeBackend(
    default=StateBackend(),
    routes={
        "/memories/": StoreBackend(
            namespace=lambda rt: (rt.server_info.assistant_id,),
        ),
    },
)
```

**Organization 范围：**
```python
backend=CompositeBackend(
    default=StateBackend(),
    routes={
        "/memories/": StoreBackend(
            namespace=lambda rt: (rt.context.org_id,),
        ),
    },
)
```

## 执行环境

在本地，代理可以直接在磁盘上读写文件并运行 shell 命令。在生产环境中，你需要考虑隔离和持久性。正确的设置取决于代理是否需要执行代码：

* **文件系统 backend** 足够了，如果你的代理只读写文件。
* **沙箱** 添加带有 `execute` 工具的隔离容器来运行 shell 命令。如果代理需要运行代码、安装包或做超出文件 I/O 的任何事情，使用沙箱。

### 文件系统

根据需要持久化的内容选择 backend：

| Backend | 范围 | 持久性 |
|---------|------|--------|
| `StateBackend`（默认） | 线程 | 检查点内，避免写大文件 |
| `StoreBackend` | 跨线程 | Store 内，用 namespace factory 范围化 |
| `CompositeBackend` | 混合 | 默认线程范围 + 特定路径跨线程 |
| `ContextHubBackend` | LangSmith Hub | 持久化文件 |

> `FilesystemBackend` 和 `LocalShellBackend` 直接访问主机。不要在已部署的代理中使用它们。

### 沙箱

如果你的代理需要运行代码（不仅仅是读写文件），使用[沙箱](/oss/python/deepagents/sandboxes)。沙箱提供文件系统和用于运行 shell 命令的 `execute` 工具，全部在隔离容器内。

#### 生命周期

关键决策是沙箱存活多久。每个对话获得新的，还是对话共享持久环境？

| 范围 | 沙箱 ID 存储在 | 生命周期 | 示例用例 |
|------|---------------|---------|---------|
| **Thread-scoped** | Thread 元数据 | 每对话新建，TTL 到期清理 | 数据分析机器人 |
| **Assistant-scoped** | Assistant 配置 | 跨所有对话共享 | 维护克隆 repo 的编程助手 |

**Thread-scoped（最常见）：**
```python
from daytona import CreateSandboxFromSnapshotParams, Daytona
from deepagents import create_deep_agent
from langchain_core.runnables import RunnableConfig
from langchain_daytona import DaytonaSandbox

client = Daytona()

async def agent(config: RunnableConfig):
    thread_id = config["configurable"]["thread_id"]
    try:
        sandbox = await client.find_one(labels={"thread_id": thread_id})
    except Exception:
        sandbox = await client.create(
            CreateSandboxFromSnapshotParams(
                labels={"thread_id": thread_id},
                auto_delete_interval=3600,
            )
        )
    return create_deep_agent(
        model="anthropic:claude-sonnet-4-6",
        backend=DaytonaSandbox(sandbox=sandbox)
    )
```

**Assistant-scoped：**
```python
async def agent(config: RunnableConfig):
    assistant_id = config["configurable"]["assistant_id"]
    try:
        sandbox = await client.find_one(labels={"assistant_id": assistant_id})
    except Exception:
        sandbox = await client.create(
            CreateSandboxFromSnapshotParams(labels={"assistant_id": assistant_id})
        )
    return create_deep_agent(
        model="anthropic:claude-sonnet-4-6",
        backend=DaytonaSandbox(sandbox=sandbox)
    )
```

#### 文件传输

沙箱是隔离容器，因此你的应用代码不能直接访问其中的文件。使用 `upload_files()` 和 `download_files()` 跨沙箱边界移动数据：

* **代理运行前种子化沙箱**：上传用户文件、[skill](/oss/python/deepagents/skills) 脚本、配置或[持久记忆](/oss/python/deepagents/memory)
* **代理完成后检索结果**：下载生成的制品（报告、图表、导出）并同步更新的记忆

#### 管理 secrets

沙箱是隔离容器，因此主机的环境变量在其中不可用。有两种方式为沙箱代码提供 API keys 和其他 secrets：

**认证代理（推荐）。** [沙箱认证代理](/langsmith/sandbox-auth-proxy)拦截来自沙箱的出站请求并自动注入认证头。沙箱代码正常调用外部 API，代理根据目标主机添加正确的凭据。

```json
{
  "proxy_config": {
    "rules": [
      {
        "name": "openai-api",
        "match_hosts": ["api.openai.com"],
        "inject_headers": {
          "Authorization": "Bearer ${OPENAI_API_KEY}"
        }
      }
    ]
  }
}
```

**工作区 secrets。** 对于不需要基于代理注入的 API keys，将它们存储为 LangSmith 中的[工作区 secrets](/langsmith/set-up-hierarchy#configure-workspace-settings)。

> 避免通过环境变量或文件上传将 secrets 传入沙箱。代理可以读取沙箱内任何可访问的文件或环境变量。认证代理将 secrets 完全隔离在沙箱之外。

## 防护栏

生产中的代理自主运行，这意味着它们可以无限循环、触发速率限制或处理包含敏感信息的用户数据。Deep Agents 提供两层保护：

* **[Permissions](/oss/python/deepagents/permissions)**：声明式允许/拒绝规则，控制代理可以读写哪些文件和目录。
* **[Middleware](/oss/python/langchain/middleware/built-in)**：包装模型和工具调用的钩子，用于速率限制、错误处理和数据隐私。

### 速率限制

没有限制时，困惑的代理可以在几分钟内耗尽你的 LLM API 预算。设置模型调用和工具执行的每次运行上限：

```python
from deepagents import create_deep_agent
from langchain.agents.middleware import ModelCallLimitMiddleware, ToolCallLimitMiddleware

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    middleware=[
        ModelCallLimitMiddleware(run_limit=50),
        ToolCallLimitMiddleware(run_limit=200),
    ],
)
```

### 处理错误

并非所有错误都应以相同方式处理。瞬态故障（网络超时、速率限制）应自动重试。LLM 可以恢复的错误（错误工具输出、解析失败）应反馈给模型。需要人工输入的错误应暂停代理。

```python
from deepagents import create_deep_agent
from langchain.agents.middleware import (
    ModelFallbackMiddleware,
    ModelRetryMiddleware,
    ToolRetryMiddleware,
)

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    middleware=[
        ModelRetryMiddleware(max_retries=3, backoff_factor=2.0, initial_delay=1.0),
        ModelFallbackMiddleware("gpt-5.4"),
        ToolRetryMiddleware(
            max_retries=2,
            tools=["search", "fetch_url"],
            retry_on=(TimeoutError, ConnectionError),
        ),
    ],
)
```

### 数据隐私

如果你的代理处理可能包含电子邮件、信用卡号或其他 PII 的用户输入，你可以在它到达模型或存储在日志中之前检测和处理它：

```python
from deepagents import create_deep_agent
from langchain.agents.middleware import PIIMiddleware

agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    middleware=[
        PIIMiddleware("email", strategy="redact", apply_to_input=True),
        PIIMiddleware("credit_card", strategy="mask", apply_to_input=True),
    ],
)
```

策略包括 `redact`（替换为 `[REDACTED_EMAIL]`）、`mask`（部分掩码如 `****-****-****-1234`）、`hash`（确定性哈希）和 `block`（引发错误）。

## 前端

Deep Agents 使用 [`useStream`](/oss/python/langchain/frontend/overview) 将 UI 连接到代理后端。`useStream` 是一个前端钩子（可用于 React、Vue、Svelte 和 Angular），实时从代理流式传输消息、子代理进度和自定义状态。

```tsx
import { useStream } from "@langchain/react";

function App() {
  const stream = useStream<typeof agent>({
    apiUrl: "https://your-deployment.langsmith.dev",
    assistantId: "agent",
    reconnectOnMount: true,    // 页面刷新后恢复流
    fetchStateHistory: true,   // 加载完整线程历史
  });
}
```

`reconnectOnMount` 自动拾取进行中的 run。`fetchStateHistory` 加载线程的完整对话历史。

对于生成许多子代理的 deep agent 工作流，提交时设置高 `recursionLimit`：

```tsx
stream.submit(
  { messages: [{ type: "human", content: text }] },
  {
    streamSubgraphs: true,
    config: { recursionLimit: 10000 },
  },
);
```
