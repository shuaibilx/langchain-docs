# Going to Production - 总结

## 核心概念

生产环境中三个关键范围原语：
- **Thread**：单个对话，消息历史默认限定
- **User**：交互者，记忆可私有或共享
- **Assistant**：配置的代理实例

## 调用代理（两个必需参数）

```python
config = {"configurable": {"thread_id": str(uuid7())}}  # 对话标识
context = Context(user_id="user-123")                    # 每次运行数据

agent.invoke(input, config=config, context=context)
```

## 多租户三层认证

| 层 | 功能 | 实现 |
|---|---|---|
| 用户身份 | 验证谁在用 | Custom Auth |
| 访问控制 | 控制能访问什么 | Authorization Handlers |
| 团队 RBAC | 团队权限管理 | LangSmith RBAC |

## 最终用户凭据

| 方式 | 适用场景 |
|------|---------|
| Agent Auth (OAuth) | 代理代表用户调用第三方 API |
| Sandbox Auth Proxy | 沙箱内代码调用外部 API |
| Workspace Secrets | 组织级共享 API keys |

## 记忆范围

| 范围 | 命名空间 | 用例 |
|------|---------|------|
| User（推荐） | `(assistant_id, user_id)` | 每用户偏好 |
| Assistant | `(assistant_id)` | 助手级共享指令 |
| User-only | `(user_id)` | 跨助手的用户全局 profile |
| Organization | `(org_id)` | 组织级只读策略 |

## 执行环境

### 文件系统 Backend
- `StateBackend`：线程范围，检查点内
- `StoreBackend`：跨线程，Store 内
- `CompositeBackend`：混合路由

### 沙箱生命周期

| 范围 | 适用场景 | 清理 |
|------|---------|------|
| Thread-scoped | 每对话干净环境 | TTL 到期 |
| Assistant-scoped | 共享 repo/依赖 | 手动/快照重置 |

### 管理 Secrets
- **Auth Proxy**（推荐）：拦截请求注入凭据，secrets 不进沙箱
- **Workspace Secrets**：组织级共享 keys

## 防护栏

| Middleware | 功能 |
|-----------|------|
| ModelCallLimitMiddleware | 模型调用次数上限 |
| ToolCallLimitMiddleware | 工具调用次数上限 |
| ModelRetryMiddleware | 模型调用重试 |
| ModelFallbackMiddleware | 模型故障回退 |
| ToolRetryMiddleware | 特定工具重试 |
| PIIMiddleware | PII 检测和脱敏 |

## 关键注意事项

1. **thread_id + context 几乎总是一起传递**
2. **异步优先**：工具/middleware/resource lifecycle 都用 async
3. **沙箱 secrets 用 Auth Proxy**，不要传环境变量
4. **检查点支持**：无限中断 + 时间旅行 + 故障恢复
5. **PII 脱敏在 middleware 层**，在数据到达模型前处理
