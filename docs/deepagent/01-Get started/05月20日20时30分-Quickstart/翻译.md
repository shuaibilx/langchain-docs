# 快速入门

> 几分钟内构建你的第一个 deep agent

本指南将引导你创建第一个具有规划、文件系统工具和子代理功能的 deep agent。你将构建一个可以进行研究并撰写报告的研究代理。

> **使用 AI 编程助手？**
>
> * 安装 [LangChain Docs MCP server](/use-these-docs) 为你的代理提供最新的 LangChain 文档和示例访问权限。
> * 安装 [LangChain Skills](https://github.com/langchain-ai/langchain-skills) 提高你的代理在 LangChain 生态任务上的表现。

## 前提条件

在开始之前，确保你有来自模型提供商（例如 Gemini、Anthropic、OpenAI）的 API key。

> Deep Agents 需要支持 [tool calling](/oss/python/langchain/models#tool-calling) 的模型。参见[自定义](/oss/python/deepagents/customization#model)了解如何配置你的模型。

## 步骤 1：安装依赖

```bash
pip install deepagents tavily-python
```

或使用 uv：

```bash
uv init
uv add deepagents tavily-python
uv sync
```

> 本指南使用 [Tavily](https://tavily.com/) 作为示例搜索提供商，但你可以替换为任何搜索 API（例如 DuckDuckGo、SerpAPI、Brave Search）。

## 步骤 2：设置 API keys

根据你的模型提供商设置相应的环境变量：

```bash
# Google
export GOOGLE_API_KEY="your-api-key"

# OpenAI
export OPENAI_API_KEY="your-api-key"

# Anthropic
export ANTHROPIC_API_KEY="your-api-key"

# OpenRouter
export OPENROUTER_API_KEY="your-api-key"

# Fireworks
export FIREWORKS_API_KEY="your-api-key"

# Baseten
export BASETEN_API_KEY="your-api-key"

# Ollama
export OLLAMA_API_KEY="your-api-key"

# 搜索 API
export TAVILY_API_KEY="your-tavily-api-key"
```

Deep Agents 可与任何 [LangChain chat model](/oss/python/deepagents/models#supported-models) 配合使用。

## 步骤 3：创建搜索工具

```python
import os
from typing import Literal

from tavily import TavilyClient
from deepagents import create_deep_agent

tavily_client = TavilyClient(api_key=os.environ["TAVILY_API_KEY"])


def internet_search(
    query: str,
    max_results: int = 5,
    topic: Literal["general", "news", "finance"] = "general",
    include_raw_content: bool = False,
):
    """Run a web search"""
    return tavily_client.search(
        query,
        max_results=max_results,
        include_raw_content=include_raw_content,
        topic=topic,
    )
```

## 步骤 4：创建 deep agent

以 `provider:model` 格式传递 `model` 字符串，或传递一个[已初始化的模型实例](/oss/python/deepagents/models#configure-model-parameters)。参见[支持的模型](/oss/python/deepagents/models#supported-models)了解所有提供商，参见[推荐模型](/oss/python/deepagents/models#suggested-models)了解经过测试的推荐。

```python
# System prompt to steer the agent to be an expert researcher
research_instructions = """You are an expert researcher. Your job is to conduct thorough research and then write a polished report.

You have access to an internet search tool as your primary means of gathering information.

## `internet_search`

Use this to run an internet search for a given query. You can specify the max number of results to return, the topic, and whether raw content should be included.
"""

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",  # 或其他提供商
    tools=[internet_search],
    system_prompt=research_instructions,
)
```

支持的提供商示例：
- `google_genai:gemini-3.1-pro-preview`
- `openai:gpt-5.4`
- `anthropic:claude-sonnet-4-6`
- `openrouter:anthropic/claude-sonnet-4-6`
- `fireworks:accounts/fireworks/models/qwen3p5-397b-a17b`
- `baseten:zai-org/GLM-5`
- `ollama:devstral-2`

## 步骤 5：运行 agent

```python
result = agent.invoke({"messages": [{"role": "user", "content": "What is langgraph?"}]})

# Print the agent's response
print(result["messages"][-1].content)
```

> 使用 [LangSmith](https://smith.langchain.com) 追踪你的代理的规划步骤、工具调用和子代理委派。参见[可观测性快速入门](/langsmith/observability-quickstart)进行设置。

## 工作原理

你的 deep agent 自动：

1. **规划方法**：使用内置的 [`write_todos`](/oss/python/deepagents/harness#planning-capabilities) 工具分解研究任务。
2. **进行研究**：调用 `internet_search` 工具收集信息。
3. **管理上下文**：使用文件系统工具（[`write_file`](/oss/python/deepagents/harness#virtual-filesystem-access)、[`read_file`](/oss/python/deepagents/harness#virtual-filesystem-access)）卸载大型搜索结果。
4. **生成子代理**：根据需要将复杂子任务委派给专门的子代理。
5. **综合报告**：将研究结果编译成连贯的响应。

## 流式传输

Deep Agents 内置[流式传输](/oss/python/langchain/event-streaming)，使用 LangGraph 从代理执行中获取实时更新。这允许你逐步观察输出，并审核和调试代理及子代理的工作，如工具调用、工具结果和 LLM 响应。
