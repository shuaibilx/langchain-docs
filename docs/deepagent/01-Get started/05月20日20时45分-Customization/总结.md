# DeepAgent Customization - 总结

## create_deep_agent 完整参数

```python
create_deep_agent(
    model,                  # 模型（字符串或实例）
    tools,                  # 自定义工具列表
    system_prompt=...,      # 系统提示
    middleware=[],          # 中间件列表
    subagents=[],           # 子代理
    skills=[],              # 技能路径
    memory=[],              # 记忆文件路径
    backend=...,            # 文件系统后端
    interrupt_on={},        # 人机交互配置
    response_format=...,    # 结构化输出 schema
    checkpointer=...,       # 检查点（HITL 必需）
    store=...,              # 长期存储
)
```

## 自定义维度总览

| 维度 | 说明 | 关键 API |
|------|------|---------|
| Model | 字符串/实例/三种配置方式 | `provider:model` |
| Tools | 自定义函数工具 | `def func(...)` |
| System Prompt | 4 层组装：USER → BASE/CUSTOM → SUFFIX | `system_prompt=` |
| Middleware | 拦截/扩展执行流程 | `@wrap_tool_call` |
| Interpreters | JS 代码执行 | `CodeInterpreterMiddleware` |
| Subagents | 子代理委派 | `{"name", "description", "tools"}` |
| Backends | 虚拟文件系统 | State/File/Store/Composite |
| Sandboxes | 隔离执行环境 | Modal/Runloop/Daytona/LangSmith |
| HITL | 人工审批 | `interrupt_on={}` |
| Skills | 按需加载的能力 | `skills=["/skills/"]` |
| Memory | AGENTS.md 上下文 | `memory=["/AGENTS.md"]` |
| Profiles | 模型特定调整 | `register_harness_profile()` |
| 结构化输出 | Pydantic schema | `response_format=Model` |

## System Prompt 组装规则

```
USER (你的 system_prompt)
  ↓
BASE (SDK 默认) 或 CUSTOM (profile 覆盖)
  ↓
SUFFIX (profile 后缀)
```

- `USER` 始终在最前
- `SUFFIX` 始终在最后
- 用 `\n\n` 连接

## 内置 Middleware

| Middleware | 功能 |
|-----------|------|
| TodoListMiddleware | 待办事项管理 |
| FilesystemMiddleware | 文件系统操作 |
| SubAgentMiddleware | 子代理协调 |
| SummarizationMiddleware | 对话摘要压缩 |
| AnthropicPromptCachingMiddleware | Anthropic 提示缓存 |
| PatchToolCallsMiddleware | 中断工具调用修复 |
| MemoryMiddleware | 跨会话记忆 |
| SkillsMiddleware | 技能加载 |

## Backend 类型

| Backend | 范围 | 持久性 |
|---------|------|--------|
| StateBackend | 线程 | 检查点内 |
| FilesystemBackend | 本地磁盘 | 永久 |
| LocalShellBackend | 本地磁盘 + shell | 永久 |
| StoreBackend | 跨线程 | Store 内 |
| CompositeBackend | 路由 | 取决于子 backend |

## 关键注意事项

1. **HITL 需要 checkpointer** — `interrupt_on` 配置必须配合 checkpointer
2. **不要变异 middleware 属性** — 用图状态而非 `self.x`
3. **Skills 按需加载** — 减少启动时 token 消耗
4. **Sandbox 需要清理** — `finally` 块中 terminate/stop
