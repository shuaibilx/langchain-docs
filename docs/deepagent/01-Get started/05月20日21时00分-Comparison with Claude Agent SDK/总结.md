# Deep Agents vs Claude Agent SDK - 总结

## 核心差异

| 维度 | Deep Agents | Claude Agent SDK |
|------|-------------|-----------------|
| 模型 | 任意（100+） | 仅 Claude |
| 沙箱模式 | 沙箱内 或 沙箱作工具 | 仅沙箱内 |
| 多租户 | 内置 | 自行构建 |
| 部署 | 托管 或 自托管 | 仅自托管 |
| 后端 | 可插拔（6种） | 本地文件系统 |
| 许可证 | MIT | MIT |

## 关键区别详解

### 1. 执行环境
- **Deep Agents**：两种模式——代理在沙箱内，或代理在外部将沙箱作为工具
- **Claude Agent SDK**：仅代理在沙箱内

### 2. 多租户
- **Deep Agents**：内置范围线程、每用户沙箱、RBAC
- **Claude Agent SDK**：需要自行构建 API 包装器

### 3. 部署
- **Deep Agents**：Managed Deep Agents（LangSmith）或 `langgraph build` 独立镜像
- **Claude Agent SDK**：自托管，需自建服务器/认证/流式层

### 4. 生态
- **Deep Agents**：LangChain 生态（LangSmith 可观测性、评估、部署）
- **Claude Agent SDK**：Anthropic 生态

## 选择建议

```
需要模型灵活性？ ──→ Deep Agents
需要多租户？ ─────→ Deep Agents
需要托管部署？ ──→ Deep Agents
已投入 Anthropic？ → Claude Agent SDK
需要自托管控制？ → Claude Agent SDK
```
