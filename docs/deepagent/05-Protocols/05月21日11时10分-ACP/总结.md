# ACP - 总结

## 核心概念

ACP（Agent Client Protocol）是 DeepAgent 的**编辑器集成协议**——标准化编码 agent 与代码编辑器/IDE 之间的通信，让 deep agent 可以在任何兼容 ACP 的客户端中运行。

## ACP vs MCP

| 协议 | 方向 | 用途 |
|------|------|------|
| **ACP** | 编辑器 → Agent | Agent 被编辑器调用，接收项目上下文 |
| **MCP** | Agent → 工具服务器 | Agent 调用外部工具服务器 |

## 架构

```
代码编辑器（Zed/JetBrains/VSCode/Neovim）
    │
    │ ACP 协议（stdio）
    ▼
ACP Agent Server（deepagents-acp）
    │
    ▼
Deep Agent（create_deep_agent）
```

## 安装

```bash
pip install deepagents-acp
# 或
uv add deepagents-acp
```

## 核心代码

```python
from acp import run_agent
from deepagents import create_deep_agent
from deepagents_acp.server import AgentServerACP

agent = create_deep_agent(model="...", checkpointer=MemorySaver())
server = AgentServerACP(agent)
await run_agent(server)  # stdio 模式
```

## 关键组件

| 组件 | 说明 |
|------|------|
| `AgentServerACP` | 将 DeepAgent 包装为 ACP 服务器 |
| `run_agent()` | 在 stdio 模式下启动 ACP 服务器 |
| `MemorySaver` | 通常需要 checkpointer 来维持会话 |

## 支持的客户端

| 客户端 | 配置方式 |
|--------|---------|
| **Zed** | `settings.json` → `agent_servers` |
| **JetBrains IDEs** | 内置 ACP 支持 |
| **VS Code** | vscode-acp 扩展 |
| **Neovim** | ACP 兼容插件 |
| **Toad** | 本地开发工具管理 |

## Zed 配置流程

```
1. git clone deepagents
2. uv sync --all-groups
3. 配置 .env（ANTHROPIC_API_KEY）
4. settings.json 添加 agent_servers
5. 打开 Agents 面板
```

## Toad 本地开发

```bash
uv tool install -U batrachian-toad
toad acp "python path/to/your_server.py" .
```

## 关键注意事项

1. **stdio 模式** — ACP 服务器通过 stdin/stdout 通信，不是 HTTP
2. **checkpointer** — 通常需要 MemorySaver 来维持会话状态
3. **ACP vs MCP** — 前者面向编辑器集成，后者面向工具调用
4. **自定义** — 可以设置自定义提示、工具、middleware、subagent
5. **开箱即用** — `deepagents-acp` 包含示例 coding agent
