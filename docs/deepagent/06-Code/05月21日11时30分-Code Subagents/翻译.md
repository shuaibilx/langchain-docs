# 在 Deep Agents Code 中使用 Subagent

> 将自定义 Deep Agents Code subagent 定义为带 YAML frontmatter 的 AGENTS.md 文件。涵盖项目和用户路径、可选的模型覆盖和示例。

将自定义同步 [subagent](/oss/python/deepagents/subagents) 定义为 markdown 文件，以便 Deep Agents Code 可以将专业化任务委派给它们。

> 异步 subagent 目前在 Deep Agents Code 中不可用。

每个 subagent 位于自己的文件夹中，包含一个 `AGENTS.md` 文件：

```
.deepagents/agents/{subagent-name}/AGENTS.md   # 项目级
~/.deepagents/{agent}/agents/{subagent-name}/AGENTS.md  # 用户级
```

项目 subagent 覆盖同名的用户 subagent（参见[优先级规则](/oss/python/deepagents/code/data-locations#subagents)）。

Frontmatter 需要 `name` 和 `description`（与 [`SubAgent` 字典规范](/oss/python/deepagents/subagents#subagent-dictionary-based)相同）。Markdown 正文成为 subagent 的 `system_prompt`。除了基本规范外，`AGENTS.md` 文件支持可选的 `model` frontmatter 字段，覆盖此 subagent 的主 agent 模型。使用 `provider:model-name` 格式（例如 `anthropic:claude-opus-4-7`、`openai:gpt-5.5`）。省略则继承主 agent 的模型。

> 其他 `SubAgent` 字段（`tools`、`middleware`、`interrupt_on`、`skills`）目前无法通过 `AGENTS.md` frontmatter 配置——这样定义的自定义 subagent 继承主 agent 的工具。使用 SDK 直接获取完全控制。

## 文件格式

Subagent `AGENTS.md` 文件使用 YAML frontmatter 后跟 markdown 正文：

```markdown
---
name: researcher
description: Research topics on the web before writing content
model: anthropic:claude-haiku-4-5-20251001
---

You are a research assistant with access to web search.

## Your Process
1. Search for relevant information
2. Summarize findings clearly
```

## 示例：成本高效的 subagent

对简单的委派任务使用更便宜、更快的模型，同时保持主 agent 使用更强大的模型：

```markdown
---
name: general-purpose
description: General-purpose agent for research and multi-step tasks
model: anthropic:claude-haiku-4-5-20251001
---

You are a general-purpose assistant. Complete the task efficiently and return a concise summary.
```

这覆盖了内置的通用 subagent，将所有委派任务路由到更便宜的模型。参见 [Override the general-purpose subagent](/oss/python/deepagents/subagents#override-the-general-purpose-subagent) 了解更多。
