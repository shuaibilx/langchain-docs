# Deep Agents Code

> 基于 Deep Agents SDK 构建的终端编码 agent

Deep Agents Code (`dcode`) 是一个基于 [Deep Agents SDK](/oss/python/deepagents/quickstart) 构建的开源编码 agent。它支持任何支持工具调用的 LLM，并允许你在输入之间切换 LLM。它保留来自对话的学习的持久记忆，跨会话维持上下文，使用可自定义的技能，并通过审批控制执行代码。

## 快速开始

### 安装和启动

OpenAI、Anthropic 和 Google 默认安装。其他提供商（Ollama、Groq、xAI 等）作为可选额外组件可用。

```bash
# 脚本安装
curl -LsSf https://langch.in/dcode | bash

# 带额外提供商安装
DEEPAGENTS_EXTRAS="fireworks,nvidia" curl -LsSf https://langch.in/dcode | bash

# 使用 uv 安装
uv tool install 'deepagents-code[fireworks,nvidia]'

# 启动
dcode
```

### 添加提供商凭证

Deep Agents Code 支持任何支持工具调用的 LLM。使用 `/auth` 命令为你选择的提供商设置 API key。

> Web 搜索使用 [Tavily](https://tavily.com)，**不**通过 `/auth` 配置。如果启动时看到 "Web search disabled — `TAVILY_API_KEY` is not set"，将 `TAVILY_API_KEY=tvly-...` 添加到 `~/.deepagents/.env` 并运行 `/reload`。

### 给 agent 任务

```txt
Create a Python script that prints "Hello, World!"
```

Agent 解释查询并在修改文件之前提出带 diff 的更改供你审批。如果需要，它可以运行 shell 命令来测试代码、检查文档或搜索网络获取最新信息。

### 启用追踪（可选）

要记录 agent 操作、工具调用和决策到 LangSmith，添加以下到 `~/.deepagents/.env`：

```bash
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=lsv2_...
LANGSMITH_PROJECT=optional-project-name
```

> Deep Agents Code 在 Windows 上未正式支持。Windows 用户可以尝试在 WSL 下运行。

## 内置能力

| 能力 | 说明 |
|------|------|
| **文件操作** | 读取、写入和编辑文件 |
| **Shell 执行** | 执行命令运行测试、构建项目、管理依赖 |
| **远程 sandbox** | 在 LangSmith/Daytona/Modal/Runloop/AgentCore 中运行工具 |
| **Web 搜索** | 搜索最新信息和文档（需要 Tavily API key） |
| **HTTP 请求** | 向 API 和外部服务发送请求 |
| **任务规划和跟踪** | 将复杂任务分解为离散步骤并跟踪进度 |
| **Subagents** | 使用 `task` 工具委派工作，通过 `AGENTS.md` 文件定义 |
| **记忆存储和检索** | 跨会话存储和检索信息 |
| **上下文压缩和卸载** | 总结旧消息并卸载到存储，释放上下文窗口空间 |
| **Human-in-the-loop** | 对敏感工具操作需要人工审批 |
| **Skills** | 通过自定义专业知识和指令扩展 agent 能力 |
| **MCP 工具** | 从 MCP 服务器加载外部工具 |
| **追踪** | 在 LangSmith 中追踪 agent 操作 |

## 内置工具

| 工具 | 描述 | 需要审批 |
|------|------|---------|
| `ls` | 列出文件和目录 | - |
| `read_file` | 读取文件内容 | - |
| `write_file` | 创建或覆盖文件 | 是 |
| `edit_file` | 对现有文件进行定向编辑 | 是 |
| `glob` | 查找匹配模式的文件 | - |
| `grep` | 跨文件搜索文本模式 | - |
| `execute` | 本地或远程 sandbox 执行 shell 命令 | 是 |
| `web_search` | 使用 Tavily 搜索网络 | 是 |
| `fetch_url` | 获取网页并转换为 markdown | 是 |
| `task` | 委派工作给 subagent 并行执行 | 是 |
| `ask_user` | 向用户提问（自由文本或选择题） | - |
| `compact_conversation` | 总结旧消息，卸载到后端存储 | 混合 |
| `write_todos` | 创建和管理任务列表 | - |

> 潜在破坏性操作在执行前需要用户审批。要跳过审批：`dcode -y` 或 `dcode --auto-approve`

> 非交互模式下（`-n` 或管道 stdin），shell 执行默认禁用。使用 `-S` 指定允许的命令。

## 命令参考

```bash
# 使用特定 agent 配置
dcode --agent mybot

# 使用特定模型
dcode --model anthropic:claude-opus-4-7

# 自动审批工具使用
dcode -y

# 带启动命令
dcode --startup-cmd "ls -la" -m "Summarize what's in this directory"

# 非交互模式
dcode --startup-cmd "git diff --stat" -n "Review these changes"
```

### 命令行选项

| 选项 | 描述 |
|------|------|
| `-a`, `--agent NAME` | 使用命名 agent，独立记忆 |
| `-M`, `--model MODEL` | 使用特定模型（`provider:model`） |
| `--model-params JSON` | 传递给模型的额外 kwargs |
| `-r`, `--resume [ID]` | 恢复会话 |
| `-m`, `--message TEXT` | 会话启动时自动提交的初始提示 |
| `--skill NAME` | 启动时调用技能 |
| `--startup-cmd CMD` | 启动时运行的 shell 命令 |
| `-n`, `--non-interactive TEXT` | 非交互运行单个任务并退出 |
| `--max-turns N` | 限制非交互模式的 agentic 轮次 |
| `--timeout SECONDS` | 非交互模式的硬性挂钟超时 |
| `-q`, `--quiet` | 干净输出适合管道 |
| `--no-stream` | 缓冲完整响应后一次性写入 stdout |
| `-y`, `--auto-approve` | 自动审批所有工具调用 |
| `-S`, `--shell-allow-list LIST` | 逗号分隔的 shell 命令白名单 |
| `--sandbox TYPE` | 远程 sandbox（`none`/`langsmith`/`agentcore`/`modal`/`daytona`/`runloop`） |
| `--mcp-config PATH` | 添加显式 MCP 配置 |
| `--no-mcp` | 禁用所有 MCP 工具加载 |
| `--acp` | 作为 ACP 服务器在 stdio 上运行 |
| `-v`, `--version` | 显示版本 |

### CLI 命令

| 命令 | 描述 |
|------|------|
| `dcode agents list` | 列出所有 agent |
| `dcode agents reset --agent NAME` | 清除 agent 记忆并重置 |
| `dcode update` | 检查并安装更新 |
| `dcode skills list [--project]` | 列出所有技能 |
| `dcode skills create NAME [--project]` | 创建新技能 |
| `dcode skills delete NAME [--project]` | 删除技能 |
| `dcode threads list [--agent NAME]` | 列出会话 |
| `dcode threads delete ID` | 删除会话 |
| `dcode mcp login NAME` | 运行 MCP 服务器的 OAuth 登录流程 |

所有管理子命令支持 `--json` 以获取机器可读输出。

## 配置

Deep Agents Code 将所有配置存储在 `~/.deepagents/` 下。每个 agent 有自己的子目录。

| 路径 | 用途 |
|------|------|
| `~/.deepagents/config.toml` | 模型和 agent 默认值、提供商设置、profile 覆盖 |
| `~/.deepagents/.env` | 全局 API key 和密钥 |
| `~/.deepagents/hooks.json` | 生命周期事件钩子 |
| `~/.deepagents/<agent_name>/` | 每个 agent 的记忆、技能和对话线程 |
| `.deepagents/`（项目根目录） | 项目特定的记忆和技能 |

```bash
# 列出所有配置的 agent
dcode agents list
```

## 非交互模式和管道

```bash
# 非交互运行
dcode -n "Write a Python script that prints hello world"

# 管道输入
echo "Explain this code" | dcode
cat error.log | dcode -n "What's causing this error?"
git diff | dcode -n "Review these changes"

# 限制轮次
dcode -n "fix the failing tests" --max-turns 10

# 限制时间
dcode -n "run the test suite" --timeout 120

# 干净输出
dcode -n "Generate a .gitignore for Python" -q > .gitignore

# Shell 允许列表
dcode -n "Run the tests" -S "pytest,git,make"
dcode -n "Build the project" -S recommended
dcode -n "Fix the build" -S all
```

> 管道输入最大 10 MiB。
> `-S all` 允许 agent 执行任意 shell 命令，无确认。谨慎使用。

## 使用 LangSmith 追踪

在 `~/.deepagents/.env` 中添加追踪键：

```bash
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=lsv2_...
LANGSMITH_PROJECT=optional-project-name
```

配置后，Deep Agents Code 显示带 LangSmith 项目链接的状态行。使用 `/trace` 打印 URL。

> 从 LangChain 应用程序中程序化调用时，使用 `DEEPAGENTS_CODE_LANGSMITH_PROJECT` 将 agent 追踪发送到专用项目。
