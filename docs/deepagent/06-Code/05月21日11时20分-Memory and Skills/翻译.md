# Memory and Skills

> Deep Agents Code 的持久记忆、AGENTS.md 文件和可复用技能，包括创建、发现和调用

有两种主要方式来自定义任何 agent：

* **[Memory](#memory)**：`AGENTS.md` 文件和自动保存的记忆，跨会话持久化。将记忆用于通用编码风格、偏好和学习到的约定。
* **[Skills](#skills)**：全局和项目特定的上下文、约定、指南或指令。将技能用于执行特定任务时适用的上下文。

使用 `/remember` 明确提示 agent 从当前对话中更新其记忆和技能。

> 使用 SDK 构建自定义 agent？参见 [Memory](/oss/python/deepagents/memory) 了解编程式记忆后端。

## Memory

### 自动记忆

当你使用 agent 时，它会自动将信息存储在 `~/.deepagents/<agent_name>/memories/` 中作为 markdown 文件，使用记忆优先协议：

1. **研究**：开始任务前搜索记忆以获取相关上下文
2. **响应**：执行过程中不确定时检查记忆
3. **学习**：自动保存新信息以供未来会话使用

Agent 按主题组织记忆，使用描述性文件名：

```
~/.deepagents/backend-dev/memories/
├── api-conventions.md
├── database-schema.md
└── deployment-process.md
```

当你教 agent 约定时：

```bash
dcode --agent backend-dev
> Our API uses snake_case and includes created_at/updated_at timestamps
```

它会在未来的会话中记住：

```bash
> Create a /users endpoint
# 无需提示即应用约定
```

### AGENTS.md 文件

[`AGENTS.md` 文件](https://agents.md/)提供在会话启动时始终加载的持久上下文：

* **全局**：`~/.deepagents/<agent_name>/AGENTS.md` — 每个会话加载。
* **项目**：任何 git 项目根目录中的 `.deepagents/AGENTS.md` — 在该项目内运行 Deep Agents Code 时加载。

两个文件在启动时都附加到系统提示中。

### 记忆工作原理

Agent 在回答项目特定问题或当你引用过去的工作或模式时，也可能读取其记忆文件。

Agent 会在你提供关于其行为方式的信息、对其工作的反馈或记住某事的指令时更新 `AGENTS.md`。如果它从你的交互中识别出模式或偏好，也会更新其记忆。

要添加更多结构化的项目知识到额外的记忆文件中，将它们添加到 `.deepagents/` 并在 `AGENTS.md` 文件中引用它们。你必须在 `AGENTS.md` 文件中引用额外文件，agent 才能知道它们。额外文件不会在启动时读取，但 agent 可以在需要时引用和更新它们。

### 何时使用全局 vs 项目 AGENTS.md

使用全局 `AGENTS.md`（`~/.deepagents/agent/AGENTS.md`）用于：

* 你的个性、风格和通用编码偏好
* 通用语气和沟通风格
* 通用编码偏好（格式、类型提示等）
* 到处适用的工具使用模式
* 不随项目变化的工作流和方法论

使用项目 `AGENTS.md`（项目根目录中的 `.deepagents/AGENTS.md`）用于：

* 项目特定的上下文和约定
* 项目架构和设计模式
* 此代码库特定的编码约定
* 测试策略和部署流程
* 团队指南和项目结构

## Skills

技能是提供专业化工作流和领域知识的可复用 agent 能力。你可以使用 [skills](/oss/python/deepagents/skills) 为你的 deep agent 提供新能力和专业知识。Deep agent 技能遵循 [Agent Skills 标准](https://agentskills.io/)。添加技能后，你的 deep agent 会自动使用它们，并在你使用 agent 并提供额外信息时更新它们。

### 添加技能

**步骤 1：创建技能**

```bash
# 用户技能（存储在 ~/.deepagents/<agent_name>/skills/）
dcode skills create test-skill

# 项目技能（存储在 .deepagents/skills/）
dcode skills create test-skill --project
```

这会生成：

```
skills/
└── test-skill
    └── SKILL.md
```

**步骤 2：编辑 SKILL.md**

打开生成的 `SKILL.md` 并编辑文件以包含你的指令。

**步骤 3：添加可选资源**

可选地向 `test-skill` 文件夹添加额外的脚本或其他资源。

你也可以直接将现有技能复制到 agent 的文件夹：

```bash
mkdir -p ~/.deepagents/<agent_name>/skills
cp -r examples/skills/web-research ~/.deepagents/<agent_name>/skills/
```

### 安装社区技能

你可以使用 Vercel 的 [Skills CLI](https://github.com/vercel-labs/skills) 等工具安装社区 [Agent Skills](https://agentskills.io/)：

```bash
# 全局安装技能
npx skills add vercel-labs/agent-skills --skill web-design-guidelines -a deepagents -g -y

# 列出已安装的技能
npx skills ls -a deepagents -g
```

全局安装（`-g`）将技能符号链接到 `~/.deepagents/agent/skills/` — 默认 agent 的用户级技能目录。项目级安装（省略 `-g`）将技能放在相对于当前目录的 `.deepagents/skills/` 中。

> 全局安装仅针对默认的 `agent` 目录。如果你使用自定义命名的 agent，使用项目级安装或手动将技能符号链接到 `~/.deepagents/{your-agent}/skills/`。

### 技能发现

启动时，Deep Agents Code 从 Deep Agents 和共享别名目录发现技能：

```
~/.deepagents/<agent_name>/skills/
~/.agents/skills/
.deepagents/skills/
.agents/skills/
~/.claude/skills/          （实验性）
.claude/skills/            （实验性）
```

当存在重复的技能名称时，后优先级目录覆盖先前的目录。

对于项目特定技能，项目的根文件夹必须有 `.git` 文件夹。当你从项目文件夹内的任何位置启动 Deep Agents Code 时，它会通过检查包含的 `.git` 文件夹来找到项目的根文件夹。

对于每个技能，Deep Agents Code 从 `SKILL.md` 文件的 frontmatter 读取名称和描述。当你使用 Deep Agents Code 时，如果任务匹配技能的描述，agent 会读取技能文件并遵循其指令。

你也可以直接用 `/skill:<name> [args]` 调用技能。技能发现在启动时运行，在 `/reload` 时再次运行。

### 从命令行调用技能

使用 `--skill` 在启动时调用技能，无需交互输入斜杠命令：

```bash
# 打开 TUI 并立即运行技能
dcode --skill code-review

# 用 -m 传递请求
dcode --skill code-review -m 'review the auth module'

# 管道内容到技能
cat diff.txt | dcode --skill code-review

# 管道内容并添加请求
cat diff.txt | dcode --skill code-review -m 'focus on security'
```

`--skill` 也适用于非交互模式：

```bash
# 无头运行技能
dcode --skill code-review -n 'review this patch'

# 安静模式
dcode --skill code-review -n 'review this patch' -q
```

> `--skill` 与 `--quiet` 或 `--no-stream` 需要 `-n`（非交互模式）。

### 列出技能

```bash
# 列出所有用户技能
dcode skills list

# 列出项目技能
dcode skills list --project

# 获取特定技能的详细信息
dcode skills info test-skill
dcode skills info test-skill --project
```
