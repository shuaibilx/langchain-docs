# Data Locations - 总结

## 核心概念

Data Locations 是 Deep Agents Code 的**数据存储位置参考**——定义了配置、会话、skills、subagents、指令等所有数据在文件系统中的存放位置和优先级规则。

## 两个目录层次

```
~/.deepagents/    ← Deep Agents 特定数据
~/.agents/        ← 工具无关数据（跨 AI CLI 共享）
```

## 完整目录结构

```
~/.deepagents/
├── .state/                      # 自动管理的状态
│   ├── sessions.db              #   会话检查点（SQLite）
│   ├── history.jsonl            #   输入历史
│   └── auth.json                #   存储的 API 密钥
└── {agent}/                     # 每个 agent（默认 "agent"）
    ├── AGENTS.md                #   用户指令自定义
    ├── skills/                  #   用户级 skills
    │   └── {skill-name}/SKILL.md
    └── agents/                  #   自定义 subagents
        └── {name}/AGENTS.md

~/.agents/
└── skills/                      # 工具无关 skills
    └── {skill-name}/SKILL.md

{project}/
├── AGENTS.md                    # 项目指令（根目录）
└── .deepagents/
│   ├── AGENTS.md                # 项目指令（首选位置）
│   ├── skills/                  # 项目 skills
│   └── agents/                  # 项目 subagents
└── .agents/
    └── skills/                  # 工具无关项目 skills
```

## 数据存放一览

| 数据 | 位置 | 读/写 |
|------|------|-------|
| 会话 | `~/.deepagents/.state/sessions.db` | R/W |
| 输入历史 | `~/.deepagents/.state/history.jsonl` | R/W |
| 基础指令 | 包内 `default_agent_prompt.md` | R |
| 用户自定义 | `~/.deepagents/{agent}/AGENTS.md` | R/W |
| 项目指令 | `.deepagents/AGENTS.md` 或 `AGENTS.md` | R |
| 用户 skills | `~/.deepagents/{agent}/skills/` | R/W |
| 共享 skills | `~/.agents/skills/` | R |
| 项目 skills | `.deepagents/skills/` 或 `.agents/skills/` | R |
| 自定义 subagents | `~/.deepagents/{agent}/agents/` | R/W |
| 项目 subagents | `.deepagents/agents/` | R |

## 优先级规则

### Skills 优先级（从低到高）

```
1. ~/.deepagents/{agent}/skills/     ← 用户 Deep Agents Code
2. ~/.agents/skills/                 ← 用户工具无关
3. .deepagents/skills/               ← 项目 Deep Agents Code
4. .agents/skills/                   ← 项目工具无关（最高）
```

### Subagents 优先级（从低到高）

```
1. ~/.deepagents/{agent}/agents/     ← 用户级
2. .deepagents/agents/               ← 项目级（最高）
```

### 指令优先级（组合，不覆盖）

```
1. 包基础提示         ← 始终加载
2. ~/.deepagents/{agent}/AGENTS.md   ← 追加
3. .deepagents/AGENTS.md             ← 追加
4. 项目根 AGENTS.md                  ← 追加
```

## `.deepagents` vs `.agents`

| 目录 | 用途 | 示例 |
|------|------|------|
| `.deepagents/` | Deep Agents 特定 | 使用 dcode 特定功能的 skills |
| `.agents/` | 工具无关 | 适用于任何 AI 编码助手的 skills |

## 清理命令

| 需求 | 命令 |
|------|------|
| 重置所有数据 | `rm -rf ~/.deepagents` |
| 清除会话 | `rm ~/.deepagents/.state/sessions.db*` |
| 清除历史 | `rm ~/.deepagents/.state/history.jsonl` |
| 清除 API 密钥 | `rm ~/.deepagents/.state/auth.json` |
| 清除 MCP 令牌 | `rm -rf ~/.deepagents/.state/mcp-tokens` |
| 重新引导 | `rm ~/.deepagents/.state/onboarding_complete` |
| 重置指令 | `dcode agents reset --agent {name}` |
| 移除 skill | `rm -rf ~/.deepagents/{agent}/skills/{skill-name}` |

## 关键注意事项

1. **两个目录层次** — `~/.deepagents/`（特定）和 `~/.agents/`（通用）
2. **Skills 四级优先级** — 项目工具无关 > 项目特定 > 用户工具无关 > 用户特定
3. **Subagents 两级优先级** — 项目 > 用户
4. **指令是组合的** — 四个来源全部追加，不覆盖
5. **符号链接验证** — skill 路径必须在允许的目录内
6. **`.state/` 自动管理** — sessions.db、history.jsonl、auth.json
7. **删除 sessions.db 不可逆** — 除非有备份
