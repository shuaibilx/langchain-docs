# Data Locations

> Deep Agents Code 存储配置、会话和自定义文件的位置

Deep Agents Code 将数据存储在两个目录层次结构中：

- **`~/.deepagents/`** — Deep Agents 特定数据（agent 记忆、skills、会话）
- **`~/.agents/`** — 工具无关数据（跨 AI CLI 工具共享的 skills）

## 目录结构

```
~/.deepagents/
├── .state/                  # 每台机器的 Deep Agents Code 状态（自动管理）
│   ├── sessions.db          #   会话检查点的 SQLite 数据库
│   ├── history.jsonl        #   命令输入历史
│   ├── ...                  #   其他标记和凭证
└── {agent}/                 # 每个 agent 的目录（默认："agent"）
    ├── AGENTS.md            #   用户对 agent 指令的自定义
    ├── skills/              #   用户级 skills
    │   └── {skill-name}/
    │       └── SKILL.md
    └── agents/              #   自定义 subagent 定义
        └── {subagent-name}/
            └── AGENTS.md

~/.agents/                   # 工具无关别名（跨 AI CLI 共享）
└── skills/                  #   对任何兼容工具可用的 skills
    └── {skill-name}/
        └── SKILL.md

{project}/                   # 项目级（在 git 仓库根目录）
├── AGENTS.md                #   项目指令（根目录级别）
└── .deepagents/
│   ├── AGENTS.md            #   项目指令（首选位置）
│   ├── skills/              #   项目特定 skills
│   │   └── {skill-name}/
│   │       └── SKILL.md
│   └── agents/              #   项目特定 subagents
│       └── {subagent-name}/
│           └── AGENTS.md
└── .agents/                 #   工具无关项目 skills
    └── skills/
        └── {skill-name}/
            └── SKILL.md
```

## 各数据存放位置

| 数据 | 位置 | 读/写 | 说明 |
|------|------|-------|------|
| **会话** | `~/.deepagents/.state/sessions.db` | R/W | SQLite 检查点数据库 |
| **输入历史** | `~/.deepagents/.state/history.jsonl` | R/W | JSON-lines，上下箭头回溯 |
| **基础指令** | 包内 `default_agent_prompt.md` | R | 不可变，随 Deep Agents Code 升级更新 |
| **用户自定义** | `~/.deepagents/{agent}/AGENTS.md` | R/W | 追加到基础指令 |
| **项目指令** | `.deepagents/AGENTS.md` 或 `AGENTS.md` | R | 两者都加载（如果存在） |
| **用户 skills** | `~/.deepagents/{agent}/skills/` | R/W | Agent 特定 skills |
| **共享 skills** | `~/.agents/skills/` | R | 工具无关，跨 CLI |
| **项目 skills** | `.deepagents/skills/` 或 `.agents/skills/` | R | 项目范围 |
| **自定义 subagents** | `~/.deepagents/{agent}/agents/` | R/W | 用户定义的 subagents |
| **项目 subagents** | `.deepagents/agents/` | R | 项目定义的 subagents |

## 优先级规则

当同一项目存在于多个位置时，**更高优先级完全胜出**（不合并）。

### Skills

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/skills/` — 用户 Deep Agents Code
2. `~/.agents/skills/` — 用户工具无关
3. `.deepagents/skills/` — 项目 Deep Agents Code
4. `.agents/skills/` — 项目工具无关（*最高*）

当加载 skill 时，Deep Agents Code 验证解析的文件路径是否保持在这些目录之一内。解析到所有 skill 根目录之外的符号链接被拒绝。要允许符号链接目标在额外目录中，请参阅 [`[skills].extra_allowed_dirs`](#skills-extra-allowed-directories)。

### Subagents

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/agents/` — 用户级
2. `.deepagents/agents/` — 项目级（*最高*）

每个 subagent 是一个带 YAML frontmatter（`name`、`description`、可选 `model`）和 markdown 正文作为系统提示的 `AGENTS.md` 文件。

### 指令

所有指令来源是**组合**的（不覆盖）：

1. 包基础提示（*始终加载*）
2. `~/.deepagents/{agent}/AGENTS.md`（*追加*）
3. `.deepagents/AGENTS.md`（*追加*）
4. 项目根目录的 `AGENTS.md`（*追加*）

## `.deepagents` vs `.agents`

| 目录 | 用途 | 何时使用 |
|------|------|---------|
| `.deepagents/` | Deep Agents Code 特定 | 使用 Deep Agents Code 特定功能的 skills 和配置 |
| `.agents/` | 工具无关 | 想要在不同 AI CLI 工具之间共享的 skills |

> **提示：** 对于适用于任何 AI 编码助手的 skills，使用 `.agents/skills/`。对于依赖 Deep Agents 特定工具或约定的 skills，使用 `.deepagents/skills/`。

## 清理

| 需求 | 操作 |
|------|------|
| 重置所有数据 | `rm -rf ~/.deepagents` |
| 仅清除会话 | `rm ~/.deepagents/.state/sessions.db*` |
| 清除输入历史 | `rm ~/.deepagents/.state/history.jsonl` |
| 清除存储的 API 密钥 | `rm ~/.deepagents/.state/auth.json` |
| 清除 MCP OAuth 令牌 | `rm -rf ~/.deepagents/.state/mcp-tokens` |
| 重新运行首次引导 | `rm ~/.deepagents/.state/onboarding_complete` |
| 重置 agent 指令 | `dcode agents reset --agent {name}` |
| 移除 skill | `rm -rf ~/.deepagents/{agent}/skills/{skill-name}` |

> **警告：** 删除 `~/.deepagents/.state/sessions.db` 将移除所有对话历史和检查点。除非你有 `sessions.db` 文件的备份，否则无法撤销。
