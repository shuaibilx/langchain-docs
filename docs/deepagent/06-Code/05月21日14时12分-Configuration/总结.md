# Configuration - 总结

## 核心概念

Configuration 是 Deep Agents Code 的**全面配置系统**——通过 `config.toml`、`.env`、`hooks.json` 和 `.mcp.json` 四个文件控制模型提供商、凭证、主题、自动更新、hooks 等所有行为。

## 四个配置文件

| 文件 | 格式 | 用途 |
|------|------|------|
| `~/.deepagents/config.toml` | TOML | 模型默认值、提供商设置、构造参数、配置文件覆盖、主题、更新 |
| `~/.deepagents/.env` | Dotenv | 全局 API 密钥 |
| `~/.deepagents/hooks.json` | JSON | 生命周期事件 hooks |
| `~/.deepagents/.mcp.json` | JSON | MCP 服务器定义 |

## 凭证系统

### 两个凭证来源（优先级从高到低）

```
1. 存储的密钥（/auth 管理）     ← 最高优先级
2. 环境变量                      ← OPENAI_API_KEY 等
   ├── shell export
   ├── 项目 .env
   └── 全局 ~/.deepagents/.env
```

### `/auth` 管理器

```
/auth  →  列出所有提供商
         ✓ credentials set        → 已有密钥
         ! missing OPENAI_API_KEY → 需要粘贴
         local provider           → 无需凭证
         implicit auth            → 隐式认证
         custom auth              → 自定义认证
```

### 环境变量加载顺序

```
shell 环境 > 项目 .env > 全局 ~/.deepagents/.env
（shell 中已设置的值永不被覆盖）
```

### `DEEPAGENTS_CODE_` 前缀

```
DEEPAGENTS_CODE_OPENAI_API_KEY  →  仅在 dcode 内优先于 OPENAI_API_KEY
DEEPAGENTS_CODE_ANTHROPIC_API_KEY=  →  阻止 dcode 中的 shell 导出密钥
```

## config.toml 结构

### 模型默认值

```toml
[models]
default = "ollama:qwen3:4b"             # 长期偏好（Ctrl+S 设置）
recent = "google_genai:gemini-3.5-flash"   # 最近切换（自动写入）
```

### Agent 默认值

```toml
[agents]
default = "backend-dev"   # 长期偏好
recent = "frontend-dev"   # 最近切换
```

### 提供商配置

```toml
[models.providers.<name>]
models = ["gpt-4o"]           # 切换器中显示的模型
api_key_env = "OPENAI_API_KEY" # 凭证环境变量名
base_url = "..."               # 自定义端点
class_path = "mod:Cls"         # 任意提供商
enabled = true                 # 是否在切换器中显示

[models.providers.<name>.params]        # 提供商级别参数
temperature = 0

[models.providers.<name>.params."gpt-4o"]  # 每模型覆盖
temperature = 0.7

[models.providers.<name>.profile]       # 配置文件覆盖
max_input_tokens = 4096

[models.providers.<name>.profile."claude-sonnet-4-5"]  # 每模型配置文件覆盖
max_input_tokens = 8192
```

### 三层参数优先级

```
CLI --model-params          ← 最高（仅会话）
config.toml 每模型子表      ← 中
config.toml 提供商级别      ← 最低
```

### 配置文件覆盖优先级

```
CLI --profile-override      ← 最高
config.toml 每模型子表      ← 中
config.toml 提供商级别      ← 最低
模型默认值                   ← 最低
```

## 任意提供商

```toml
[models.providers.my_custom]
class_path = "my_package.models:MyChatModel"
api_key_env = "MY_API_KEY"
base_url = "https://my-endpoint.example.com"
```

- 使用 `class_path` 直接导入 Python 类
- 类必须是 `BaseChatModel` 子类
- 包必须与 `deepagents-code` 同环境

## 兼容 API

```toml
# OpenAI 兼容
[models.providers.openai]
base_url = "https://api.example.com/v1"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"]

# Anthropic 兼容
[models.providers.anthropic]
base_url = "https://api.example.com"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"
```

> **注意：** OpenAI 兼容 API 可能需要禁用 Responses API：`use_responses_api = false`

## 主题系统

```toml
[ui]
theme = "langchain-dark"

# 自定义主题
[themes.my-solarized]
label = "My Solarized"
dark = true
primary = "#268BD2"

# 终端映射
[ui.terminal_themes]
"Apple_Terminal" = "langchain-light"
"iTerm.app" = "langchain"
```

主题解析顺序：`DEEPAGENTS_CODE_THEME` > `terminal_themes` > `[ui] theme` > 默认

## 自动更新

```toml
[update]
auto_update = true
```

或 `DEEPAGENTS_CODE_AUTO_UPDATE=1`

## Hooks 系统

```json
{
  "hooks": [
    {
      "command": ["python3", "handler.py"],
      "events": ["session.start", "task.complete"]
    }
  ]
}
```

### 8 个事件

| 事件 | 触发时机 |
|------|---------|
| `session.start` | 会话开始 |
| `session.end` | 会话结束 |
| `user.prompt` | 用户提交消息 |
| `input.required` | agent 需要人工输入 |
| `permission.request` | 工具需要许可 |
| `tool.error` | 工具调用出错 |
| `task.complete` | 任务完成 |
| `context.compact` | 上下文压缩前 |

### 执行模型

- 后台线程，fire-and-forget
- 并发分发，5 秒超时
- 不阻塞主循环，失败只记录

## 环境变量参考

| 变量 | 说明 |
|------|------|
| `DEEPAGENTS_CODE_AUTO_UPDATE` | 启用自动更新 |
| `DEEPAGENTS_CODE_DEBUG` | 启用调试日志 |
| `DEEPAGENTS_CODE_DEBUG_FILE` | 调试日志路径 |
| `DEEPAGENTS_CODE_EXTRA_SKILLS_DIRS` | skill 允许列表路径 |
| `DEEPAGENTS_CODE_LANGSMITH_PROJECT` | LangSmith 项目名 |
| `DEEPAGENTS_CODE_NO_UPDATE_CHECK` | 禁用更新检查 |
| `DEEPAGENTS_CODE_SHELL_ALLOW_LIST` | shell 命令白名单 |
| `DEEPAGENTS_CODE_USER_ID` | 用户标识符 |
| `DEEPAGENTS_CODE_THEME` | 主题覆盖 |
| `DEEPAGENTS_CODE_OLLAMA_DISCOVERY` | Ollama 自动发现 |

## 关键注意事项

1. **四个配置文件** — config.toml、.env、hooks.json、.mcp.json
2. **凭证优先级** — 存储密钥 > 环境变量（shell > 项目 .env > 全局 .env）
3. **`DEEPAGENTS_CODE_` 前缀** — 隔离 dcode 专用凭证
4. **`/auth` 只管 LLM 凭证** — 工具凭证（Tavily 等）从环境读取
5. **参数三层优先级** — CLI > 每模型 > 提供商级别
6. **配置文件覆盖** — 控制 `max_input_tokens` 等运行时能力
7. **任意提供商** — `class_path` 导入任意 `BaseChatModel` 子类
8. **Hooks fire-and-forget** — 后台线程，5 秒超时，不阻塞
9. **主题解析** — 环境变量 > 终端映射 > 保存偏好 > 默认
10. **兼容 API** — 可能需要 `use_responses_api = false`
