# MCP Tools - 总结

## 核心概念

MCP Tools 是 Deep Agents Code 的**外部工具扩展系统**——通过 MCP（Model Context Protocol）协议连接外部服务器，将文件系统、API、数据库等工具加载到 agent 中，无需修改 agent 本身。

## 配置文件位置（优先级从低到高）

| 优先级 | 位置 | 作用域 |
|--------|------|--------|
| 1（最低） | `~/.deepagents/.mcp.json` | 用户级——所有项目 |
| 2 | `<project>/.deepagents/.mcp.json` | 项目级——隐藏子目录 |
| 3（最高） | `<project>/.mcp.json` | 项目级——根目录（Claude Code 兼容） |

多个配置文件的 `mcpServers` 条目会合并；同名服务器高优先级胜出。

## 配置格式

```json
{
  "mcpServers": {
    "server-name": {
      "type": "http",
      "url": "https://example.com/mcp",
      "headers": { "Authorization": "Bearer ${TOKEN}" },
      "auth": "oauth",
      "allowedTools": ["tool1", "tool2"],
      "disabledTools": ["dangerous_tool"]
    }
  }
}
```

## 三种传输类型

| 类型 | 用途 | 必需字段 | 可选字段 |
|------|------|---------|---------|
| **stdio**（默认） | 子进程 | `command` | `args`、`env` |
| **sse** | 远程 SSE | `type: "sse"`、`url` | `headers`、`auth` |
| **http** | 远程 HTTP | `type: "http"`、`url` | `headers`、`auth` |

```json
// stdio 示例
{
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
    "env": { "GITHUB_TOKEN": "token" }
  }
}

// HTTP 示例
{
  "remote-api": {
    "type": "http",
    "url": "https://api.example.com/mcp",
    "headers": { "Authorization": "Bearer ${TOKEN}" }
  }
}
```

## 工具过滤

| 字段 | 作用 | 互斥 |
|------|------|------|
| `allowedTools` | 仅保留列出的工具 | 与 `disabledTools` |
| `disabledTools` | 丢弃列出的工具 | 与 `allowedTools` |

- 支持字面名称和 `fnmatch` glob 模式
- 同时匹配裸名称和 `{server}_{tool}` 前缀形式
- 不匹配的条目记录为警告，不报错

```json
{
  "allowedTools": ["read_file", "fs_list_*"],
  "disabledTools": ["delete_repository", "delete_*_branch"]
}
```

## OAuth 登录

```json
{
  "linear": {
    "type": "http",
    "url": "https://mcp.linear.app/mcp",
    "auth": "oauth"
  }
}
```

```bash
# 运行登录流程
dcode mcp login linear

# 使用特定配置文件
dcode mcp login linear --config ./mcp-config.json
```

三种 OAuth 流程：
- **符合规范的服务器**：动态客户端注册 + 授权码 PKCE
- **Slack**：预置公共客户端 + 可选团队 ID
- **GitHub**：RFC 8628 设备授权

## 服务器状态

| 状态 | 含义 | 处理 |
|------|------|------|
| `ok` | 已连接，工具可用 | — |
| `unauthenticated` | 需要登录或刷新失败 | `dcode mcp login <server>` |
| `error` | 预检/发现/传输失败 | 查看 `/mcp` 详情 |

单个失败服务器不中止启动。

## 项目级信任

```
默认拒绝策略：
├── 交互模式 → 提示批准（SHA-256 指纹持久化）
├── 非交互模式 → 静默跳过（除非 --trust-project-mcp）
├── 用户级配置 → 始终信任
└── dcode mcp login → 遵守项目信任
```

信任存储在 `~/.deepagents/config.toml`：

```toml
[mcp_trust.projects]
"/Users/you/myproject" = "sha256:abc123..."
```

## 标志

| 标志 | 说明 |
|------|------|
| `--mcp-config PATH` | 添加显式配置（最高优先级） |
| `--no-mcp` | 完全禁用 MCP |
| `--trust-project-mcp` | 不提示即信任项目服务器 |

## Header 环境变量

```json
{
  "headers": { "Authorization": "Bearer ${INTERNAL_API_TOKEN}" }
}
```

- `${VAR}` 在服务器激活时解析（非配置加载时）
- 未设置的变量只使需要它的服务器失败

## 关键注意事项

1. **三个发现位置** — 用户级 < 项目隐藏 < 项目根
2. **Claude Code 兼容** — 根目录 `.mcp.json` 自动识别
3. **三种传输** — stdio（子进程）、sse（远程 SSE）、http（远程 HTTP）
4. **工具过滤** — `allowedTools` / `disabledTools` 互斥
5. **OAuth 令牌** — 持久化到 `~/.deepagents/.state/mcp-tokens/`
6. **项目级信任** — 默认拒绝，需显式批准
7. **`${VAR}` 插值** — 激活时解析，未设置只影响单个服务器
8. **服务器名** — 必须匹配 `[A-Za-z0-9_-]+`
9. **失败不中止** — 单个服务器失败不影响其他
10. **`/mcp` 命令** — 查看服务器状态和工具列表
