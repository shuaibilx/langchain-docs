# Model Providers

> 为 Deep Agents Code 配置任何 LangChain 兼容的模型提供商

Deep Agents Code 支持任何与 [LangChain 兼容的聊天模型提供商](/oss/python/integrations/chat)，几乎可以使用任何支持工具调用的 LLM。任何暴露 OpenAI 兼容或 Anthropic 兼容 API 的服务也可以开箱即用——参见[兼容 API](/oss/python/deepagents/code/configuration#compatible-apis)。

## 快速开始

Deep Agents Code 与以下[模型提供商](#provider-reference)自动集成：除了安装相关的提供商包外，无需额外配置。

1. **安装提供商包**
   每个模型提供商需要安装其对应的 LangChain 集成包。这些包在安装 Deep Agents Code 时作为可选额外组件提供，这样做是为了保持应用程序的轻量级：
   
   ```bash
   # 快速安装选定的提供商
   # OpenAI、Anthropic 和 Gemini 默认包含
   DEEPAGENTS_EXTRAS="baseten,groq" curl -LsSf https://langch.in/dcode | bash
   
   # 或使用 uv 直接安装
   uv tool install 'deepagents-code[baseten,groq]'
   
   # 稍后添加额外的包
   uv tool install deepagents-code --with langchain-ollama
   
   # 所有提供商
   uv tool install 'deepagents-code[anthropic,baseten,bedrock,cohere,deepseek,fireworks,google-genai,groq,huggingface,ibm,litellm,mistralai,nvidia,ollama,openai,openrouter,perplexity,vertexai,xai]'
   ```

2. **设置凭证**
   将 API 密钥存储在 `~/.deepagents/.env` 中，以便在所有项目中可用，或在 shell 中导出它们：
   **OpenAI：**
   
   ```bash
   # 永久添加
   mkdir -p ~/.deepagents
   echo 'OPENAI_API_KEY=your-api-key' >> ~/.deepagents/.env
   
   # 为当前会话添加
   export OPENAI_API_KEY="your-api-key"
   ```
   
   **Anthropic：**
   
   ```bash
   # 永久添加
   mkdir -p ~/.deepagents
   echo 'ANTHROPIC_API_KEY=your-api-key' >> ~/.deepagents/.env
   
   # 为当前会话添加
   export ANTHROPIC_API_KEY="your-api-key"
   ```
   
   **Google：**
   
   ```bash
   # 永久添加
   mkdir -p ~/.deepagents
   echo 'GOOGLE_API_KEY=your-api-key' >> ~/.deepagents/.env
   
   # 为当前会话添加
   export GOOGLE_API_KEY="your-api-key"
   ```
   
   **其他：** Deep Agents Code 支持任何支持工具调用的 LLM。有关支持的提供商及其所需环境变量的完整列表，请参阅[提供商参考](#provider-reference)。
   要配置模型参数，请参阅[模型参数](#model-parameters)。
   你也可以使用 [`DEEPAGENTS_CODE_` 前缀](#deepagents_code_-prefix)将凭证限定到 Deep Agents Code。

### 模型路由器和代理

[OpenRouter](https://openrouter.ai/) 和 [LiteLLM](https://docs.litellm.ai/) 等模型路由器通过单个端点提供对多个提供商模型的访问。

使用这些服务的专用集成包：

| 路由器        | 包                      | 配置                       |
| ---------- | ---------------------- | ------------------------ |
| OpenRouter | `langchain-openrouter` | `openrouter:<model>`（内置） |
| LiteLLM    | `langchain-litellm`    | `litellm:<model>`（内置）    |

**OpenRouter** 是内置提供商——安装包即可直接使用：

```bash
uv tool install 'deepagents-code[openrouter]'
```

**LiteLLM** 也是内置提供商：

```bash
uv tool install 'deepagents-code[litellm]'
```

## 切换模型

要在 Deep Agents Code 中切换模型，可以：

1. **使用交互式模型切换器**，通过 `/model` 命令。这会显示从每个已安装 LangChain 提供商包的[模型配置文件](/oss/python/langchain/models#model-profiles)中获取的可用模型。
   
   > **注意：** 并非所有模型都会显示在这里。如果你的模型缺失，请直接传递模型名称（例如 `/model gpt-5.5`）。详见[哪些模型出现在切换器中](#which-models-appear-in-the-switcher)。

2. **直接指定模型名称**作为参数，例如 `/model gpt-5.5`。你可以使用所选提供商支持的任何模型，无论它是否出现在选项 1 的列表中。模型名称将被传递给 API 请求。

3. **在启动时指定模型**，通过 `--model`，例如：
   
   ```bash
   dcode --model openai:gpt-5.5
   ```

### 模型解析顺序

当 Deep Agents Code 启动时，它按以下顺序解析使用哪个模型：

1. **`--model` 标志**——提供时始终优先。
2. **`[models].default`**——在 `~/.deepagents/config.toml` 中，用户的长期偏好。
3. **`[models].recent`**——在 `~/.deepagents/config.toml` 中，通过 `/model` 切换的最后一个模型。自动写入；永不覆盖 `[models].default`。
4. **环境自动检测**——回退到第一个可用的启动凭证，按以下顺序检查：`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`、`GOOGLE_CLOUD_PROJECT`（Vertex AI）。

此启动回退故意只检查这四个凭证。其他支持的提供商（例如 Groq）仍然可以通过 `--model`、`/model` 和保存的默认值（`[models].default` / `[models].recent`）使用。

### 哪些模型出现在切换器中

`/model` 选择器从已安装的提供商包中动态构建其列表。模型出现在切换器中需要**同时满足**以下条件：

1. **提供商包已安装。** 每个提供商（例如 `langchain-anthropic`、`langchain-openai`）必须与 `deepagents-code` 一起安装——可以作为[安装额外组件](#quickstart)（例如 `uv tool install 'deepagents-code[ollama]'`）或稍后通过 `uv tool install deepagents-code --with <package>` 添加。如果包缺失，其整个提供商部分将不在切换器中显示。

2. **模型的配置文件启用了 `tool_calling`。** Deep Agents Code 需要工具调用支持，因此配置文件中没有 `tool_calling: true` 的模型会被排除。这是模型缺失的最常见原因。对于不附带配置文件的提供商（参见[提供商参考](#provider-reference)表），你可以在 `config.toml` 中定义一个：
   
   ```toml
   [models.providers.ollama.profile."qwen3:4b"]
   tool_calling = true
   max_input_tokens = 32768
   max_output_tokens = 8192
   ```
   
   这对于模型出现在切换器中并非严格要求——将其添加到 [`models` 列表](#adding-models-to-the-interactive-switcher)也可以，而且更简单。配置文件在你希望 Deep Agents Code 了解模型的上下文窗口和能力（如自动摘要功能）时很有用。参见[配置文件覆盖](#profile-overrides-advanced)了解所有可覆盖字段。

3. **模型接受并产生文本。** 配置文件明确将 `text_inputs` 或 `text_outputs` 设置为 `false` 的模型（例如嵌入或图像生成模型）会被排除。

在 `config.toml` 的 `[models.providers.<name>].models` 下定义的模型绕过配置文件过滤——无论配置文件元数据如何，它们始终出现在切换器中。这是添加缺失模型的推荐方式。

> **提示：** 凭证状态**不会**影响模型是否被列出。切换器显示所有符合条件的模型，并在每个提供商标题旁显示凭证指示器：确认凭证为勾选，缺失凭证为警告，凭证状态未知为问号。你仍然可以选择凭证缺失的模型——提供商将在请求时报告认证错误。

#### 缺失模型故障排除

| 症状                            | 可能原因                                | 修复方法                                                                                         |
| ----------------------------- | ----------------------------------- | -------------------------------------------------------------------------------------------- |
| 整个提供商在切换器中缺失                  | 提供商包未安装                             | 安装包（例如 `uv tool install deepagents-code --with langchain-groq`）                              |
| 提供商显示但特定模型缺失                  | 模型配置文件 `tool_calling: false` 或无配置文件 | 在 `config.toml` 的 `[models.providers.<name>].models` 中添加模型，或直接使用 `/model <provider>:<model>` |
| 提供商显示 ⚠ "missing credentials" | API 密钥环境变量未设置                       | 从[提供商参考](#provider-reference)表中设置凭证环境变量                                                      |
| 提供商显示 ? "credentials unknown" | 提供商使用非标准认证，Deep Agents Code 无法验证    | 凭证可能仍然有效——尝试切换到该模型。如果认证失败，请查看提供商文档                                                           |

### 设置默认模型

你可以设置一个持久的默认模型，用于所有未来的 CLI 启动：

- **通过模型选择器：** 打开 `/model`，导航到所需模型，按 `Ctrl+S` 将其固定为默认。再次按 `Ctrl+S` 清除当前默认。

- **通过命令：** `/model --default provider:model`（例如 `/model --default anthropic:claude-opus-4-7`）

- **通过配置文件：** 在 `~/.deepagents/config.toml` 中设置 `[models].default`（参见[配置](#configuration)）。

- **从 shell：**
  
  ```bash
  dcode --default-model anthropic:claude-opus-4-7
  ```

查看当前默认：

```bash
dcode --default-model
```

清除默认：

```bash
dcode --clear-default-model
```

- **通过命令：** `/model --default --clear`
- **通过模型选择器：** 在当前固定的默认模型上按 `Ctrl+S`。

没有默认值时，Deep Agents Code 将默认使用最近使用的模型。

### 模型参数

向模型传递额外的构造 kwargs——采样控制、推理/思考预算、上下文窗口大小、请求超时，以及底层聊天模型类接受的任何其他参数。有三个设置位置，优先级从高到低：

1. **启动时一次性设置，使用 `--model-params`。** JSON 字符串，仅限会话：
   
   ```bash
   # OpenAI 推理努力程度
   dcode --model openai:gpt-5.5 --model-params '{"reasoning": {"effort": "high"}}'
   
   # Anthropic 扩展思考
   dcode --model anthropic:claude-opus-4-7 --model-params '{"thinking": {"type": "enabled", "budget_tokens": 10000}, "max_tokens": 16000}'
   ```

2. **会话中通过 `/model --model-params`。** 相同的 JSON 语法——无需重启即可交换参数（以及可选的模型）：
   
   ```
   /model --model-params '{"temperature": 0.7}' anthropic:claude-opus-4-7
   /model --model-params '{"num_ctx": 16384}'           # 打开选择器，对选择应用参数
   ```

3. **在 `config.toml` 中持久化。** 提供商级别的默认值（带有可选的每模型子表），每次启动时应用：
   
   ```toml
   [models.providers.anthropic.params]
   thinking = { type = "enabled", budget_tokens = 10000 }
   max_tokens = 16000
   
   [models.providers.openai.params]
   reasoning = { effort = "high", summary = "auto" }
   output_version = "responses/v1"
   
   [models.providers.ollama.params]
   num_ctx = 16384
   temperature = 0
   
   # 每模型覆盖——优先于提供商级别的键
   [models.providers.ollama.params."qwen3:4b"]
   temperature = 0.5
   ```

CLI 标志覆盖配置文件的 `params`，且仅限会话（会话中的更改不会被持久化）。`config.toml` 中的每模型子表覆盖提供商级别的键（浅合并——参见[模型构造参数](#model-constructor-params)了解完整语义）。`--model-params` 不能与 `--default` 组合使用。

> **提示：** 底层聊天模型构造函数接受的任何 kwargs 都是有效的。请参阅提供商的参考文档获取完整列表——例如 [`ChatAnthropic`](https://reference.langchain.com/python/langchain-anthropic/langchain_anthropic/chat_models/ChatAnthropic)、[`ChatOpenAI`](https://reference.langchain.com/python/langchain-openai/langchain_openai/chat_models/base/ChatOpenAI)、[`ChatOllama`](https://reference.langchain.com/python/langchain-ollama/langchain_ollama/chat_models/ChatOllama)。未知的 kwargs 会被转发到上游 API 请求，因此新发布的参数无需 CLI 更新即可工作。

> **注意：** 不要在 `params` 中放置凭证（`api_key`）——使用 [`api_key_env`](#provider-configuration) 指向环境变量。

要覆盖模型运行时*配置文件*的字段（`max_input_tokens`、`tool_calling`、能力标志）——与构造参数不同——请参阅[配置文件覆盖](#profile-overrides-advanced)。
