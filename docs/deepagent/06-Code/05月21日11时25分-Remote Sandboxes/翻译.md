# 使用远程 Sandbox

> 在 LangSmith、Daytona、Modal、Runloop 或 AgentCore sandbox 中运行 Deep Agents Code 工具执行。安装提供商额外组件、设置凭证，使用标志和设置脚本。

Deep Agents Code 使用 [sandbox as tool](/oss/python/deepagents/sandboxes#sandbox-as-tool-pattern) 模式：`dcode` 进程（LLM 循环、记忆、工具分发）在你的机器上运行，但 agent 工具调用（`read_file`、`write_file`、`execute` 等）指向远程 sandbox，而不是你的本地文件系统。要将文件放入 sandbox，使用[设置脚本](#设置脚本)或提供商的文件传输 API。

关于 sandbox 架构、集成模式和安全最佳实践的深入了解，参见 [Sandboxes](/oss/python/deepagents/sandboxes)。

## 步骤 1：安装提供商依赖

### LangSmith

安装 `deepagents-code` 时默认包含。无需额外安装。

### Daytona

```bash
uv tool install deepagents-code --with langchain-daytona
```

### Modal

```bash
uv tool install deepagents-code --with langchain-modal
```

### Runloop

```bash
uv tool install deepagents-code --with langchain-runloop
```

### AgentCore

```bash
uv tool install deepagents-code --with langchain-agentcore-codeinterpreter
```

## 步骤 2：设置提供商凭证

### LangSmith

```bash
export LANGSMITH_API_KEY="your-key"
```

### Daytona

```bash
export DAYTONA_API_KEY="your-key"
```

### Modal

```bash
modal setup
```

### Runloop

```bash
export RUNLOOP_API_KEY="your-key"
```

### AgentCore

```bash
export AWS_ACCESS_KEY_ID="your-key"
export AWS_SECRET_ACCESS_KEY="your-secret"
export AWS_SESSION_TOKEN="session-token"
export AWS_REGION="us-west-2"
```

## 步骤 3：使用 sandbox 运行 Deep Agents Code

```bash
# LangSmith
dcode --sandbox langsmith

# Daytona
dcode --sandbox daytona

# Modal
dcode --sandbox modal

# Runloop
dcode --sandbox runloop

# AgentCore
dcode --sandbox agentcore
```

## Sandbox 标志和示例

| 标志 | 描述 |
|------|------|
| `--sandbox TYPE` | Sandbox 提供商：`langsmith`、`agentcore`、`modal`、`daytona` 或 `runloop`（默认：`none`） |
| `--sandbox-id ID` | 通过 ID 重用现有 sandbox，跳过创建和清理 |
| `--sandbox-setup PATH` | 创建后在 sandbox 内运行的设置脚本路径 |

示例：

```bash
# 创建新的 Daytona sandbox
dcode --sandbox daytona

# 重用现有 sandbox（跳过创建和清理）
dcode --sandbox runloop --sandbox-id dbx_abc123

# 创建后运行设置脚本
dcode --sandbox modal --sandbox-setup ./setup.sh
```

## 设置脚本

使用 `--sandbox-setup` 在创建后在 sandbox 内运行 shell 脚本。这对于克隆仓库、安装依赖和配置环境变量非常有用。

```bash
#!/bin/bash
set -e

# 使用 GitHub token 克隆仓库
git clone https://x-access-token:${GITHUB_TOKEN}@github.com/username/repo.git $HOME/workspace
cd $HOME/workspace

# 使环境变量持久化
cat >> ~/.bashrc <<'EOF'
export GITHUB_TOKEN="${GITHUB_TOKEN}"
export OPENAI_API_KEY="${OPENAI_API_KEY}"
cd $HOME/workspace
EOF
source ~/.bashrc
```

Deep Agents Code 使用你的本地环境变量扩展设置脚本中的 `${VAR}` 引用。将密钥存储在本地 `.env` 文件中供设置脚本访问。

> Sandbox 隔离代码执行，但 agent 在处理不受信任的输入时仍容易受到 prompt 注入攻击。使用 human-in-the-loop 审批、短期密钥和仅受信任的设置脚本。详情参见 [安全注意事项](/oss/python/deepagents/sandboxes#security-considerations)。
