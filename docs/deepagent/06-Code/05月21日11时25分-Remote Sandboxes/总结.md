# Remote Sandboxes - 总结

## 核心概念

Remote Sandboxes 是 Deep Agents Code 的**远程代码执行系统**——dcode 进程在本地运行，但工具调用（文件操作、shell 执行）定向到远程 sandbox，实现代码执行隔离。

## Sandbox as Tool 模式

```
本地机器                          远程 Sandbox
┌─────────────────┐              ┌─────────────────┐
│ dcode 进程       │              │ 代码执行环境     │
│ ├── LLM 循环     │──工具调用──→│ ├── read_file   │
│ ├── 记忆         │              │ ├── write_file  │
│ └── 工具分发     │←──结果─────│ ├── edit_file   │
│                  │              │ └── execute     │
└─────────────────┘              └─────────────────┘
```

## 5 个提供商

| 提供商 | 依赖包 | 凭证 |
|--------|--------|------|
| **LangSmith** | 默认包含 | `LANGSMITH_API_KEY` |
| **Daytona** | `langchain-daytona` | `DAYTONA_API_KEY` |
| **Modal** | `langchain-modal` | `modal setup` |
| **Runloop** | `langchain-runloop` | `RUNLOOP_API_KEY` |
| **AgentCore** | `langchain-agentcore-codeinterpreter` | AWS 凭证 |

## 三个 Sandbox 标志

| 标志 | 说明 |
|------|------|
| `--sandbox TYPE` | 选择提供商（`langsmith`/`daytona`/`modal`/`runloop`/`agentcore`） |
| `--sandbox-id ID` | 重用现有 sandbox，跳过创建和清理 |
| `--sandbox-setup PATH` | 创建后运行的设置脚本 |

## 使用流程

```bash
# 1. 安装
uv tool install deepagents-code --with langchain-daytona

# 2. 设置凭证
export DAYTONA_API_KEY="your-key"

# 3. 运行
dcode --sandbox daytona

# 带设置脚本
dcode --sandbox daytona --sandbox-setup ./setup.sh

# 重用现有 sandbox
dcode --sandbox daytona --sandbox-id abc123
```

## 设置脚本

```bash
#!/bin/bash
set -e
git clone https://x-access-token:${GITHUB_TOKEN}@github.com/user/repo.git $HOME/workspace
cd $HOME/workspace
cat >> ~/.bashrc <<'EOF'
export GITHUB_TOKEN="${GITHUB_TOKEN}"
cd $HOME/workspace
EOF
```

- `${VAR}` 引用使用本地环境变量扩展
- 密钥存储在本地 `.env` 文件中

## 关键注意事项

1. **本地进程 + 远程执行** — dcode 在本地，工具在远程
2. **LangSmith 默认包含** — 无需额外安装
3. **`--sandbox-id`** — 重用现有 sandbox 跳过创建/清理
4. **设置脚本** — 用本地环境变量扩展 `${VAR}`
5. **安全** — sandbox 隔离代码执行，但 agent 仍可能受 prompt 注入
6. **文件传输** — 通过设置脚本或提供商 API 传入文件
