# LangSmith Studio

在本地使用 LangChain 构建代理时，可视化代理内部发生的事情、实时交互和调试问题很有帮助。**LangSmith Studio** 是一个免费的可视化界面，用于从本地机器开发和测试你的 LangChain 代理。

Studio 连接到你本地运行的代理，显示代理采取的每一步：发送给模型的提示、工具调用及其结果，以及最终输出。你可以测试不同的输入、检查中间状态，无需额外代码或部署即可迭代代理的行为。

本页介绍如何将 Studio 与本地 LangChain 代理一起设置。

## 前提条件

开始之前，确保你有：

* **LangSmith 账户**：免费注册或登录 [smith.langchain.com](https://smith.langchain.com)。
* **LangSmith API 密钥**：按照[创建 API 密钥](/langsmith/create-account-api-key)指南。
* 如果不想将数据[追踪](/langsmith/observability-concepts#traces)到 LangSmith，在应用的 `.env` 文件中设置 `LANGSMITH_TRACING=false`。

## 设置本地 Agent Server

### 1. 安装 LangGraph CLI

[LangGraph CLI](/langsmith/cli) 提供本地开发服务器（也称 [Agent Server](/langsmith/agent-server)），将你的代理连接到 Studio。

```bash
pip install --upgrade "langgraph-cli[inmem]"
```

### 2. 准备你的代理

如果你已有 LangChain 代理，可以直接使用。本示例使用一个简单的邮件代理：

```python
# agent.py
from langchain.agents import create_agent

def send_email(to: str, subject: str, body: str):
    """Send an email"""
    email = {"to": to, "subject": subject, "body": body}
    return f"Email sent to {to}"

agent = create_agent(
    "gpt-5.4",
    tools=[send_email],
    system_prompt="You are an email assistant. Always use the send_email tool.",
)
```

### 3. 环境变量

Studio 需要 LangSmith API 密钥来连接你的本地代理。在项目根目录创建 `.env` 文件：

```bash
LANGSMITH_API_KEY=lsv2...
```

> **警告：** 确保 `.env` 文件不被提交到版本控制。

### 4. 创建 LangGraph 配置文件

在应用目录创建 `langgraph.json` 文件：

```json
{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/agent.py:agent"
  },
  "env": ".env"
}
```

项目结构如下：

```
my-app/
├── src/
│   └── agent.py
├── .env
└── langgraph.json
```

### 5. 安装依赖

```bash
pip install langchain langchain-openai
```

### 6. 在 Studio 中查看代理

启动开发服务器将代理连接到 Studio：

```bash
langgraph dev
```

服务器运行后，你的代理可通过以下方式访问：

* API：`http://127.0.0.1:2024`
* Studio UI：`https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`

Studio 连接到本地代理后，你可以快速迭代代理的行为：

* 运行测试输入
* 检查完整执行追踪，包括提示、工具参数、返回值和 token/延迟指标
* 当出现问题时，Studio 捕获异常及周围状态
* 开发服务器支持**热重载**——修改提示或工具签名，Studio 立即反映
* 从任何步骤重新运行对话线程
