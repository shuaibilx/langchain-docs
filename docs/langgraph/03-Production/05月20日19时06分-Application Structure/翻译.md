# 应用结构（Application Structure）

LangGraph 应用由一个或多个图、一个配置文件（`langgraph.json`）、一个指定依赖的文件和一个可选的指定环境变量的 `.env` 文件组成。

本指南展示应用的典型结构，以及如何提供所需配置来使用 [LangSmith Deployment](/langsmith/deployment) 部署应用。

> **信息：** LangSmith Deployment 是一个托管托管平台，用于部署和扩展 LangGraph 代理。它处理基础设施、扩展和运维问题，让你可以直接从仓库部署有状态的长时间运行的代理。

## 关键概念

要使用 LangSmith 部署，需要提供以下信息：

1. 一个 [LangGraph 配置文件](#配置文件)（`langgraph.json`），指定应用使用的依赖、图和环境变量。
2. 实现应用逻辑的[图](#图)。
3. 指定运行应用所需[依赖](#依赖)的文件。
4. 应用运行所需的[环境变量](#环境变量)。

## 文件结构

以下是应用目录结构示例：

**Python（requirements.txt）：**

```
my-app/
├── my_agent/
│   ├── utils/
│   │   ├── __init__.py
│   │   ├── tools.py       # 图的工具
│   │   ├── nodes.py       # 图的节点函数
│   │   └── state.py       # 图的状态定义
│   ├── __init__.py
│   └── agent.py           # 构建图的代码
├── .env                    # 环境变量
├── requirements.txt        # 包依赖
└── langgraph.json          # LangGraph 配置文件
```

**Python（pyproject.toml）：**

```
my-app/
├── my_agent/
│   ├── utils/
│   │   ├── __init__.py
│   │   ├── tools.py
│   │   ├── nodes.py
│   │   └── state.py
│   ├── __init__.py
│   └── agent.py
├── .env
├── langgraph.json
└── pyproject.toml          # 项目依赖
```

## 配置文件

`langgraph.json` 文件是一个 JSON 文件，指定部署 LangGraph 应用所需的依赖、图、环境变量和其他设置。

### 示例

```json
{
  "dependencies": ["langchain_openai", "./your_package"],
  "graphs": {
    "my_agent": "./your_package/your_file.py:agent"
  },
  "env": "./.env"
}
```

* 依赖涉及自定义本地包和 `langchain_openai` 包。
* 单个图从 `./your_package/your_file.py` 文件加载，变量名为 `agent`。
* 环境变量从 `.env` 文件加载。

## 依赖

LangGraph 应用可能依赖其他 Python 包。

通常需要指定以下信息来正确设置依赖：

1. 目录中指定依赖的文件（例如 `requirements.txt`、`pyproject.toml` 或 `package.json`）。
2. [配置文件](#配置文件)中的 `dependencies` 键，指定运行 LangGraph 应用所需的依赖。
3. 任何额外的二进制文件或系统库可以使用配置文件中的 `dockerfile_lines` 键指定。

## 图

使用[配置文件](#配置文件)中的 `graphs` 键指定部署的 LangGraph 应用中可用的图。

你可以在配置文件中指定一个或多个图。每个图由一个名称（应该是唯一的）和以下之一的路径标识：（1）编译后的图或（2）创建图的函数。

## 环境变量

如果你在本地使用已部署的 LangGraph 应用，可以在[配置文件](#配置文件)的 `env` 键中配置环境变量。

对于生产部署，你通常需要在部署环境中配置环境变量。
