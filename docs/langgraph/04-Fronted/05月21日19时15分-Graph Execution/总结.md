# Graph Execution - 总结

## 核心概念

Graph Execution 是 LangGraph 前端的**图执行卡片模式**——将多步骤图 pipeline 的每个节点可视化为 UI 卡片，实时显示状态和流式内容。

## 核心模式

```
图节点 → UI 卡片
┌──────────────┐     ┌─────────────────────┐
│ classify     │────→│ Classify  [✓ Done]   │
│ do_research  │────→│ Research  [◉ Running] │
│ analyze      │────→│ Analyze   [○ Waiting] │
│ synthesize   │────→│ Synthesize [○ Waiting]│
└──────────────┘     └─────────────────────┘
```

## 三个核心步骤

### 1. 定义节点映射
```ts
const PIPELINE_NODES = [
  { name: "classify", stateKey: "classification", label: "Classify" },
  { name: "research", stateKey: "research", label: "Research" },
  { name: "analyze", stateKey: "analysis", label: "Analyze" },
];
```

### 2. 路由流式 token 到节点
```ts
const metadata = getMetadata(message);
const node = metadata?.streamMetadata?.langgraph_node;
// node 自动标识 token 来自哪个节点
```

### 3. 确定节点状态
```ts
function getNodeStatus(node, streamingContent, values) {
  if (values?.[node.stateKey]) return "complete";
  if (streamingContent[node.name]) return "streaming";
  return "idle";
}
```

## 四种节点状态

| 状态 | 含义 | UI 表现 |
|------|------|---------|
| `idle` | 未开始 | 灰色 |
| `streaming` | 正在流式传输 | 蓝色脉冲 |
| `complete` | 已完成 | 绿色 |
| *(动态跳过)* | 条件分支跳过 | 隐藏或变暗 |

## 两个内容来源

| 来源 | 时机 | 用途 |
|------|------|------|
| `streamingContent[node.name]` | 节点活跃时 | 实时 token |
| `stream.values[node.stateKey]` | 节点完成后 | 最终输出 |

## UI 组件

| 组件 | 功能 |
|------|------|
| `PipelineProgress` | 顶部进度条，鸟瞰整个 pipeline |
| `NodeCard` | 每个节点的卡片，可折叠 |
| `NodeCardList` | 组合所有 NodeCard |

## 用例

- 研究 pipeline：分类 → 收集 → 分析 → 综合
- 内容生成：大纲 → 草稿 → 核查 → 编辑
- 数据处理：摄取 → 验证 → 转换 → 聚合
- 代码生成：需求 → 架构 → 编码 → 审查 → 测试

## 关键注意事项

1. **langgraph_node 自动设置** — 无需后端配置
2. **声明式节点定义** — PIPELINE_NODES 作为唯一来源
3. **流式优先** — 实时更新，完成后回退到状态值
4. **动态 pipeline** — 过滤出有值的节点
5. **条件分支** — 跳过的节点不显示
6. **错误隔离** — 单节点失败不影响其他
