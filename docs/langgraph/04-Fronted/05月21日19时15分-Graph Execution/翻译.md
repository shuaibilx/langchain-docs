# Graph Execution

> 可视化多步骤图 pipeline，包含每个节点的状态和流式内容

LangGraph agent 不是黑盒。每个图由**命名节点**组成，这些节点按顺序或并行执行：分类、研究、分析、综合。图执行卡片通过为每个节点渲染一张卡片使这个 pipeline 可见，显示其状态、实时流式内容，并跟踪整个工作流的完成情况。用户可以准确看到 agent 在做什么、正在哪个步骤、以及每个步骤产生了什么。

## 图节点如何映射到 UI 卡片

LangGraph 图定义了一系列节点，每个节点负责特定任务。例如，研究 pipeline 可能有：

1. **Classify** — 对用户查询进行分类
2. **Research** — 收集相关信息
3. **Analyze** — 从研究中得出结论
4. **Synthesize** — 产生最终的、精炼的响应

每个节点将其输出写入图状态中的特定键。通过将这些节点名称和状态键映射到 UI 组件，你可以创建整个 pipeline 的可视化表示。

```ts
const PIPELINE_NODES = [
  { name: "classify", stateKey: "classification", label: "Classify" },
  { name: "do_research", stateKey: "research", label: "Research" },
  { name: "analyze", stateKey: "analysis", label: "Analyze" },
  { name: "synthesize", stateKey: "synthesis", label: "Synthesize" },
];

const PIPELINE_NODE_NAMES = new Set(PIPELINE_NODES.map((n) => n.name));
```

## 设置 useStream

照常连接 `useStream`。你将使用的关键属性是 `messages`（用于流式内容路由）、`values`（用于已完成的节点输出）和 `getMessagesMetadata`（用于识别哪个节点产生了每个 token）。

定义一个匹配你的 agent 状态 schema 的 TypeScript 接口，并将其作为类型参数传递给 `useStream`，以获得对状态值的类型安全访问，包括每个 pipeline 节点的自定义状态键。在下面的示例中，将 `typeof myAgent` 替换为你的接口名称：

```ts
import type { BaseMessage } from "@langchain/core/messages";

interface AgentState {
  messages: BaseMessage[];
  classification: string;
  research: string;
  analysis: string;
  synthesis: string;
}
```

**React：**

```tsx
import { useStream } from "@langchain/react";
```

## 将流式 token 路由到节点

随着 agent 流式传输，每条消息都带有标识哪个图节点产生了它的元数据。使用 `getMessagesMetadata` 提取 `langgraph_node` 值并将 token 路由到正确的卡片：

```ts
function getStreamingContent(
  messages: BaseMessage[],
  getMetadata: (msg: BaseMessage) => MessageMetadata | undefined
): Record<string, string> {
  const content: Record<string, string> = {};

  for (const message of messages) {
    if (message.type !== "ai") continue;

    const metadata = getMetadata(message);
    const node = metadata?.streamMetadata?.langgraph_node;

    if (node && PIPELINE_NODE_NAMES.has(node)) {
      content[node] = typeof message.content === "string"
        ? message.content
        : "";
    }
  }

  return content;
}
```

这给你一个从节点名称到其当前流式内容的映射。当 token 到达时，对应的卡片实时更新。

> **注意：** `streamMetadata.langgraph_node` 字段由 LangGraph 自动设置。你不需要在后端进行任何特殊配置。只需照常流式传输消息，元数据就会被包含。

## 确定节点状态

每个节点可以处于四种状态之一：未开始、流式传输中、完成或空闲。你从两个来源派生状态：流式内容映射（用于活动节点）和 `stream.values`（用于已完成节点）：

```ts
type NodeStatus = "idle" | "streaming" | "complete";

function getNodeStatus(
  node: { name: string; stateKey: string },
  streamingContent: Record<string, string>,
  values: Record<string, unknown>
): NodeStatus {
  if (values?.[node.stateKey]) return "complete";
  if (streamingContent[node.name]) return "streaming";
  return "idle";
}
```

## 构建 pipeline 进度条

顶部的水平进度条给用户一个整个 pipeline 的鸟瞰视图。每个步骤是一个带标签的段，当节点完成时填充：

```tsx
function PipelineProgress({
  nodes,
  values,
  streamingContent,
}: {
  nodes: typeof PIPELINE_NODES;
  values: Record<string, unknown>;
  streamingContent: Record<string, string>;
}) {
  return (
    <div className="flex items-center gap-1">
      {nodes.map((node, i) => {
        const status = getNodeStatus(node, streamingContent, values);
        const colors = {
          idle: "bg-gray-200 text-gray-500",
          streaming: "bg-blue-400 text-white animate-pulse",
          complete: "bg-green-500 text-white",
        };

        return (
          <div key={node.name} className="flex items-center">
            <div
              className={`rounded-full px-3 py-1 text-xs font-medium ${colors[status]}`}
            >
              {node.label}
            </div>
            {i < nodes.length - 1 && (
              <div
                className={`mx-1 h-0.5 w-6 ${
                  status === "complete" ? "bg-green-500" : "bg-gray-200"
                }`}
              />
            )}
          </div>
        );
      })}
    </div>
  );
}
```

## 构建可折叠的 NodeCard 组件

每个节点获得自己的卡片，显示状态徽章、内容（流式或最终）和用于长输出的可折叠主体：

```tsx
function NodeCard({
  node,
  status,
  streamingContent,
  completedContent,
}: {
  node: { name: string; stateKey: string; label: string };
  status: NodeStatus;
  streamingContent: string | undefined;
  completedContent: unknown;
}) {
  const [collapsed, setCollapsed] = useState(false);

  const displayContent =
    status === "complete"
      ? formatContent(completedContent)
      : streamingContent ?? "";

  const statusBadge = {
    idle: { text: "Waiting", className: "bg-gray-100 text-gray-600" },
    streaming: {
      text: "Running",
      className: "bg-blue-100 text-blue-700 animate-pulse",
    },
    complete: { text: "Done", className: "bg-green-100 text-green-700" },
  };

  const badge = statusBadge[status];

  return (
    <div className="rounded-lg border bg-white shadow-sm">
      <button
        onClick={() => setCollapsed(!collapsed)}
        className="flex w-full items-center justify-between p-4"
      >
        <div className="flex items-center gap-3">
          <h3 className="font-semibold">{node.label}</h3>
          <span
            className={`rounded-full px-2 py-0.5 text-xs font-medium ${badge.className}`}
          >
            {badge.text}
          </span>
        </div>
        <ChevronIcon direction={collapsed ? "down" : "up"} />
      </button>

      {!collapsed && displayContent && (
        <div className="border-t px-4 py-3">
          <div className="prose prose-sm max-w-none">
            {displayContent}
            {status === "streaming" && (
              <span className="inline-block h-4 w-1 animate-pulse bg-blue-500" />
            )}
          </div>
        </div>
      )}
    </div>
  );
}

function formatContent(value: unknown): string {
  if (typeof value === "string") return value;
  if (value == null) return "";
  return JSON.stringify(value, null, 2);
}
```

## 流式 vs 已完成内容

每个节点有两个内容来源，选择正确的来源对流畅的用户体验很重要：

| 来源 | 何时使用 |
|------|---------|
| `streamingContent[node.name]` | 节点正在活跃流式传输时，包含到达的 token |
| `stream.values[node.stateKey]` | 节点完成后，包含最终的、已提交的输出 |

模式是：为实时更新显示流式内容，节点完成后回退到已提交的状态值。

```ts
for (const node of PIPELINE_NODES) {
  const status = getNodeStatus(node, streamingContent, stream.values);

  const content =
    status === "streaming"
      ? streamingContent[node.name]
      : stream.values?.[node.stateKey];
}
```

> **提示：** 流式内容可能包含部分 token 或尚未完全形成的 markdown。如果你渲染 markdown，确保你的渲染器能够优雅地处理不完整的语法（例如未关闭的粗体标记 `**`）。

## 完整整合

这是组合路由、状态检测和卡片渲染的完整卡片列表：

```tsx
function NodeCardList({
  nodes,
  messages,
  values,
  getMetadata,
}: {
  nodes: typeof PIPELINE_NODES;
  messages: BaseMessage[];
  values: Record<string, unknown>;
  getMetadata: (msg: BaseMessage) => MessageMetadata | undefined;
}) {
  const streamingContent = getStreamingContent(messages, getMetadata);

  return (
    <div className="space-y-3">
      {nodes.map((node) => {
        const status = getNodeStatus(node, streamingContent, values);
        return (
          <NodeCard
            key={node.name}
            node={node}
            status={status}
            streamingContent={streamingContent[node.name]}
            completedContent={values?.[node.stateKey]}
          />
        );
      })}
    </div>
  );
}
```

## 用例

图执行卡片适用于任何需要可见性的多步骤 pipeline：

- **研究 pipeline** — 分类 → 收集来源 → 分析 → 综合报告
- **内容生成** — 大纲 → 草稿 → 事实核查 → 编辑 → 发布
- **数据处理** — 摄取 → 验证 → 转换 → 聚合 → 导出
- **代码生成** — 理解需求 → 规划架构 → 编写代码 → 审查 → 测试
- **决策工作流** — 收集上下文 → 评估选项 → 评分备选方案 → 推荐

## 处理动态 pipeline

并非所有图都有固定数量的节点。一些 pipeline 根据输入添加或跳过节点。通过检查哪些状态键实际有值来处理：

```ts
const activeNodes = PIPELINE_NODES.filter(
  (node) =>
    streamingContent[node.name] ||
    values?.[node.stateKey] ||
    node.name === currentNode
);
```

这确保你的 UI 只显示与当前执行相关的节点的卡片，避免空的占位符卡片。

> **注意：** 如果你的图有条件分支（例如对简单的事实查询跳过 "Research"），跳过的节点永远不会出现在流式内容或状态值中。你的 pipeline 进度条应该通过变暗或隐藏跳过的步骤来反映这一点。

## 最佳实践

1. **声明式定义节点** — 将 `PIPELINE_NODES` 数组作为映射节点名称、状态键和显示标签的唯一真实来源。
2. **优先使用流式内容** — 它给用户即时反馈。只在节点完成后回退到已提交的状态值。
3. **自动折叠已完成节点** — 在长 pipeline 中，自动折叠已完成的卡片，让用户专注于当前活跃的步骤。
4. **显示预估时间** — 如果你有关于每个节点需要多长时间的历史数据，显示时间估计以设定用户期望。
5. **添加全局进度指示器** — 在 pipeline 视图顶部用整体进度条（例如 "Step 2 of 4"）补充每个节点的卡片。
6. **每个节点处理错误** — 如果节点失败，在其卡片中显示错误而不折叠整个 pipeline。其他节点可能仍然成功完成。
