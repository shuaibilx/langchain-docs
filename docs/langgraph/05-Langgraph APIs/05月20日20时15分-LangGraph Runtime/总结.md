# LangGraph 运行时 - 总结

## 核心概念

Pregel 是 LangGraph 的底层运行时引擎，基于 Google Pregel 算法（批量同步并行模型）。

## 架构

```
Pregel = Actors + Channels

Actors（PregelNode）：读取/写入 Channels
Channels：Actor 之间的通信机制
```

## 执行模型（三阶段循环）

```
Plan → Execution → Update → Plan → ...
  │         │          │
  │         │          └─ 用 actor 写入的值更新 channels
  │         └─ 并行执行所有选中的 actors
  └─ 确定哪些 actors 需要执行
```

重复直到无 actor 可执行或达到最大步骤数。

## Channel 类型

| Channel | 特点 | 用途 |
|---------|------|------|
| `LastValue` | 只存最后写入的值（覆盖） | 默认类型，输入/输出/步骤间传递 |
| `Topic` | PubSub，可累积/去重 | 多值传递，跨步骤累积输出 |
| `BinaryOperatorAggregate` | 二元运算符聚合 | 运行总计、累加 |
| `DeltaChannel` (beta) | 只存增量 delta | 频繁写入的大 channel，减小检查点 |

## DeltaChannel 要点

- reducer 必须是**关联的**（batching-invariant）
- reducer 在**重构时**运行，不是写入时
- 必须是 `(state, writes)` 的纯函数
- `snapshot_frequency=K` 每 K 步写快照，限制读取延迟到 O(K)

## 直接使用 Pregel API

```python
from langgraph.channels import EphemeralValue
from langgraph.pregel import Pregel, NodeBuilder

node = (
    NodeBuilder().subscribe_only("a")
    .do(lambda x: x + x)
    .write_to("b")
)

app = Pregel(
    nodes={"node1": node},
    channels={"a": EphemeralValue(str), "b": EphemeralValue(str)},
    input_channels=["a"],
    output_channels=["b"],
)
```

## 与高层 API 的关系

| 高层 API | 编译结果 |
|----------|---------|
| `StateGraph.compile()` | 自动创建 Pregel 实例，生成 nodes + channels |
| `@entrypoint` | 产生 Pregel 实例，包含 `__start__`、`__end__`、`__previous__` channels |

## StateGraph 编译后的内部结构

```
nodes: {__start__, 节点1, 节点2, ...}
channels:
  ├── State 字段 → LastValue
  ├── __start__ → EphemeralValue
  ├── 各节点触发 channel → EphemeralValue
  └── 分支 channel → EphemeralValue
```
