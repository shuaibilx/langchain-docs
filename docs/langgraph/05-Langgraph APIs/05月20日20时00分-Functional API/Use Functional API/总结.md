# 使用 Functional API - 总结

## 核心内容

本页是 Functional API 的实践指南，展示如何在实际代码中使用 `@entrypoint` 和 `@task`。

## 主要模式

### 1. 基础工作流
```python
@task
def step(input): ...

@entrypoint(checkpointer=checkpointer)
def workflow(inputs: dict):
    return step(inputs["key"]).result()

workflow.invoke({"key": "value"}, config)
```

### 2. 并行执行
```python
futures = [task(i) for i in items]
results = [f.result() for f in futures]
```

### 3. 调用 Graph API 图
```python
@entrypoint()
def workflow(x):
    result = graph.invoke({"foo": x})  # 调用 Graph API 编译的图
    return result
```

### 4. 调用其他 Entrypoint
```python
@entrypoint()
def sub(inputs): ...

@entrypoint()
def main(inputs):
    return sub.invoke({...})
```

### 5. 流式传输
```python
from langgraph.config import get_stream_writer

@entrypoint(checkpointer=checkpointer)
def main(inputs):
    writer = get_stream_writer()
    writer("custom data")
    return result

for mode, chunk in main.stream(input, stream_mode=["custom", "updates"], config):
    print(mode, chunk)
```

### 6. 重试与超时
```python
@task(retry_policy=RetryPolicy(retry_on=ValueError))
def unreliable(): ...

@task(timeout=1.0)
async def slow(): ...
```

### 7. 缓存
```python
@task(cache_policy=CachePolicy(ttl=120))
def expensive(): ...

@entrypoint(cache=InMemoryCache())
def workflow(inputs): ...
```

### 8. 人机交互（HITL）
```python
@task
def human_step(input):
    feedback = interrupt("Please review")
    return feedback

# 恢复
workflow.stream(Command(resume=value), config)
```

### 9. Chatbot with 短期记忆
```python
@entrypoint(checkpointer=checkpointer)
def chatbot(inputs, *, previous):
    if previous:
        inputs = add_messages(previous, inputs)
    response = call_model(inputs).result()
    return entrypoint.final(value=response, save=add_messages(inputs, response))
```

## 关键 API

| API | 用途 |
|-----|------|
| `get_stream_writer()` | 获取流式写入器 |
| `RetryPolicy(retry_on=Exception)` | 配置重试策略 |
| `CachePolicy(ttl=秒)` | 配置缓存 |
| `timeout=秒` | 设置超时（仅异步） |
| `entrypoint.final(value, save)` | 解耦返回值和保存值 |
| `Command(resume=value)` | 恢复中断 |
| `graph.get_state(config)` | 查看线程状态 |
| `graph.get_state_history(config)` | 查看线程历史 |

## 错误后恢复

```python
# 第一次调用（会失败）
try:
    workflow.invoke(input, config)
except ValueError:
    pass

# 恢复（使用 None + 相同 thread_id）
workflow.invoke(None, config)
```

## 管理检查点

- `graph.get_state(config)` - 查看当前状态
- `graph.get_state_history(config)` - 查看历史快照列表
- 每个 `StateSnapshot` 包含：values, next, config, metadata, created_at, parent_config, tasks, interrupts
