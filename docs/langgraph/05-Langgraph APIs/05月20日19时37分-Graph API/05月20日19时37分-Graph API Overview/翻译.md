# Graph API 概述

## 图

LangGraph 的核心是将代理工作流建模为图。你使用三个关键组件定义代理的行为：

1. **State（状态）**：表示应用当前快照的共享数据结构。可以是任何数据类型，但通常使用共享状态 schema 定义。

2. **Nodes（节点）**：编码代理逻辑的函数。它们接收当前状态作为输入，执行一些计算或副作用，并返回更新的状态。

3. **Edges（边）**：根据当前状态确定接下来执行哪个节点的函数。它们可以是条件分支或固定转换。

通过组合节点和边，你可以创建随时间演变状态的复杂循环工作流。真正的力量来自 LangGraph 如何管理状态。

简而言之：*节点做工作，边告诉下一步做什么*。

LangGraph 的底层图算法使用[消息传递](https://en.wikipedia.org/wiki/Message_passing)来定义通用程序。灵感来自 Google 的 [Pregel](https://research.google/pubs/pregel-a-system-for-large-scale-graph-processing/) 系统，程序以离散的"超级步骤"进行。

超级步骤可以认为是对图节点的单次迭代。并行运行的节点属于同一超级步骤，而顺序运行的节点属于不同超级步骤。在图执行开始时，所有节点处于 `inactive` 状态。当节点在任何传入边上接收到新消息时变为 `active`。活跃节点运行其函数并返回更新。在每个超级步骤结束时，没有传入消息的节点投票 `halt`。当所有节点都 `inactive` 且没有消息在传输中时，图执行终止。

### StateGraph

[`StateGraph`](https://reference.langchain.com/python/langgraph/graph/state/StateGraph) 类是要使用的主要图类。它由用户定义的 `State` 对象参数化。

### 编译图

构建图时，你首先定义状态，然后添加节点和边，然后编译它。编译提供图结构的一些基本检查（无孤立节点等）。你可以在编译时指定运行时参数如检查点器和断点。

```python
graph = graph_builder.compile(...)
```

> **警告：** 你**必须**编译图才能使用它。

## 状态

定义图时首先要做的就是定义图的 `State`。State 由图的 schema 和 reducer 函数组成，reducer 指定如何应用状态更新。State 的 schema 将是所有节点和边的输入 schema。

### Schema

指定图 schema 的主要方式是使用 [`TypedDict`](https://docs.python.org/3/library/typing.html#typing.TypedDict)。如果你想提供默认值，使用 [`dataclass`](https://docs.python.org/3/library/dataclasses.html)。也支持 Pydantic `BaseModel`。

#### 多个 Schema

通常所有图节点使用单一 schema 通信。但有时需要更多控制：

* 内部节点可以传递不需要在图输入/输出中的信息
* 我们可能想使用不同的输入/输出 schema

可以定义私有 schema `PrivateState` 用于内部节点通信，以及显式的 `input` 和 `output` schema。

```python
class InputState(TypedDict):
    user_input: str

class OutputState(TypedDict):
    graph_output: str

class OverallState(TypedDict):
    foo: str
    user_input: str
    graph_output: str

class PrivateState(TypedDict):
    bar: str

def node_1(state: InputState) -> OverallState:
    return {"foo": state["user_input"] + " name"}

def node_2(state: OverallState) -> PrivateState:
    return {"bar": state["foo"] + " is"}

def node_3(state: PrivateState) -> OutputState:
    return {"graph_output": state["bar"] + " Lance"}

builder = StateGraph(OverallState, input_schema=InputState, output_schema=OutputState)
```

两个要点：
1. 节点可以写入图状态中的任何状态通道
2. 节点可以声明额外的状态通道

### Reducers

Reducers 是理解节点更新如何应用于 State 的关键。每个 State 键有自己的独立 reducer 函数。

#### 默认 reducer

没有显式指定 reducer 时，所有更新覆盖该键。

```python
class State(TypedDict):
    foo: int
    bar: list[str]
# 更新 {"foo": 2} 会覆盖 foo
```

#### 自定义 reducer

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    foo: int
    bar: Annotated[list[str], add]
# 更新 {"bar": ["bye"]} 会追加到 bar
```

#### Overwrite

有时你想绕过 reducer 直接覆盖状态值。LangGraph 提供 `Overwrite` 类型。

```python
from langgraph.types import Overwrite

def replace_messages(state: State):
    return {"messages": Overwrite(["replacement message"])}
```

### 在图状态中使用消息

#### 为什么使用消息？

大多数现代 LLM 提供商有接受消息列表作为输入的聊天模型接口。

#### 在图中使用消息

在图状态中存储先前对话历史作为消息列表很有帮助。使用 reducer 函数（如 `add_messages`）来控制如何更新消息列表。

```python
from langgraph.graph.message import add_messages

class GraphState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
```

#### MessagesState

LangGraph 包含预构建的 `MessagesState`：

```python
from langgraph.graph import MessagesState

class State(MessagesState):
    documents: list[str]
```

## 节点

节点是 Python 函数，接受以下参数：

1. `state` — 图的状态
2. `config` — `RunnableConfig` 对象
3. `runtime` — `Runtime` 对象，包含 context、store、stream_writer、execution_info 等

```python
from langgraph.runtime import Runtime

def node_with_runtime(state: State, runtime: Runtime[Context]):
    print(runtime.context.user_id)
    return {"results": f"Hello, {state['input']}!"}
```

### START 节点

`START` 是特殊节点，表示将用户输入发送到图的节点。

```python
graph.add_edge(START, "node_a")
```

### END 节点

`END` 是特殊节点，表示终端节点。

```python
graph.add_edge("node_a", END)
```

### 节点缓存

LangGraph 支持基于节点输入的缓存。

```python
from langgraph.types import CachePolicy
from langgraph.cache.memory import InMemoryCache

builder.add_node("node", node_fn, cache_policy=CachePolicy(ttl=120))
graph = builder.compile(cache=InMemoryCache())
```

## 边

边定义逻辑如何路由以及图如何决定停止。

* **普通边**：直接从一个节点到下一个
* **条件边**：调用函数确定接下来去哪个节点
* **入口点**：用户输入到达时首先调用哪个节点
* **条件入口点**：调用函数确定首先调用哪个节点

### 普通边

```python
graph.add_edge("node_a", "node_b")
```

### 条件边

```python
graph.add_conditional_edges("node_a", routing_function)
graph.add_conditional_edges("node_a", routing_function, {True: "node_b", False: "node_c"})
```

### 入口点

```python
graph.add_edge(START, "node_a")
```

### 条件入口点

```python
graph.add_conditional_edges(START, routing_function)
```

## Send

LangGraph 支持从条件边返回 `Send` 对象。`Send` 接受两个参数：节点名称和传递给该节点的状态。

```python
from langgraph.types import Send

def continue_to_jokes(state):
    return [Send("generate_joke", {"subject": s}) for s in state['subjects']]

graph.add_conditional_edges("node_a", continue_to_jokes)
```

## Command

`Command` 是控制图执行的多功能原语。接受四个参数：

* `update`：应用状态更新
* `goto`：导航到特定节点
* `graph`：从子图导航时指定父图
* `resume`：提供值以在中断后恢复执行

### 从节点返回

```python
def my_node(state: State) -> Command[Literal["my_other_node"]]:
    return Command(
        update={"foo": "bar"},
        goto="my_other_node"
    )
```

### 作为 invoke/stream 输入

```python
# 恢复中断
result = graph.invoke(Command(resume="yes"), config)
```

### 从工具返回

```python
@tool
def lookup_user_info(runtime: ToolRuntime):
    user_info = get_user_info(runtime.server_info.user.identity)
    return Command(
        update={
            "user_info": user_info,
            "messages": [ToolMessage("成功", tool_call_id=runtime.tool_call_id)]
        }
    )
```

## 图迁移

LangGraph 可以轻松处理图定义的迁移：

* 对于图末尾的线程（未中断），可以更改整个拓扑
* 对于当前中断的线程，支持除重命名/删除节点外的所有拓扑更改
* 添加和删除键有完全的向后和向前兼容性

## 运行时上下文

创建图时可以指定 `context_schema` 用于传递给节点的运行时上下文。

```python
@dataclass
class ContextSchema:
    llm_provider: str = "openai"

graph = StateGraph(State, context_schema=ContextSchema)
graph.invoke(inputs, context={"llm_provider": "anthropic"})
```

### 递归限制

递归限制设置图在单次执行期间可以执行的最大超级步骤数。默认 1000 步。

```python
graph.invoke(inputs, config={"recursion_limit": 5})
```

### 访问和处理递归计数器

当前步骤计数器可在 `config["metadata"]["langgraph_step"]` 中访问。

```python
def my_node(state, config):
    current_step = config["metadata"]["langgraph_step"]
```

LangGraph 提供 `RemainingSteps` 托管值来跟踪剩余步骤数。

```python
from langgraph.managed import RemainingSteps

class State(TypedDict):
    messages: Annotated[list, lambda x, y: x + y]
    remaining_steps: RemainingSteps
```
