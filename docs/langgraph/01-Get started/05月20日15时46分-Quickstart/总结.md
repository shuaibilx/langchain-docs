# LangGraph Quickstart 文档总结

## 一句话概述

LangGraph 提供两种 API 构建 Agent：Graph API（节点+边）和 Functional API（单函数），本快速开始用计算器 Agent 演示两者。

---

## 两种 API 对比

| 维度  | Graph API                                | Functional API    |
| --- | ---------------------------------------- | ----------------- |
| 风格  | 节点 + 边的图                                 | 单个函数 + 控制流        |
| 状态  | `TypedDict` 定义                           | 函数参数              |
| 节点  | `add_node()`                             | `@task` 装饰器       |
| 边   | `add_edge()` / `add_conditional_edges()` | `while` / `if` 逻辑 |
| 入口  | `START` 边                                | `@entrypoint()`   |
| 编译  | `.compile()`                             | 直接调用              |

---

## Graph API 6 步流程

```
1. 定义工具和模型    → @tool + init_chat_model
2. 定义状态          → TypedDict + Annotated
3. 定义模型节点      → 函数调用 model_with_tools.invoke()
4. 定义工具节点      → 遍历 tool_calls 执行工具
5. 定义结束逻辑      → 条件边函数 should_continue()
6. 构建并编译        → StateGraph → add_node → add_edge → compile()
```

---

## Graph API 核心代码模式

```python
# 状态
class MessagesState(TypedDict):
    messages: Annotated[list[AnyMessage], operator.add]

# 条件边
def should_continue(state) -> Literal["tool_node", END]:
    if last_message.tool_calls:
        return "tool_node"
    return END

# 构建图
builder = StateGraph(MessagesState)
builder.add_node("llm_call", llm_call)
builder.add_node("tool_node", tool_node)
builder.add_edge(START, "llm_call")
builder.add_conditional_edges("llm_call", should_continue, ["tool_node", END])
builder.add_edge("tool_node", "llm_call")
agent = builder.compile()
```

---

## Functional API 核心代码模式

```python
@task
def call_llm(messages): ...

@task
def call_tool(tool_call): ...

@entrypoint()
def agent(messages):
    response = call_llm(messages).result()
    while response.tool_calls:
        results = [call_tool(tc).result() for tc in response.tool_calls]
        messages = add_messages(messages, [response, *results])
        response = call_llm(messages).result()
    return add_messages(messages, response)
```

---

## 关键概念

| 概念                              | 说明                    |
| ------------------------------- | --------------------- |
| `StateGraph`                    | 图定义的核心类               |
| `START` / `END`                 | 特殊的起始/结束节点            |
| `add_conditional_edges`         | 条件路由                  |
| `Annotated[list, operator.add]` | 消息追加而非替换              |
| `@task`                         | Functional API 中的任务标记 |
| `@entrypoint`                   | Functional API 中的入口点  |
| `bind_tools`                    | 将工具绑定到模型              |

---

## 与 LangChain 的关系

| LangChain 概念      | 在 LangGraph 中的角色 |
| ----------------- | ---------------- |
| `@tool`           | 定义工具（完全相同）       |
| `init_chat_model` | 初始化模型（完全相同）      |
| `create_agent`    | Graph API 的高层封装  |
| Messages          | 状态中传递的消息         |

---

## 重点与难点

### 重点

- Graph API 的 6 步流程
- Functional API 的 @task + @entrypoint 模式
- 状态定义中 `Annotated` 的作用

### 难点

- 条件边的路由逻辑
- 消息追加 vs 替换的区别
- 两种 API 的选择场景
