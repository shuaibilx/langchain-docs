# Local Server 文档总结

## 一句话概述

使用 LangGraph CLI 在本地启动 Agent Server，通过 Studio UI 或 API 测试应用。

---

## 7 步流程

```
1. 安装 CLI        → pip install -U "langgraph-cli[inmem]"
2. 创建应用        → langgraph new path --template new-langgraph-project-python
3. 安装依赖        → pip install -e .
4. 创建 .env       → LANGSMITH_API_KEY=lsv2...
5. 启动服务器      → langgraph dev
6. Studio 测试     → 浏览器打开 Studio URL
7. API 测试        → Python SDK / REST API
```

---

## 关键命令

| 命令 | 作用 |
|------|------|
| `langgraph new` | 从模板创建新应用 |
| `langgraph dev` | 以内存模式启动开发服务器 |
| `langgraph dev --tunnel` | 创建安全隧道（Safari 兼容） |

---

## 启动后的端点

| 端点 | URL |
|------|-----|
| API | http://127.0.0.1:2024 |
| Studio UI | https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024 |
| API Docs | http://127.0.0.1:2024/docs |

---

## API 测试方式

| 方式 | 特点 |
|------|------|
| Python SDK (async) | `get_client()` + `async for` |
| Python SDK (sync) | `get_sync_client()` + `for` |
| REST API | `curl POST /runs/stream` |

---

## 内存模式 vs 生产模式

| 模式 | 特点 | 适用 |
|------|------|------|
| 内存模式 (`langgraph dev`) | 数据不持久化 | 开发/测试 |
| 生产模式 (LangSmith Deployment) | 持久化存储 | 生产环境 |

---

## 重点

- `langgraph-cli[inmem]` 是本地开发的核心工具
- `.env` 文件配置 API 密钥
- Studio UI 提供可视化调试
- 内存模式仅用于开发，生产需要部署
