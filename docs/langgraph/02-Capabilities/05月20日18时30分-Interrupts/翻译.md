# 中断（Interrupts）

中断允许你在特定点暂停图执行并等待外部输入后继续。这使你需要外部输入才能继续的人机交互模式成为可能。当中断被触发时，LangGraph 使用其[持久化](/oss/python/langgraph/persistence)层保存图状态并无限期等待直到你恢复执行。

中断通过在图节点的任何位置调用 `interrupt()` 函数来工作。该函数接受任何可 JSON 序列化的值，该值会暴露给调用者。当你准备好继续时，使用 `Command` 重新调用图来恢复执行，该值随后成为节点内 `interrupt()` 调用的返回值。

与静态断点（在特定节点之前或之后暂停）不同，中断是**动态的**：它们可以放置在代码中的任何位置，并且可以基于应用逻辑有条件地执行。

* **检查点保存你的位置**：检查点器写入确切的图状态，以便你稍后恢复，即使在错误状态下。
* **`thread_id` 是你的指针**：设置 `config={"configurable": {"thread_id": ...}}` 告诉检查点器加载哪个状态。
* **中断负载通过 `chunk["interrupts"]` 暴露**：使用 `version="v2"` 流式传输时，你传递给 `interrupt()` 的值出现在 `values` 流部分的 `interrupts` 字段中，让你知道图在等待什么。

你选择的 `thread_id` 实际上是你的持久化游标。重用它会恢复相同的检查点；使用新值会启动一个全新状态的新线程。

## 使用 `interrupt` 暂停

[`interrupt`](https://reference.langchain.com/python/langgraph/types/interrupt) 函数暂停图执行并向调用者返回一个值。当你在节点内调用 [`interrupt`](https://reference.langchain.com/python/langgraph/types/interrupt) 时，LangGraph 保存当前图状态并等待你用输入恢复执行。

使用 [`interrupt`](https://reference.langchain.com/python/langgraph/types/interrupt) 你需要：

1. 一个**检查点器**来持久化图状态（生产中使用持久化检查点器）
2. 配置中的**线程 ID**，以便运行时知道从哪个状态恢复
3. 在你想暂停的地方调用 `interrupt()`（负载必须可 JSON 序列化）

```python
from langgraph.types import interrupt

def approval_node(state: State):
    # 暂停并请求批准
    approved = interrupt("Do you approve this action?")

    # 当你恢复时，Command(resume=...) 在这里返回该值
    return {"approved": approved}
```

当你调用 [`interrupt`](https://reference.langchain.com/python/langgraph/types/interrupt) 时，会发生以下情况：

1. **图执行被挂起**在调用 [`interrupt`](https://reference.langchain.com/python/langgraph/types/interrupt) 的确切位置
2. **状态被保存**使用检查点器，以便稍后可以恢复执行
3. **值被返回**给调用者在 `__interrupt__` 下；可以是任何可 JSON 序列化的值
4. **图无限期等待**直到你用响应恢复执行
5. **响应被传回**节点内，成为 `interrupt()` 调用的返回值

## 恢复中断

中断暂停执行后，你通过再次调用图并使用包含恢复值的 `Command` 来恢复图。恢复值被传回 `interrupt` 调用，允许节点用外部输入继续执行。

v2 格式（LangGraph >= 1.1）：

```python
from langgraph.types import Command

# 初始运行 - 命中断点并暂停
config = {"configurable": {"thread_id": "thread-1"}}
result = graph.invoke({"input": "data"}, config=config, version="v2")

# result 是带有 .value 和 .interrupts 的 GraphOutput
print(result.interrupts)
# > (Interrupt(value='Do you approve this action?'),)

# 用人类的响应恢复
graph.invoke(Command(resume=True), config=config, version="v2")
```

**恢复的关键点：**

* 恢复时必须使用与中断发生时**相同的线程 ID**
* 传递给 `Command(resume=...)` 的值成为 `interrupt` 调用的返回值
* 恢复时，节点从调用 `interrupt` 的节点**开头重新开始**，因此 `interrupt` 之前的代码会再次运行
* 你可以传递任何可 JSON 序列化的值作为恢复值

> **警告：** `Command(resume=...)` 是**唯一**作为 `invoke()`/`stream()` 输入的 `Command` 模式。其他 `Command` 参数（`update`、`goto`、`graph`）设计用于[从节点函数返回](/oss/python/langgraph/graph-api#command)。

## 常见模式

### 带人机交互中断的流式传输

使用多流模式（`"messages"` 和 `"updates"`）配合 `subgraphs=True`：

```python
async for chunk in graph.astream(
    initial_input,
    stream_mode=["messages", "updates"],
    subgraphs=True,
    config=config,
    version="v2",
):
    if chunk["type"] == "messages":
        msg, _ = chunk["data"]
        if isinstance(msg, AIMessageChunk) and msg.content:
            display_streaming_content(msg.content)
    elif chunk["type"] == "updates":
        if "__interrupt__" in chunk["data"]:
            interrupt_info = chunk["data"]["__interrupt__"][0].value
            user_response = get_user_input(interrupt_info)
            initial_input = Command(resume=user_response)
            break
```

### 处理多个中断

当并行分支同时中断时（例如扇出到多个节点，每个调用 `interrupt()`），你可能需要在单次调用中恢复多个中断。将每个中断 ID 映射到其恢复值：

```python
from langgraph.types import Command, interrupt
from langgraph.checkpoint.memory import InMemorySaver

class State(TypedDict):
    vals: Annotated[list[str], operator.add]

def node_a(state):
    answer = interrupt("question_a")
    return {"vals": [f"a:{answer}"]}

def node_b(state):
    answer = interrupt("question_b")
    return {"vals": [f"b:{answer}"]}

graph = (
    StateGraph(State)
    .add_node("a", node_a)
    .add_node("b", node_b)
    .add_edge(START, "a")
    .add_edge(START, "b")
    .add_edge("a", END)
    .add_edge("b", END)
    .compile(checkpointer=InMemorySaver())
)

config = {"configurable": {"thread_id": "1"}}
interrupted_result = graph.invoke({"vals": []}, config, version="v2")

# 恢复所有待处理的中断
resume_map = {i.id: f"answer for {i.value}" for i in interrupted_result.interrupts}
result = graph.invoke(Command(resume=resume_map), config, version="v2")
```

### 批准或拒绝

最常见的中断用途之一是在关键操作前暂停并请求批准：

```python
from langgraph.types import interrupt, Command

def approval_node(state: State) -> Command[Literal["proceed", "cancel"]]:
    is_approved = interrupt({
        "question": "Do you want to proceed with this action?",
        "details": state["action_details"]
    })

    if is_approved:
        return Command(goto="proceed")
    else:
        return Command(goto="cancel")
```

恢复时传递 `True` 批准或 `False` 拒绝：

```python
graph.invoke(Command(resume=True), config=config)   # 批准
graph.invoke(Command(resume=False), config=config)   # 拒绝
```

### 审查和编辑状态

有时你想让人类审查和编辑部分图状态：

```python
from langgraph.types import interrupt

def review_node(state: State):
    edited_content = interrupt({
        "instruction": "Review and edit this content",
        "content": state["generated_text"]
    })
    return {"generated_text": edited_content}
```

恢复时提供编辑后的内容：

```python
graph.invoke(Command(resume="The edited and improved text"), config=config)
```

### 工具中的中断

你也可以在工具函数内直接放置中断：

```python
from langchain.tools import tool
from langgraph.types import interrupt

@tool
def send_email(to: str, subject: str, body: str):
    """Send an email to a recipient."""
    response = interrupt({
        "action": "send_email",
        "to": to,
        "subject": subject,
        "body": body,
        "message": "Approve sending this email?"
    })

    if response.get("action") == "approve":
        final_to = response.get("to", to)
        final_subject = response.get("subject", subject)
        final_body = response.get("body", body)
        return f"Email sent to {final_to} with subject '{final_subject}'"
    return "Email cancelled by user"
```

### 验证人类输入

有时你需要验证人类输入，如果无效则再次询问。使用循环中的多个 `interrupt` 调用：

```python
from langgraph.types import interrupt

def get_age_node(state: State):
    prompt = "What is your age?"

    while True:
        answer = interrupt(prompt)

        if isinstance(answer, int) and answer > 0:
            break
        else:
            prompt = f"'{answer}' is not a valid age. Please enter a positive number."

    return {"age": answer}
```

## 中断规则

### 不要将 `interrupt` 调用包装在 try/except 中

`interrupt` 通过抛出特殊异常来暂停执行。如果你在 try/except 块中包装 `interrupt` 调用，你会捕获此异常且中断不会传回图。

```python
# ❌ 错误
def node_a(state: State):
    try:
        interrupt("What's your name?")
    except Exception as e:
        print(e)
    return state

# ✅ 正确：分离逻辑
def node_a(state: State):
    interrupt("What's your name?")
    try:
        fetch_data()
    except Exception as e:
        print(e)
    return state

# ✅ 正确：使用特定异常类型
def node_a(state: State):
    try:
        name = interrupt("What's your name?")
        fetch_data()
    except NetworkException as e:
        print(e)
    return state
```

### 不要重新排序节点内的 `interrupt` 调用

当节点包含多个 `interrupt` 调用时，匹配是**严格基于索引的**。

```python
# ✅ 正确：顺序一致
def node_a(state: State):
    name = interrupt("What's your name?")
    age = interrupt("What's your age?")
    city = interrupt("What's your city?")
    return {"name": name, "age": age, "city": city}

# ❌ 错误：条件跳过
def node_a(state: State):
    name = interrupt("What's your name?")
    if state.get("needs_age"):
        age = interrupt("What's your age?")  # 索引不匹配
    city = interrupt("What's your city?")
    return {"name": name, "city": city}
```

### 不要在 `interrupt` 调用中返回复杂值

```python
# ✅ 正确：简单类型
def node_a(state: State):
    name = interrupt("What's your name?")
    count = interrupt(42)
    approved = interrupt(True)
    return {"name": name, "count": count, "approved": approved}

# ❌ 错误：函数或类实例
def node_a(state: State):
    response = interrupt({
        "question": "What's your name?",
        "validator": validate_input  # 无法序列化
    })
    return {"name": response}
```

### `interrupt` 之前的副作用必须是幂等的

因为中断通过重新运行它们被调用的节点来工作，`interrupt` 之前的副作用应该是幂等的。

```python
# ✅ 正确：幂等操作
def node_a(state: State):
    db.upsert_user(user_id=state["user_id"], status="pending_approval")
    approved = interrupt("Approve this change?")
    return {"approved": approved}

# ✅ 正确：副作用在 interrupt 之后
def node_a(state: State):
    approved = interrupt("Approve this change?")
    if approved:
        db.create_audit_log(user_id=state["user_id"], action="approved")
    return {"approved": approved}

# ✅ 正确：分离到不同节点
def approval_node(state: State):
    approved = interrupt("Approve this change?")
    return {"approved": approved}

def notification_node(state: State):
    if state.approved:
        send_notification(user_id=state["user_id"], status="approved")
    return state
```

## 使用作为函数调用的子图

在节点内调用子图时，父图将从调用子图和触发 `interrupt` 的**节点开头**恢复执行。同样，**子图**也将从调用 `interrupt` 的节点开头恢复。

```python
def node_in_parent_graph(state: State):
    some_code()  # <-- 恢复时会重新执行
    subgraph_result = subgraph.invoke(some_input)

def node_in_subgraph(state: State):
    some_other_code()  # <-- 恢复时也会重新执行
    result = interrupt("What's your name?")
```

## 使用中断调试

要调试和测试图，你可以使用静态中断作为断点，一次一个节点地逐步执行图执行。静态中断在节点执行之前或之后的定义点触发。你可以在编译图时通过指定 `interrupt_before` 和 `interrupt_after` 来设置这些。

> **注意：** 静态中断**不**推荐用于人机交互工作流。使用 `interrupt` 函数代替。

编译时设置：

```python
graph = builder.compile(
    interrupt_before=["node_a"],
    interrupt_after=["node_b", "node_c"],
    checkpointer=checkpointer,
)

config = {"configurable": {"thread_id": "some_thread"}}
graph.invoke(inputs, config=config)  # 运行到断点
graph.invoke(None, config=config)    # 恢复
```

运行时设置：

```python
graph.invoke(
    inputs,
    interrupt_before=["node_a"],
    interrupt_after=["node_b", "node_c"],
    config=config,
)
graph.invoke(None, config=config)  # 恢复
```
