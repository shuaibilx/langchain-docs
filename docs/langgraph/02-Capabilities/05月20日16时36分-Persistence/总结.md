# Persistence 文档总结

## 一句话概述

LangGraph 的持久化层将图状态保存为检查点，支持人机交互、对话记忆、时间旅行和容错。

---

## 四大功能

| 功能   | 说明                     |
| ---- | ---------------------- |
| 人机交互 | interrupt() 暂停，检查点保存状态 |
| 对话记忆 | thread_id 关联对话，保留历史    |
| 时间旅行 | 重放/分叉历史检查点             |
| 容错   | 节点失败后从最后成功步骤恢复         |

---

## 核心概念

### 线程（Thread）

```python
config = {"configurable": {"thread_id": "1"}}
```

- 唯一标识一个对话
- 检查点器的主键
- 必须指定才能持久化

### 检查点（Checkpoint）

- 每个超级步骤边界创建一个
- `StateSnapshot` 对象表示
- 包含 values、next、config、metadata

### 超级步骤（Super-step）

- 图的单个"tick"
- 所有调度的节点执行（可能并行）
- 每个超级步骤后产生检查点

---

## 状态操作

| 操作     | 方法                                   | 说明               |
| ------ | ------------------------------------ | ---------------- |
| 获取最新状态 | `graph.get_state(config)`            | 返回 StateSnapshot |
| 获取历史   | `graph.get_state_history(config)`    | 返回列表             |
| 重放     | 使用旧 checkpoint_id 调用                 | 从该点重新执行          |
| 更新状态   | `graph.update_state(config, values)` | 创建新检查点           |

---

## 记忆存储（Store）

```
检查点器：状态 → 线程（单对话）
Store：    数据 → 跨线程（全局）
```

### 基本操作

```python
store = InMemoryStore()
store.put(namespace, key, value)
store.search(namespace, query="...", limit=3)
```

### 语义搜索

```python
store = InMemoryStore(index={
    "embed": init_embeddings("openai:text-embedding-3-small"),
    "dims": 1536,
    "fields": ["$"]
})
```

## 检查点器库

| 库                                                  | 后端         | 适用    |
| -------------------------------------------------- | ---------- | ----- |
| `langgraph-checkpoint` | 内存         | 开发/测试 |
| `langgraph-checkpoint-sqlite`                      | SQLite     | 本地开发  |
| `langgraph-checkpoint-postgres`                    | PostgreSQL | 生产环境  |

## 序列化与加密

- `JsonPlusSerializer`：默认，支持多种类型
- `pickle_fallback=True`：回退到 pickle
- `EncryptedSerializer`：AES 加密持久化状态
