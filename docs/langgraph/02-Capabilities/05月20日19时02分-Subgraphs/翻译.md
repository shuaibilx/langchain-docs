# 子图（Subgraphs）

本指南介绍使用子图的机制。子图是作为另一个图的[节点](/oss/python/langgraph/graph-api#节点)使用的[图](/oss/python/langgraph/graph-api#图)。

子图适用于：

* 构建[多代理系统](/oss/python/langchain/multi-agent)
* 在多个图中复用一组节点
* 分发开发：当你希望不同团队独立工作于图的不同部分时，可以将每个部分定义为子图，只要遵守子图接口（输入和输出 schema），父图就可以在不知道子图任何细节的情况下构建

## 定义子图通信

添加子图时，你需要定义父图和子图如何通信：

| 模式 | 何时使用 | 状态 Schema |
|------|---------|-------------|
| [在节点内调用子图](#在节点内调用子图) | 父图和子图有**不同的状态 schema**（无共享键），或需要在它们之间转换状态 | 你编写包装函数将父状态映射到子图输入，将子图输出映射回父状态 |
| [将子图添加为节点](#将子图添加为节点) | 父图和子图**共享状态键**——子图从父图的相同通道读取和写入 | 直接将编译后的子图传递给 `add_node`——无需包装函数 |

### 在节点内调用子图

当父图和子图有**不同的状态 schema**（无共享键）时，在节点函数内调用子图。当你想在[多代理](/oss/python/langchain/multi-agent)系统中为每个代理保留私有消息历史时很常见。

节点函数在调用子图之前将父状态转换为子图状态，在返回之前将结果转换回父状态。

```python
from typing_extensions import TypedDict
from langgraph.graph.state import StateGraph, START

class SubgraphState(TypedDict):
    bar: str

# 子图
def subgraph_node_1(state: SubgraphState):
    return {"bar": "hi! " + state["bar"]}

subgraph_builder = StateGraph(SubgraphState)
subgraph_builder.add_node(subgraph_node_1)
subgraph_builder.add_edge(START, "subgraph_node_1")
subgraph = subgraph_builder.compile()

# 父图
class State(TypedDict):
    foo: str

def call_subgraph(state: State):
    # 将状态转换为子图状态
    subgraph_output = subgraph.invoke({"bar": state["foo"]})
    # 将响应转换回父状态
    return {"foo": subgraph_output["bar"]}

builder = StateGraph(State)
builder.add_node("node_1", call_subgraph)
builder.add_edge(START, "node_1")
graph = builder.compile()
```

### 将子图添加为节点

当父图和子图**共享状态键**时，你可以将编译后的子图直接传递给 `add_node`。无需包装函数——子图自动从父图的状态通道读取和写入。例如，在[多代理](/oss/python/langchain/multi-agent)系统中，代理通常通过共享的 [messages](/oss/python/langgraph/graph-api#为什么使用-messages) 键通信。

步骤：
1. 定义子图工作流并编译它
2. 将编译后的子图传递给父图的 [`add_node`](https://reference.langchain.com/python/langgraph/graph/state/StateGraph/add_node) 方法

```python
from typing_extensions import TypedDict
from langgraph.graph.state import StateGraph, START

class State(TypedDict):
    foo: str

# 子图
def subgraph_node_1(state: State):
    return {"foo": "hi! " + state["foo"]}

subgraph_builder = StateGraph(State)
subgraph_builder.add_node(subgraph_node_1)
subgraph_builder.add_edge(START, "subgraph_node_1")
subgraph = subgraph_builder.compile()

# 父图
builder = StateGraph(State)
builder.add_node("node_1", subgraph)  # 直接传递编译后的子图
builder.add_edge(START, "node_1")
graph = builder.compile()
```

## 子图持久化

使用子图时，你需要决定其内部数据在调用之间如何处理。`checkpointer` 参数控制子图持久化：

| 模式 | `checkpointer=` | 行为 |
|------|-----------------|------|
| [每次调用](#每次调用默认) | `None`（默认） | 每次调用开始时全新，继承父图的检查点器以支持单次调用内的[中断](/oss/python/langgraph/interrupts)和[持久执行](/oss/python/langgraph/durable-execution)。 |
| [每线程](#每线程) | `True` | 状态在同一线程的调用之间累积。每次调用从上次离开的地方继续。 |
| [无状态](#无状态) | `False` | 完全无检查点——像普通函数调用一样运行。无中断或持久执行。 |

### 有状态

有状态子图继承父图的检查点器，支持[中断](/oss/python/langgraph/interrupts)、[持久执行](/oss/python/langgraph/durable-execution)和状态检查。

#### 每次调用（默认）

> **提示：** 这是大多数应用的推荐模式，包括子代理作为工具调用的[多代理](/oss/python/langchain/multi-agent)系统。它支持中断、持久执行和并行调用，同时保持每次调用隔离。

省略 `checkpointer` 或设置为 `None`。每次调用开始时全新，但在单次调用内子图继承父图的检查点器并可以使用 `interrupt()` 暂停和恢复。

```python
from langchain.agents import create_agent
from langchain.tools import tool
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import Command, interrupt

@tool
def fruit_info(fruit_name: str) -> str:
    """Look up fruit info."""
    return f"Info about {fruit_name}"

# 子代理 - 无检查点器设置（继承父图）
fruit_agent = create_agent(
    model="gpt-5.4-mini",
    tools=[fruit_info],
    prompt="You are a fruit expert. Respond in one sentence.",
)

# 将子代理包装为外层代理的工具
@tool
def ask_fruit_expert(question: str) -> str:
    """Ask the fruit expert."""
    response = fruit_agent.invoke({"messages": [{"role": "user", "content": question}]})
    return response["messages"][-1].content

# 带检查点器的外层代理
agent = create_agent(
    model="gpt-5.4-mini",
    tools=[ask_fruit_expert],
    prompt="ALWAYS delegate fruit questions to ask_fruit_expert.",
    checkpointer=MemorySaver(),
)
```

#### 每线程

当子代理需要记住之前的交互时使用。子代理的对话历史和数据在同一线程的调用之间累积。

用 `checkpointer=True` 编译。

> **警告：** 每线程子图不支持并行工具调用。使用 `ToolCallLimitMiddleware` 防止并行调用。

```python
# 带 checkpointer=True 的子代理
fruit_agent = create_agent(
    model="gpt-5.4-mini",
    tools=[fruit_info],
    prompt="You are a fruit expert.",
    checkpointer=True,
)

# 使用 ToolCallLimitMiddleware 防止并行调用
agent = create_agent(
    model="gpt-5.4-mini",
    tools=[ask_fruit_expert],
    prompt="...",
    middleware=[ToolCallLimitMiddleware(tool_name="ask_fruit_expert", run_limit=1)],
    checkpointer=MemorySaver(),
)
```

### 无状态

当你想像普通函数调用一样运行子代理，无检查点开销时使用。用 `checkpointer=False` 编译。

> **警告：** 无检查点，子图无持久执行。如果进程在运行中崩溃，子图无法恢复。

```python
subgraph = subgraph_builder.compile(checkpointer=False)
```

### 检查点器参考

| 特性 | 每次调用（默认） | 每线程 | 无状态 |
|------|-----------------|--------|--------|
| `checkpointer=` | `None` | `True` | `False` |
| 中断（HITL） | ✅ | ✅ | ❌ |
| 多轮记忆 | ❌ | ✅ | ❌ |
| 多次调用（不同子图） | ✅ | ⚠️ | ✅ |
| 多次调用（同子图） | ✅ | ❌ | ✅ |
| 状态检查 | ⚠️ | ✅ | ❌ |

## 查看子图状态

启用[持久化](/oss/python/langgraph/persistence)后，你可以使用 subgraphs 选项检查子图状态。

每次调用模式：返回**仅当前调用**的子图状态。

```python
# 查看当前调用的子图状态
subgraph_state = graph.get_state(config, subgraphs=True).tasks[0].state
```

每线程模式：返回**累积的**子图状态。

```python
# 查看累积的子图状态（包含所有调用的消息）
subgraph_state = graph.get_state(config, subgraphs=True).tasks[0].state
```

## 流式传输子图输出

在父图的流方法中设置 subgraphs 选项以包含子图输出：

```python
for chunk in graph.stream(
    {"foo": "foo"},
    subgraphs=True,
    stream_mode="updates",
    version="v2",
):
    print(chunk["type"])  # "updates"
    print(chunk["ns"])    # () 根图, ("node_2:<task_id>",) 子图
    print(chunk["data"])  # {"node_name": {"key": "value"}}
```
