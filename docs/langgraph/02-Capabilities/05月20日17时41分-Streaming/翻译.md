# Streaming（流式传输）

> **提示：** 对于新应用，我们推荐 [Event Streaming](/oss/python/langgraph/event-streaming)——LangGraph v1.2 引入的类型化投影 API。Event Streaming 为每个投影（messages、values、subgraphs、output）提供独立的迭代器，让你可以独立消费它们，而不是基于 `stream_mode` 块进行分支。

本页介绍 LangGraph 的 stream-mode API。它通过流模式如 `updates`、`values`、`messages`、`custom`、`checkpoints`、`tasks` 和 `debug` 暴露图执行。当你需要直接访问图运行时事件或特定流模式输出时使用它。

## 入门

### 基本用法

LangGraph 图暴露 [`stream`](https://reference.langchain.com/python/langgraph/pregel/#langgraph.pregel.Pregel.stream)（同步）和 [`astream`](https://reference.langchain.com/python/langgraph/pregel/#langgraph.pregel.Pregel.astream)（异步）方法，以迭代器形式产出流式输出。传递一个或多个[流模式](#流模式)来控制你接收的数据。

```python
for chunk in graph.stream(
    {"topic": "ice cream"},
    stream_mode=["updates", "custom"],
    version="v2",
):
    if chunk["type"] == "updates":
        for node_name, state in chunk["data"].items():
            print(f"Node {node_name} updated: {state}")
    elif chunk["type"] == "custom":
        print(f"Status: {chunk['data']['status']}")
```

### 流输出格式（v2）

> 需要 LangGraph >= 1.1。本页所有示例使用 `version="v2"`。

将 `version="v2"` 传递给 `stream()` 或 `astream()` 以获得统一的输出格式。每个块是一个具有统一形状的 `StreamPart` 字典——无论流模式、模式数量或子图设置：

```python
{
    "type": "values" | "updates" | "messages" | "custom" | "checkpoints" | "tasks" | "debug",
    "ns": (),           # 命名空间元组，子图事件时填充
    "data": ...,        # 实际负载（类型因流模式而异）
}
```

每个流模式都有对应的 `TypedDict`：`ValuesStreamPart`、`UpdatesStreamPart`、`MessagesStreamPart`、`CustomStreamPart`、`CheckpointStreamPart`、`TasksStreamPart`、`DebugStreamPart`。联合类型 `StreamPart` 是基于 `part["type"]` 的不相交联合，支持编辑器和类型检查器中的完整类型收窄。

使用 v2 格式：

```python
for chunk in graph.stream(inputs, stream_mode="updates", version="v2"):
    print(chunk["type"])  # "updates"
    print(chunk["ns"])    # ()
    print(chunk["data"])  # {"node_name": {"key": "value"}}
```

v2 格式还支持类型收窄，你可以按 `chunk["type"]` 过滤块并获得正确的负载类型：

```python
for part in graph.stream(
    {"topic": "ice cream"},
    stream_mode=["values", "updates", "messages", "custom"],
    version="v2",
):
    if part["type"] == "values":
        # ValuesStreamPart — 每步后的完整状态快照
        print(f"State: topic={part['data']['topic']}")
    elif part["type"] == "updates":
        # UpdatesStreamPart — 仅来自每个节点的变更键
        for node_name, state in part["data"].items():
            print(f"Node `{node_name}` updated: {state}")
    elif part["type"] == "messages":
        # MessagesStreamPart — 来自 LLM 调用的 (message_chunk, metadata)
        msg, metadata = part["data"]
        print(msg.content, end="", flush=True)
    elif part["type"] == "custom":
        # CustomStreamPart — 来自 get_stream_writer() 的任意数据
        print(f"Progress: {part['data']['progress']}%")
```

## 流模式

将以下一个或多个流模式作为列表传递给 [`stream`](https://reference.langchain.com/python/langgraph/graphs/#langgraph.graph.state.CompiledStateGraph.stream) 或 [`astream`](https://reference.langchain.com/python/langgraph/graphs/#langgraph.graph.state.CompiledStateGraph.astream) 方法：

| 模式 | 类型 | 描述 |
|------|------|------|
| [values](#图状态) | `ValuesStreamPart` | 每步后的完整状态。 |
| [updates](#图状态) | `UpdatesStreamPart` | 每步后的状态更新。同一步骤中的多个更新分别流式传输。 |
| [messages](#llm-token) | `MessagesStreamPart` | 来自 LLM 调用的 (LLM token, metadata) 二元组。 |
| [custom](#自定义数据) | `CustomStreamPart` | 通过 [`get_stream_writer`](https://reference.langchain.com/python/langgraph/config/get_stream_writer) 从节点发出的自定义数据。 |
| [checkpoints](#检查点) | `CheckpointStreamPart` | 检查点事件（与 `get_state()` 相同格式）。需要检查点器。 |
| [tasks](#任务) | `TasksStreamPart` | 带结果和错误的任务开始/完成事件。需要检查点器。 |
| [debug](#调试) | `DebugStreamPart` | 所有可用信息——结合 `checkpoints` 和 `tasks` 加额外元数据。 |

### 图状态

使用流模式 `updates` 和 `values` 来流式传输图执行时的状态。

* `updates` 流式传输图每步后的状态**更新**。
* `values` 流式传输图每步后的状态**完整值**。

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
  topic: str
  joke: str

def refine_topic(state: State):
    return {"topic": state["topic"] + " and cats"}

def generate_joke(state: State):
    return {"joke": f"This is a joke about {state['topic']}"}

graph = (
  StateGraph(State)
  .add_node(refine_topic)
  .add_node(generate_joke)
  .add_edge(START, "refine_topic")
  .add_edge("refine_topic", "generate_joke")
  .add_edge("generate_joke", END)
  .compile()
)
```

**updates 模式：**

```python
for chunk in graph.stream(
    {"topic": "ice cream"},
    stream_mode="updates",
    version="v2",
):
    if chunk["type"] == "updates":
        for node_name, state in chunk["data"].items():
            print(f"Node `{node_name}` updated: {state}")
```

输出：
```
Node `refine_topic` updated: {'topic': 'ice cream and cats'}
Node `generate_joke` updated: {'joke': 'This is a joke about ice cream and cats'}
```

**values 模式：**

```python
for chunk in graph.stream(
    {"topic": "ice cream"},
    stream_mode="values",
    version="v2",
):
    if chunk["type"] == "values":
        print(f"topic: {chunk['data']['topic']}, joke: {chunk['data']['joke']}")
```

输出：
```
topic: ice cream, joke:
topic: ice cream and cats, joke:
topic: ice cream and cats, joke: This is a joke about ice cream and cats
```

### LLM Token

使用 `messages` 流模式从图的任何部分（包括节点、工具、子图或任务）**逐 token** 流式传输大语言模型（LLM）输出。

`messages` 模式的流式输出是元组 `(message_chunk, metadata)`：

* `message_chunk`：来自 LLM 的 token 或消息段。
* `metadata`：包含图节点和 LLM 调用详情的字典。

```python
from dataclasses import dataclass
from langchain.chat_models import init_chat_model
from langgraph.graph import StateGraph, START

@dataclass
class MyState:
    topic: str
    joke: str = ""

model = init_chat_model(model="gpt-5.4-mini")

def call_model(state: MyState):
    model_response = model.invoke(
        [{"role": "user", "content": f"Generate a joke about {state.topic}"}]
    )
    return {"joke": model_response.content}

graph = (
    StateGraph(MyState)
    .add_node(call_model)
    .add_edge(START, "call_model")
    .compile()
)

for chunk in graph.stream(
    {"topic": "ice cream"},
    stream_mode="messages",
    version="v2",
):
    if chunk["type"] == "messages":
        message_chunk, metadata = chunk["data"]
        if message_chunk.content:
            print(message_chunk.content, end="|", flush=True)
```

#### 按 LLM 调用过滤

你可以将 `tags` 与 LLM 调用关联，以按 LLM 调用过滤流式 token：

```python
model_1 = init_chat_model(model="gpt-5.4-mini", tags=['joke'])
model_2 = init_chat_model(model="gpt-5.4-mini", tags=['poem'])

async for chunk in graph.astream(
    {"topic": "cats"},
    stream_mode="messages",
    version="v2",
):
    if chunk["type"] == "messages":
        msg, metadata = chunk["data"]
        if metadata["tags"] == ["joke"]:
            print(msg.content, end="|", flush=True)
```

#### 从流中排除消息

使用 `nostream` 标签完全从流中排除 LLM 输出：

```python
stream_model = ChatAnthropic(model_name="claude-haiku-4-5-20251001")
internal_model = ChatAnthropic(model_name="claude-haiku-4-5-20251001").with_config(
    {"tags": ["nostream"]}
)
```

#### 按节点过滤

使用 `stream_mode="messages"` 并按元数据中的 `langgraph_node` 字段过滤：

```python
for chunk in graph.stream(
    inputs,
    stream_mode="messages",
    version="v2",
):
    if chunk["type"] == "messages":
        msg, metadata = chunk["data"]
        if msg.content and metadata["langgraph_node"] == "some_node_name":
            ...
```

### 自定义数据

要从 LangGraph 节点或工具内部发送**自定义用户定义数据**：

1. 使用 [`get_stream_writer`](https://reference.langchain.com/python/langgraph/config/get_stream_writer) 访问流写入器并发出自定义数据。
2. 调用 `.stream()` 或 `.astream()` 时设置 `stream_mode="custom"` 以在流中获取自定义数据。

```python
from typing import TypedDict
from langgraph.config import get_stream_writer
from langgraph.graph import StateGraph, START

class State(TypedDict):
    query: str
    answer: str

def node(state: State):
    writer = get_stream_writer()
    writer({"custom_key": "Generating custom data inside node"})
    return {"answer": "some data"}

graph = (
    StateGraph(State)
    .add_node(node)
    .add_edge(START, "node")
    .compile()
)

for chunk in graph.stream(inputs, stream_mode="custom", version="v2"):
    if chunk["type"] == "custom":
        print(f"Custom event: {chunk['data']['custom_key']}")
```

工具中使用：

```python
from langchain.tools import tool
from langgraph.config import get_stream_writer

@tool
def query_database(query: str) -> str:
    """Query the database."""
    writer = get_stream_writer()
    writer({"data": "Retrieved 0/100 records", "type": "progress"})
    # 执行查询
    writer({"data": "Retrieved 100/100 records", "type": "progress"})
    return "some-answer"
```

### 子图输出

要在流式输出中包含[子图](/oss/python/langgraph/use-subgraphs)的输出，在父图的 `.stream()` 方法中设置 `subgraphs=True`。

输出将以元组 `(namespace, data)` 流式传输，其中 `namespace` 是调用子图的节点路径的元组。

v2 格式：

```python
for chunk in graph.stream(
    {"foo": "foo"},
    subgraphs=True,
    stream_mode="updates",
    version="v2",
):
    print(chunk["type"])  # "updates"
    print(chunk["ns"])    # () 表示根，("node_name:<task_id>",) 表示子图
    print(chunk["data"])  # {"node_name": {"key": "value"}}
```

### 检查点

使用 `checkpoints` 流模式在图执行时接收检查点事件。每个检查点事件与 `get_state()` 的输出格式相同。需要[检查点器](/oss/python/langgraph/persistence)。

```python
from langgraph.checkpoint.memory import MemorySaver

graph = builder.compile(checkpointer=MemorySaver())
config = {"configurable": {"thread_id": "1"}}

for chunk in graph.stream(
    {"topic": "ice cream"},
    config=config,
    stream_mode="checkpoints",
    version="v2",
):
    if chunk["type"] == "checkpoints":
        print(chunk["data"])
```

### 任务

使用 `tasks` 流模式在图执行时接收任务开始和完成事件。需要[检查点器](/oss/python/langgraph/persistence)。

```python
for chunk in graph.stream(
    {"topic": "ice cream"},
    config=config,
    stream_mode="tasks",
    version="v2",
):
    if chunk["type"] == "tasks":
        print(chunk["data"])
```

### 调试

使用 `debug` 流模式在图执行期间流式传输尽可能多的信息。调试模式结合了 `checkpoints` 和 `tasks` 事件并添加额外元数据。

```python
for chunk in graph.stream(
    {"topic": "ice cream"},
    stream_mode="debug",
    version="v2",
):
    if chunk["type"] == "debug":
        print(chunk["data"])
```

### 同时使用多个模式

你可以将列表作为 `stream_mode` 参数传递以同时流式传输多个模式。

```python
for chunk in graph.stream(inputs, stream_mode=["updates", "custom"], version="v2"):
    if chunk["type"] == "updates":
        for node_name, state in chunk["data"].items():
            print(f"Node `{node_name}` updated: {state}")
    elif chunk["type"] == "custom":
        print(f"Custom event: {chunk['data']}")
```

## 高级

### 与任意 LLM 配合使用

你可以使用 `stream_mode="custom"` 从**任何 LLM API** 流式传输数据——即使该 API **没有**实现 LangChain 聊天模型接口。

```python
from langgraph.config import get_stream_writer

def call_arbitrary_model(state):
    writer = get_stream_writer()
    for chunk in your_custom_streaming_client(state["topic"]):
        writer({"custom_llm_chunk": chunk})
    return {"result": "completed"}

graph = (
    StateGraph(State)
    .add_node(call_arbitrary_model)
    .compile()
)

for chunk in graph.stream(
    {"topic": "cats"},
    stream_mode="custom",
    version="v2",
):
    if chunk["type"] == "custom":
        print(chunk["data"])
```

### 禁用特定聊天模型的流式传输

设置 `streaming=False`：

```python
from langchain.chat_models import init_chat_model

model = init_chat_model(
    "claude-sonnet-4-6",
    streaming=False
)
```

### 迁移到 v2

| 场景 | v1（默认） | v2（`version="v2"`） |
|------|-----------|---------------------|
| 单流模式 | 原始数据（dict） | `StreamPart` 字典 |
| 多流模式 | `(mode, data)` 元组 | 统一 `StreamPart` 字典 |
| 子图流式传输 | `(namespace, data)` 元组 | 统一 `StreamPart` 字典 |
| `invoke()` 返回类型 | 普通 dict | `GraphOutput` 对象 |
| 中断位置（stream） | 状态 dict 中的 `__interrupt__` 键 | `values` 流部分的 `interrupts` 字段 |
| 中断位置（invoke） | 结果 dict 中的 `__interrupt__` 键 | `GraphOutput` 的 `.interrupts` 属性 |

#### v2 invoke 格式

传递 `version="v2"` 给 `invoke()` 或 `ainvoke()` 返回 `GraphOutput` 对象：

```python
from langgraph.types import GraphOutput

result = graph.invoke(inputs, version="v2")

assert isinstance(result, GraphOutput)
result.value       # 输出
result.interrupts  # 中断元组
```

#### Pydantic 和 dataclass 状态强制转换

当图状态是 Pydantic 模型或 dataclass 时，v2 `values` 模式自动将输出强制转换为正确的类型：

```python
from pydantic import BaseModel
from typing import Annotated
import operator

class MyState(BaseModel):
    value: str
    items: Annotated[list[str], operator.add]

for chunk in graph.stream(
    {"value": "x", "items": []}, stream_mode="values", version="v2"
):
    print(type(chunk["data"]))  # <class 'MyState'>
```

### Python < 3.11 的异步

在 Python < 3.11 中：

1. 你**必须**显式将 [`RunnableConfig`](https://python.langchain.com/docs/concepts/runnables/#runnableconfig) 传递给异步 LLM 调用。
2. 你**不能**在异步节点或工具中使用 `get_stream_writer`——必须直接传递 `writer` 参数。

异步 LLM 调用手动配置：

```python
async def call_model(state, config):
    topic = state["topic"]
    joke_response = await model.ainvoke(
        [{"role": "user", "content": f"Write a joke about {topic}"}],
        config,
    )
    return {"joke": joke_response.content}
```

异步自定义流式传输：

```python
from langgraph.types import StreamWriter

async def generate_joke(state: State, writer: StreamWriter):
    writer({"custom_key": "Streaming custom data while generating a joke"})
    return {"joke": f"This is a joke about {state['topic']}"}
```
