# Built-in Middleware 文档总结

## 一句话概述

LangChain 提供 16+ 个预构建中间件，覆盖总结、人类审批、限流、重试、PII 检测、工具选择等常见场景。

---

## 中间件全景图

### 安全与合规
| 中间件 | 作用 |
|--------|------|
| **HumanInTheLoop** | 人类审批工具调用 |
| **PIIMiddleware** | 检测/脱敏个人信息 |
| **ModelCallLimit** | 限制模型调用次数 |
| **ToolCallLimit** | 限制工具调用次数 |

### 可靠性
| 中间件 | 作用 |
|--------|------|
| **ModelFallback** | 模型失败自动回退 |
| **ToolRetry** | 工具调用自动重试 |
| **ModelRetry** | 模型调用自动重试 |

### 上下文管理
| 中间件 | 作用 |
|--------|------|
| **Summarization** | 自动总结对话历史 |
| **ContextEditing** | 清除旧工具输出 |
| **Filesystem** | 文件系统存储长期记忆 |

### 智能增强
| 中间件 | 作用 |
|--------|------|
| **LLMToolSelector** | LLM 智能选择工具 |
| **TodoList** | 任务规划和跟踪 |
| **SubAgent** | 子 Agent 隔离上下文 |

### 开发辅助
| 中间件 | 作用 |
|--------|------|
| **LLMToolEmulator** | 模拟工具执行（测试） |
| **ShellTool** | 持久 shell 会话 |
| **FileSearch** | Glob/Grep 文件搜索 |

---

## 使用模式

所有中间件通过同一方式添加：

```python
agent = create_agent(
    model="gpt-5.4",
    tools=[...],
    middleware=[
        中间件1(...),
        中间件2(...),
        # 可以组合多个
    ],
)
```

---

## 最常用的中间件（推荐学习顺序）

1. **Summarization** — 长对话必备
2. **HumanInTheLoop** — 高风险操作必备
3. **ToolRetry / ModelRetry** — 提高可靠性
4. **ModelFallback** — 多 provider 冗余
5. **ToolCallLimit / ModelCallLimit** — 成本控制
6. **PIIMiddleware** — 数据安全

---

## 重点与难点

### 重点
- 每个中间件的适用场景
- `middleware=[...]` 列表式组合
- 大多数中间件可独立使用也可组合

### 难点
- Summarization 的 trigger/keep 条件配置
- HumanInTheLoop 的 interrupt_on 配置和决策类型
- ToolCallLimit 的三种 exit_behavior 区别
- Filesystem 的短期 vs 长期存储

---

## 学习建议

1. **按需学习**：不需要一次掌握所有中间件
2. **先掌握核心 6 个**：Summarization、HITL、Retry、Fallback、Limit、PII
3. **组合使用**：实际项目通常组合多个中间件
4. **注意依赖**：HITL 需要 checkpointer，Filesystem 需要 deepagents
