# Middleware Overview 文档总结

## 一句话概述

Middleware 是 Agent 的"插件系统"，在 Agent 循环的每一步（模型调用前后、工具执行前后）注入自定义逻辑。

---

## 核心架构

```
Agent 循环：
  before_model → 模型调用 → after_model → 工具判断
       ↑                                    ↓
       └── before_tool → 工具执行 → after_tool ┘
```

中间件在这些钩子位置插入自定义逻辑。

---

## 四大应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **跟踪** | 日志、分析、调试 | LangSmith 追踪 |
| **转换** | 提示、工具选择、输出格式 | 动态提示、工具过滤 |
| **容错** | 重试、回退、早期终止 | 工具重试、模型回退 |
| **安全** | 速率限制、护栏、PII 检测 | 人类审批、敏感信息过滤 |

---

## 使用方式

```python
agent = create_agent(
    model="gpt-5.4",
    tools=[...],
    middleware=[中间件1, 中间件2, ...]  # ← 传入列表
)
```

---

## 与 LangGraph 的关系

**中间件不是独立运行时**，它运行在 `create_agent` 返回的 LangGraph 图内部。

这意味着：
- 中间件可以作为子图嵌入更大的 StateGraph
- HITL、总结、PII 等功能随 Agent 节点一起移动
- 支持复杂的拓扑（分类→路由→多个 Agent）

---

## 重点

- 中间件通过 `middleware=[...]` 参数传入 `create_agent`
- 钩子位置：before_model、after_model、before_tool、after_tool
- 内置中间件覆盖常见场景（总结、HITL、PII 等）
- 可嵌入 LangGraph 工作流作为子图
