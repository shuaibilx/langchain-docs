# Custom Middleware 文档总结

## 一句话概述

自定义中间件通过装饰器或类实现，在 Agent 执行的 6 个钩子点注入自定义逻辑。

---

## 6 个钩子点

```
Agent 执行流程：
  before_agent → before_model → wrap_model_call → 模型调用 → after_model → after_agent
                       ↑              ↑                              ↑
                   节点风格        包装风格                        节点风格
```

### 节点风格（顺序执行）

| 钩子 | 时机 | 返回 |
|------|------|------|
| `before_agent` | Agent 开始前 | dict 或 None |
| `before_model` | 模型调用前 | dict 或 None |
| `after_model` | 模型响应后 | dict 或 None |
| `after_agent` | Agent 完成后 | dict 或 None |

### 包装风格（嵌套执行，可控制 handler 调用次数）

| 钩子 | 时机 | 返回 |
|------|------|------|
| `wrap_model_call` | 围绕模型调用 | ModelResponse |
| `wrap_tool_call` | 围绕工具调用 | ToolMessage 或 Command |

---

## 两种创建方式

| 方式 | 适用场景 | 示例 |
|------|----------|------|
| **装饰器** | 单钩子、简单逻辑 | `@before_model` |
| **类** | 多钩子、同步+异步、配置 | `class MyMW(AgentMiddleware)` |

---

## 执行顺序

```
middleware=[A, B, C]

before_*:  A → B → C（正序）
wrap_*:    A(B(C(调用)))（嵌套）
after_*:   C → B → A（倒序）
```

---

## 跳转（Jumps）

从中间件提前退出 Agent 循环：

```python
@before_model(can_jump_to=["end"])
def check(state, runtime):
    if 条件:
        return {"jump_to": "end"}  # 跳到结束
```

目标：`'end'`、`'tools'`、`'model'`

---

## 自定义状态

```python
class CustomState(AgentState):
    counter: NotRequired[int]

@before_model(state_schema=CustomState)
def check(state: CustomState, runtime):
    return {"counter": state.get("counter", 0) + 1}
```

---

## 常见模式

| 模式 | 钩子 | 说明 |
|------|------|------|
| 动态提示 | `wrap_model_call` | 修改 system_message |
| 动态模型 | `wrap_model_call` | 根据条件切换模型 |
| 动态工具 | `wrap_model_call` | 过滤可用工具 |
| 重试 | `wrap_model_call` | handler 调用多次 |
| 监控 | `wrap_tool_call` | 记录工具调用日志 |
| 缓存 | `wrap_model_call` | 缓存模型响应 |

---

## 重点与难点

### 重点
- 6 个钩子的时机和返回类型
- 装饰器 vs 类两种创建方式
- 执行顺序（before 正序、after 倒序、wrap 嵌套）

### 难点
- `wrap_model_call` 中 handler 的调用控制（0次/1次/多次）
- `ExtendedModelResponse` + `Command` 的状态更新机制
- `can_jump_to` 和 `jump_to` 的配合使用

---

## 学习建议

1. **先用装饰器**：快速原型，简单场景
2. **再用类**：需要多钩子或配置时
3. **理解执行顺序**：调试时很重要
4. **跳转是高级功能**：需要时再深入
