# Models 文档总结

## 一句话概述

LangChain 的 Models 模块提供了统一的接口来调用各种 LLM provider（OpenAI、Anthropic、Google 等），支持独立使用和与 Agent 配合使用。

---

## 核心架构

```
用户代码 → init_chat_model / ChatXxx 类 → 统一接口 → 各 Provider API
```

关键设计：**所有 provider 实现相同接口**，切换 provider 不需要改应用逻辑。

---

## 两种使用模式

| 模式           | 说明                  | 场景          |
| ------------ | ------------------- | ----------- |
| **独立使用**     | 直接 `model.invoke()` | 文本生成、分类、提取  |
| **配合 Agent** | 传给 `create_agent()` | 需要工具调用、多步推理 |

---

## 初始化方式

| 方式                | 代码                            | 适用场景               |
| ----------------- | ----------------------------- | ------------------ |
| `init_chat_model` | `init_chat_model("gpt-5.4")`  | 快速开发，自动推断 provider |
| 模型类               | `ChatOpenAI(model="gpt-5.4")` | 需要精细配置             |

模型标识格式：`"provider:model"`（如 `"openai:gpt-5.4"`），也可只写模型名自动推断。

---

## 参数速查

| 参数            | 作用     | 建议值                |
| ------------- | ------ | ------------------ |
| `model`       | 模型名称   | 根据需求选择             |
| `temperature` | 随机性    | 0=确定，1=创意          |
| `max_tokens`  | 最大输出长度 | 根据任务调整             |
| `timeout`     | 超时时间   | 默认不限，生产环境建议设置      |
| `max_retries` | 重试次数   | 默认 6，不可靠网络建议 10-15 |

---

## 调用方法

| 方法                                                 | 说明          | 返回                   |
| -------------------------------------------------- | ----------- | -------------------- |
| `invoke()`                                         | 同步调用        | 单个 `AIMessage`       |
| `stream()`                                         | 流式输出        | `AIMessageChunk` 迭代器 |
| `batch()`                                          | 批量调用        | 结果列表                 |
| `batch_as_completed()` | 批量调用（按完成顺序） | 结果迭代器                |

---

## 四大高级能力

### 1. 工具调用（Tool Calling）

- 用 `model.bind_tools([tool])` 绑定工具
- 模型返回工具调用请求，由你执行并回传结果
- 支持并行工具调用、流式工具调用
- 可强制模型使用特定工具

### 2. 结构化输出（Structured Output）

- `model.with_structured_output(Schema)` 约束输出格式
- 支持 Pydantic、TypedDict、JSON Schema 三种 schema 类型
- 两种策略：`ProviderStrategy`（原生）/ `ToolStrategy`（通用）

### 3. 多模态（Multimodal）

- 支持图像、音频、视频的输入和输出
- 通过内容块（content blocks）传递非文本数据

### 4. 推理（Reasoning）

- 展示模型的多步推理过程
- 可控制推理努力程度

## 生产就绪功能

| 功能            | 说明                           |
| ------------- | ---------------------------- |
| **连接弹性**      | 自动指数退避重试                     |
| **速率限制**      | `InMemoryRateLimiter` 控制请求频率 |
| **提示缓存**      | 隐式（自动）和显式（手动）缓存              |
| **Token 使用量** | 回调/上下文管理器跟踪消耗                |
| **可配置模型**     | 运行时动态切换模型                    |
| **本地模型**      | Ollama 等本地部署方案               |

---

## 与 Agents 文档的关系

- Models 是 Agent 的**推理引擎**（Agent 文档中的 Model 组件）
- Agent 使用 `create_agent(model, tools)` 时，`model` 就是这里的模型
- 独立使用 `model.bind_tools()` 是手动版的 Agent 循环
- 结构化输出在 Agent 中通过 `response_format` 参数使用

---

## 重点与难点

### 重点

- `init_chat_model` 是最简单的入门方式
- `invoke` / `stream` / `batch` 三种调用模式
- `bind_tools` 和 `with_structured_output` 是两个核心方法

### 难点

- 结构化输出的三种 schema 类型和两种策略的选择
- 流式传输中块的累积和处理
- 可配置模型的高级用法

---

## 学习建议

1. **先跑通基础**：选一个 provider，用 `init_chat_model` 完成基本调用
2. **掌握三种调用**：invoke → stream → batch
3. **工具调用**：理解 `bind_tools` 的手动工具执行循环
4. **结构化输出**：Pydantic 是最推荐的 schema 类型
5. **生产考虑**：设置 timeout、max_retries、rate_limiter
