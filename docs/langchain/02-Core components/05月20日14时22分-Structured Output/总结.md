# Structured Output 文档总结

## 一句话概述

结构化输出让 Agent 返回可预测格式的数据（而非自由文本），通过 `response_format` 参数配置，支持两种策略：ProviderStrategy（原生）和 ToolStrategy（通用）。

---

## 两种策略

| 策略 | 原理 | 适用模型 | 可靠性 |
|------|------|----------|--------|
| **ProviderStrategy** | provider API 原生支持 | OpenAI、Anthropic、Gemini 等 | 高 |
| **ToolStrategy** | 通过工具调用实现 | 所有支持工具调用的模型 | 中 |

**自动选择逻辑：**
```python
# 直接传 schema 类型 → 自动选择最佳策略
response_format=ContactInfo
# 等同于：
# 如果支持原生 → ProviderStrategy(ContactInfo)
# 否则 → ToolStrategy(ContactInfo)
```

---

## 支持的 Schema 类型

| 类型 | 返回形式 | 验证 |
|------|----------|------|
| **Pydantic BaseModel** | Pydantic 实例 | 自动验证 |
| **Dataclass** | dict | 无运行时验证 |
| **TypedDict** | dict | 无运行时验证 |
| **JSON Schema** | dict | 无运行时验证 |
| **Union 类型** | Pydantic 实例（ToolStrategy） | 自动选择 |

**推荐：Pydantic BaseModel** — 有字段验证、描述、嵌套支持。

---

## 使用方式

```python
from langchain.agents import create_agent
from pydantic import BaseModel, Field

class Result(BaseModel):
    name: str = Field(description="名称")
    score: float = Field(description="分数")

agent = create_agent(
    "gpt-5.4",
    tools=[...],              # 可选：结构化输出可与工具共存
    response_format=Result    # ← 关键参数
)

result = agent.invoke({"messages": [...]})
data = result["structured_response"]  # ← 结构化结果
```

---

## ToolStrategy 的错误处理

ToolStrategy 有智能重试机制：

```
模型生成输出 → 验证 schema → 失败？→ 返回错误给模型 → 模型重试
                                      成功？→ 返回结果
```

两种常见错误：
1. **多个结构化输出**：Union 类型时模型同时调用多个 → 提示只选一个
2. **Schema 验证失败**：字段值超范围 → 提示修正

`handle_errors` 参数控制重试行为：`True`（默认重试）、`str`（自定义消息）、`False`（不重试）。

## 与之前文档的关系

| 文档 | 结构化输出的角色 |
|------|----------------|
| Agents | `response_format` 是 create_agent 的参数 |
| Models | `model.with_structured_output()` 是独立使用的方式 |
| Messages | 结构化结果存在 `structured_response` 键中 |
| Tools | ToolStrategy 本质上是用工具调用实现的 |

---

## 重点与难点

### 重点
- `response_format` 参数的四种取值
- Pydantic 是最推荐的 schema 类型
- `result["structured_response"]` 获取结果

### 难点
- ProviderStrategy vs ToolStrategy 的自动选择逻辑
- ToolStrategy 的 Union 类型和错误重试机制
- `handle_errors` 的不同取值和行为

---

## 学习建议

1. **先用 Pydantic + 自动选择**：`response_format=MyModel` 最简单
2. **理解两种策略的区别**：原生更可靠，通用更兼容
3. **掌握错误处理**：实际项目中模型可能犯错，重试机制很重要
4. **Union 类型是高级用法**：需要时再深入
