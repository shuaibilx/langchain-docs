# Tools 文档总结

## 一句话概述

Tools 是 Agent 的执行能力来源，通过 `@tool` 装饰器定义函数，模型决定何时调用，工具通过 `ToolRuntime` 访问运行时上下文。

---

## 核心架构

```
用户提问 → Agent(Model) → 决定调用工具 → 执行工具函数 → 结果回传 → 继续推理
```

---

## 工具定义方式

| 方式 | 代码 | 说明 |
|------|------|------|
| **基础** | `@tool` + 函数 | 最简单，docstring 作为描述 |
| **自定义名称** | `@tool("web_search")` | 覆盖默认函数名 |
| **自定义描述** | `@tool(name, description=...)` | 覆盖自动生成的描述 |
| **Pydantic schema** | `@tool(args_schema=Model)` | 复杂输入验证 |
| **JSON schema** | `@tool(args_schema=dict)` | 最灵活的 schema 定义 |

关键规则：
- 类型提示是**必需的**
- 名称用 `snake_case`
- 保留参数名：`config`、`runtime`

---

## ToolRuntime：工具的运行时上下文

这是本文档的**核心概念**。`ToolRuntime` 让工具能访问 8 种运行时信息：

| 组件 | 持久性 | 说明 |
|------|--------|------|
| `runtime.state` | 短期 | 当前对话的可变状态（消息、自定义字段） |
| `runtime.context` | 不可变 | 调用时传入的配置（用户 ID、会话信息） |
| `runtime.store` | 长期 | 跨对话持久化存储（用户偏好、知识库） |
| `runtime.stream_writer` | 临时 | 实时流式输出进度更新 |
| `runtime.execution_info` | 临时 | 线程 ID、运行 ID、重试次数 |
| `runtime.server_info` | 临时 | LangGraph Server 的助手 ID、图 ID |
| `runtime.config` | 临时 | RunnableConfig（回调、标签、元数据） |
| `runtime.tool_call_id` | 临时 | 当前工具调用的唯一 ID |

**记忆模型：**
```
State (短期)  →  当前对话存在，结束后消失
Store (长期)  →  跨对话持久化，未来会话可访问
Context       →  调用时传入，对话期间不变
```

---

## 工具返回值

| 返回类型 | 说明 | 适用场景 |
|----------|------|----------|
| `str` | 转为 ToolMessage | 人类可读的文本结果 |
| `dict` | 序列化为工具输出 | 结构化数据，模型可解析字段 |
| `Command` | 更新 Agent 状态 | 需要修改状态变量时 |

`Command` 是最特殊的返回类型——它不仅返回数据，还能直接修改 Agent 的状态。

---

## 错误处理

通过 `@wrap_tool_call` 中间件捕获工具异常，返回友好的 `ToolMessage` 给模型：

```python
@wrap_tool_call
def handle_errors(request, handler):
    try:
        return handler(request)
    except Exception as e:
        return ToolMessage(content=f"错误: {e}", tool_call_id=...)
```

## 与前四篇文档的知识拼图

| 文档 | 核心 | 与 Tools 的关系 |
|------|------|----------------|
| Agents | Agent 运行循环 | Agent 调用 Tools |
| Models | 模型推理 | Model 决定调用哪个 Tool |
| Messages | 消息格式 | Tool 通过 ToolMessage 回传结果 |
| **Tools** | **工具定义和执行** | **本篇，Agent 的执行能力** |

完整闭环：
```
HumanMessage → Agent(Model) → 决定调用 Tool → 执行 @tool 函数
→ 返回 str/dict/Command → ToolMessage → Model 继续推理 → AIMessage
```

---

## 重点与难点

### 重点
- `@tool` 装饰器是创建工具的核心
- `ToolRuntime` 提供 8 种运行时上下文访问
- 三种返回类型：str、dict、Command

### 难点
- `ToolRuntime` 的 State vs Context vs Store 三者区别
- `Command` 返回类型如何更新 Agent 状态
- 保留参数名 `config` 和 `runtime` 的限制

---

## 学习建议

1. **先掌握 `@tool` 基础**：定义简单工具，理解类型提示和文档字符串的作用
2. **理解 `ToolRuntime`**：这是工具与 Agent 交互的桥梁
3. **区分三种记忆**：State（短期）vs Store（长期）vs Context（不可变）
4. **实践 Command 返回**：这是工具主动修改 Agent 状态的唯一方式
