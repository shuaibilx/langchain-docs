# Agents 文档总结

## 一句话概述

LangChain 的 Agent 是将 LLM 与工具结合的系统，通过 ReAct 循环（推理→行动→观察）自动完成任务。

---

## 核心架构

```
用户输入 → LLM(推理) → 工具调用(行动) → 返回结果(观察) → LLM(继续推理) → 最终输出
```

Agent 的底层是 LangGraph 构建的**图（Graph）**，由节点和边组成执行流程。

---

## 核心组件

### 1. Model（模型）— 推理引擎

| 方式    | 说明                                       | 适用场景                               |
| ----- | ---------------------------------------- | ---------------------------------- |
| 字符串标识 | `"openai:gpt-5.4"`                       | 快速开发                               |
| 模型实例  | `ChatOpenAI(model=..., temperature=...)` | 需要精细配置 |
| 动态模型  | 通过 `@wrap_model_call` 中间件                | 根据上下文切换模型                          |

### 2. Tools（工具）— 执行能力

| 方式      | 说明                          |
| ------- | --------------------------- |
| 静态工具    | 创建时定义，始终可用                  |
| 动态工具-过滤 | 预注册全部工具，运行时按条件过滤            |
| 动态工具-注册 | 运行时从 MCP 服务器等外部源加载工具        |
| 错误处理    | 通过 `@wrap_tool_call` 中间件自定义 |

### 3. System Prompt（系统提示）— 行为塑造

- 静态：直接传字符串
- 高级：用 `SystemMessage` 支持缓存控制
- 动态：用 `@dynamic_prompt` 中间件根据上下文生成

### 4. Memory（记忆）— 短期状态

- 自定义状态必须用 `TypedDict` 扩展 `AgentState`
- 推荐通过中间件定义，也可用 `state_schema` 参数

---

## 高级功能

| 功能        | 说明                                                        |
| --------- | --------------------------------------------------------- |
| **结构化输出** | `ToolStrategy`（通用）/ `ProviderStrategy`（原生）让 Agent 返回结构化数据 |
| **流式传输**  | `agent.stream()` 实时获取中间步骤和 token                          |
| **中间件**   | 在 Agent 执行各阶段注入自定义逻辑                                      |

---

## 关键概念速记

1. **ReAct 模式**：推理 → 行动 → 观察 → 循环直到完成
2. **中间件（Middleware）**：Agent 的扩展机制，类似 Web 框架的中间件
3. **动态工具**：两种模式——过滤预注册工具 vs 运行时注册新工具
4. **结构化输出**：让 LLM 返回符合 schema 的数据，而非自由文本
5. **Graph API**：Agent 底层基于 LangGraph 的图执行引擎

---

## 重点与难点

### 重点

- `create_agent` 是核心入口函数
- 中间件系统是扩展 Agent 的关键机制
- 工具可以是普通函数，用 `@tool` 装饰器增强

### 难点

- 动态工具的两种模式（过滤 vs 注册）适用场景不同
- 中间件的执行顺序和数据流
- `ToolStrategy` vs `ProviderStrategy` 的选择

---

## 学习建议

1. **先掌握基础**：从静态模型 + 静态工具开始，理解 ReAct 循环
2. **再学中间件**：中间件是 LangChain Agent 的核心扩展点
3. **最后进阶**：动态工具、结构化输出、多 Agent 系统
4. **配合 LangGraph**：Agent 底层是 LangGraph，建议同步学习 Graph API
5. **动手实践**：每个概念都写一个最小 demo 跑通
