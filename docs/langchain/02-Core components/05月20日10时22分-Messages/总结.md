# Messages 文档总结

## 一句话概述

Messages 是 LangChain 中模型交互的基本单元，定义了系统、用户、AI 和工具之间通信的标准格式。

---

## 四种消息类型

```
SystemMessage  →  引导模型行为（角色设定、规则）
HumanMessage   →  用户输入（文本、图片、音频、文件）
AIMessage      →  模型输出（文本、工具调用、推理）
ToolMessage    →  工具执行结果回传给模型
```

**对话流程：**
```
System → Human → AI(可能含工具调用) → Tool(结果) → AI(最终回答)
```

---

## 消息的三大组成

| 组成 | 说明 | 示例 |
|------|------|------|
| **Role** | 消息类型 | system / user / assistant / tool |
| **Content** | 实际内容 | 文本、图像、音频等 |
| **Metadata** | 附加信息 | token 用量、响应元数据 |

---

## 内容的三种传入方式

| 方式 | 代码 | 说明 |
|------|------|------|
| 字符串 | `HumanMessage("你好")` | 最简单 |
| Provider 原生格式 | `HumanMessage(content=[{"type":"text",...}])` | 多模态输入 |
| 标准内容块 | `HumanMessage(content_blocks=[...])` | 类型安全，跨 provider |

---

## 内容块体系（核心难点）

LangChain v1 引入的标准内容块系统，将不同 provider 的内容格式统一化：

```
TextContentBlock       → 文本
ReasoningContentBlock  → 推理过程
ImageContentBlock      → 图片
AudioContentBlock      → 音频
VideoContentBlock      → 视频
FileContentBlock       → 文件（PDF等）
ToolCall               → 工具调用
ToolCallChunk          → 流式工具调用
ServerToolCall         → 服务端工具调用
ServerToolResult       → 服务端工具结果
```

关键点：`content`（原始格式）vs `content_blocks`（标准格式），后者是前者的延迟解析版本。

---

## 各消息类型要点

### SystemMessage
- 用于设定模型角色和行为规则
- 支持 `cache_control` 实现提示缓存（Anthropic）

### HumanMessage
- 代表用户输入
- 支持 `name`（标识用户）和 `id`（追踪）
- 支持多模态内容（图片、音频、视频、文件）

### AIMessage
- 模型输出，最丰富的消息类型
- `text` — 文本内容
- `content_blocks` — 标准化内容块
- `tool_calls` — 工具调用请求列表
- `usage_metadata` — token 使用量
- `response_metadata` — provider 响应元数据
- 流式传输时为 `AIMessageChunk`，可累加

### ToolMessage
- 将工具执行结果回传给模型
- `tool_call_id` 必须与 AIMessage 中的调用 ID 匹配
- `artifact` 存储不发给模型但程序可访问的数据

## 多模态输入速查

| 类型 | type 值 | 三种来源 |
|------|---------|----------|
| 图片 | `"image"` | URL / base64 / file_id |
| 音频 | `"audio"` | base64 / file_id |
| 视频 | `"video"` | base64 / file_id |
| 文件 | `"file"` | URL / base64 / file_id |

---

## 与前两篇文档的关系

```
Agents  →  使用 Messages 作为状态传递
Models  →  接收 Messages 作为输入，输出 Messages
Tools   →  通过 ToolMessage 将结果回传
```

Messages 是连接 Agent、Model、Tools 三者的**数据载体**。

---

## 重点与难点

### 重点
- 四种消息类型及各自的使用场景
- `AIMessage` 的属性（tool_calls、usage_metadata）
- 多模态输入的三种来源方式

### 难点
- `content` vs `content_blocks` 的区别（原始 vs 标准化）
- 标准内容块体系的完整类型列表
- `ToolMessage.tool_call_id` 的匹配机制

---

## 学习建议

1. **先记住四种消息类型**：System → Human → AI → Tool
2. **重点理解 AIMessage**：它是信息量最丰富的类型
3. **多模态按需学习**：用到时再查具体格式
4. **content_blocks 是 v1 新特性**：理解它与 content 的关系
