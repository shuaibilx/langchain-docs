# Event Streaming 文档总结

## 一句话概述

Event Streaming 通过 `stream_events(version="v3")` 提供类型化的流式投影，让你独立消费消息、工具调用、状态等不同数据流。

---

## 核心 API

```python
stream = agent.stream_events(input, version="v3")
```

返回一个 `stream` 对象，通过不同的投影（projection）访问不同类型的数据。

---

## 投影体系（核心）

### 消息投影

```
stream.messages          → 每次 LLM 调用的消息流
  ├── message.text       → 文本增量（逐字输出）
  ├── message.reasoning  → 推理过程增量
  ├── message.tool_calls → 工具调用参数增量
  └── message.output     → 最终完整消息对象
```

### 状态投影

```
stream.values            → Agent 状态快照序列
stream.output            → 最终 Agent 状态
```

### 工具投影

```
stream.tool_calls        → 工具执行生命周期
  ├── call.tool_name     → 工具名称
  ├── call.input         → 输入参数
  ├── call.output_deltas → 输出增量
  ├── call.output        → 最终输出
  └── call.error         → 错误信息
```

### 子图投影

```
stream.subgraphs         → 嵌套 Agent/子图的事件
  ├── subagent.graph_name → 子 Agent 名称
  ├── subagent.messages   → 子 Agent 消息流
  ├── subagent.values     → 子 Agent 状态
  └── subagent.output     → 子 Agent 最终状态
```

### 扩展投影

```
stream.extensions        → 自定义转换器投影
stream["tool_activity"]  → 自定义的工具活动流
```

---

## 两种工具调用投影的区别

| 投影 | 时机 | 内容 |
|------|------|------|
| `message.tool_calls` | 模型**生成**工具调用时 | 参数的增量片段 |
| `stream.tool_calls` | 工具**执行**期间 | 执行生命周期、输入输出 |

简单理解：
- `message.tool_calls` → 看模型正在"说"什么工具调用
- `stream.tool_calls` → 看工具正在"做"什么

---

## 同步 vs 异步

| 场景 | 方法 |
|------|------|
| 同步消费单个投影 | `for msg in stream.messages` |
| 同步交叉消费多个投影 | `stream.interleave("messages", "tool_calls")` |
| 异步并发消费 | `asyncio.gather(consume_messages(), consume_tool_calls())` |
| 访问原始事件 | `for event in stream` |

## 重点与难点

### 重点
- `stream_events(version="v3")` 是推荐的流式 API
- `stream.messages` + `message.text` 是最常用的组合
- `stream.output` 获取最终状态

### 难点
- `message.tool_calls` vs `stream.tool_calls` 的区别
- 子 Agent 流式传输的命名空间过滤
- `interleave` 的交叉消费模式

---

## 学习建议

1. **先掌握 `stream.messages` + `message.text`**：覆盖 80% 的流式需求
2. **理解两种 tool_calls 投影**：一个看生成，一个看执行
3. **子 Agent 流式按需学习**：多 Agent 系统时再深入
4. **前端开发重点关注**：Event Streaming 是构建实时 UI 的基础
