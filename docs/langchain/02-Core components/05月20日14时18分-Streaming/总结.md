# Streaming 文档总结

## 一句话概述

Streaming 通过 `stream_mode` 参数提供底层的流式传输能力，支持三种模式：`updates`（Agent 进度）、`messages`（LLM token）、`custom`（自定义数据）。

---

## 与 Event Streaming 的关系

**这篇文档是底层 API，Event Streaming 是上层封装。**

| API | 复杂度 | 推荐度 | 说明 |
|-----|--------|--------|------|
| `stream_events(v3)` | 低 | **推荐新应用** | 类型化投影，一行代码 |
| `stream(stream_mode=...)` | 中 | 底层控制 | 需要手动解析 chunk |

官方建议：**新应用用 Event Streaming**，本篇的 Streaming 是更底层的实现。

---

## 三种 stream_mode

```
stream_mode="updates"   → Agent 步骤级状态更新
stream_mode="messages"  → LLM token 级增量输出
stream_mode="custom"    → 自定义数据（get_stream_writer）
```

### updates 模式
```python
for chunk in agent.stream(input, stream_mode="updates", version="v2"):
    if chunk["type"] == "updates":
        for step, data in chunk["data"].items():
            # step = "model" 或 "tools"
            # data = {"messages": [...]}
```
每个 Agent 步骤后发出一次更新，包含该步骤的完整消息。

### messages 模式
```python
for chunk in agent.stream(input, stream_mode="messages", version="v2"):
    if chunk["type"] == "messages":
        token, metadata = chunk["data"]
        # token = AIMessageChunk（增量）
        # metadata = {"langgraph_node": "model"}
```
每个 LLM token 生成时发出，适合逐字输出。

### custom 模式
```python
# 工具中
from langgraph.config import get_stream_writer
writer = get_stream_writer()
writer("进度更新...")

# 消费端
for chunk in agent.stream(input, stream_mode="custom", version="v2"):
    if chunk["type"] == "custom":
        print(chunk["data"])  # "进度更新..."
```

---

## 多模式组合

```python
stream_mode=["updates", "messages", "custom"]
```

每个 chunk 通过 `chunk["type"]` 区分来源。

---

## 常见模式

### 1. 推理 token 流式
- 需要支持推理的模型（如 Claude with thinking）
- 用 `stream_mode="messages"` + 过滤 `content_blocks` 中 `type="reasoning"`

### 2. 工具调用流式
- `stream_mode="messages"` → 工具调用参数的增量
- `stream_mode="updates"` → 工具调用的完成消息
- 组合使用：`stream_mode=["messages", "updates"]`

### 3. 人类在环流式
- `stream_mode=["messages", "updates"]`
- 在 `updates` 中检测 `__interrupt__`
- 用 `Command(resume=...)` 恢复

### 4. 子 Agent 流式
- 设置 `name=` 给每个 Agent
- `subgraphs=True`
- 通过 `metadata["lc_agent_name"]` 区分来源

---

## v2 格式 vs v1 格式

| 版本 | 格式 | 说明 |
|------|------|------|
| v1（默认） | `(mode, data)` 元组 | 需要解包 |
| v2 | `StreamPart` 字典 | 统一格式，推荐 |

```python
# v2 推荐
for chunk in agent.stream(input, version="v2"):
    print(chunk["type"], chunk["data"])
```

---

## 与 Event Streaming 的对比

| 功能 | Streaming (本篇) | Event Streaming |
|------|-----------------|-----------------|
| API | `stream(stream_mode=...)` | `stream_events(version="v3")` |
| 消息 | `stream_mode="messages"` | `stream.messages` |
| 工具 | 需手动解析 | `stream.tool_calls` |
| 状态 | `stream_mode="updates"` | `stream.values` |
| 自定义 | `stream_mode="custom"` | `stream.extensions` |
| 子图 | `subgraphs=True` | `stream.subgraphs` |
| 推荐 | 底层控制 | **新应用推荐** |

---

## 重点与难点

### 重点
- 三种 stream_mode 的区别和用途
- v2 统一格式的 chunk 结构
- 多模式组合使用

### 难点
- messages 模式中 token 的聚合（获取完整消息）
- 人类在环的中断检测和恢复
- 子 Agent 的名称标识和过滤

---

## 学习建议

1. **新项目直接用 Event Streaming**（上一篇），它更简洁
2. **本篇理解底层原理**：Event Streaming 底层就是这些 stream_mode
3. **需要精细控制时用本篇 API**：比如自定义 chunk 解析逻辑
4. **v2 格式统一且推荐**：始终传 `version="v2"`
