# Short-term Memory 文档总结

## 一句话概述

短期记忆让 Agent 在单个对话线程中记住先前交互，通过 checkpointer 持久化状态，通过修剪/删除/总结管理上下文窗口。

---

## 核心机制

```
Agent 状态 (AgentState)
  ├── messages: [对话历史]     ← 默认的短期记忆
  ├── 自定义字段               ← 扩展 AgentState 添加
  └── checkpointer 持久化      ← InMemorySaver / PostgresSaver
```

**关键概念：Thread（线程）**
- 一个 thread = 一次对话会话
- 通过 `thread_id` 区分不同对话
- 同一 thread 内共享状态

---

## 启用短期记忆

```python
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    "gpt-5.4",
    tools=[...],
    checkpointer=InMemorySaver(),  # ← 关键：添加 checkpointer
)

# 调用时指定 thread_id
agent.invoke(..., {"configurable": {"thread_id": "1"}})
```

| 环境 | Checkpointer |
|------|-------------|
| 开发/测试 | `InMemorySaver()` |
| 生产 | `PostgresSaver` |

## 上下文窗口管理（核心难点）

长对话会超出 LLM 的上下文窗口，三种解决方案：

### 1. Trim（修剪）— 最简单
```python
@before_model
def trim(state, runtime):
    messages = state["messages"]
    if len(messages) > N:
        return {"messages": [RemoveMessage(id=REMOVE_ALL_MESSAGES), *保留的消息]}
```
- 时机：模型调用前
- 优点：简单高效
- 缺点：丢失被修剪消息的信息

### 2. Delete（删除）— 针对性
```python
@after_model
def delete(state, runtime):
    return {"messages": [RemoveMessage(id=m.id) for m in old_messages]}
```
- 时机：模型调用后
- 优点：可以选择性删除
- 缺点：同样丢失信息

### 3. Summarize（总结）— 最智能
```python
SummarizationMiddleware(
    model="gpt-5.4-mini",
    trigger=("tokens", 4000),   # token 达到 4000 时触发
    keep=("messages", 20)       # 保留最近 20 条
)
```
- 时机：自动触发
- 优点：保留关键信息
- 缺点：额外调用模型，增加延迟和成本

**选择建议：**
```
简单场景 → Trim
需要保留信息 → Summarize
需要精确控制 → Delete
```

## 访问记忆的四个入口

| 入口 | 时机 | 用途 |
|------|------|------|
| **Tool (ToolRuntime)** | 工具执行时 | 读写状态、访问 Store |
| **@dynamic_prompt** | 生成提示时 | 根据状态创建动态提示 |
| **@before_model** | 模型调用前 | 修剪/过滤消息 |
| **@after_model** | 模型调用后 | 删除敏感内容、验证响应 |

## 重点与难点

### 重点
- `checkpointer` 是启用短期记忆的关键
- `thread_id` 区分不同对话
- 三种上下文管理策略：Trim、Delete、Summarize

### 难点
- `@before_model` vs `@after_model` 的执行时机
- `RemoveMessage` + `REMOVE_ALL_MESSAGES` 的用法
- `SummarizationMiddleware` 的触发条件配置
- 删除消息后确保消息历史仍然有效

---

## 学习建议

1. **先理解 checkpointer**：它是短期记忆的基础设施
2. **掌握 Trim**：最常用的上下文管理方式
3. **了解 Summarize**：适合长对话场景
4. **注意消息有效性**：删除消息后要确保格式正确
