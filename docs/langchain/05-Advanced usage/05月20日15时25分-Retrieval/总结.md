# Retrieval 文档总结

## 一句话概述

检索通过在查询时获取外部知识来解决 LLM 的有限上下文和静态知识问题，是 RAG 的基础。

---

## Mermaid 图：检索管道

```mermaid
flowchart LR
  S(["Sources"]) --> L[Document Loaders]
  L --> A([Documents])
  A --> B[Split into chunks]
  B --> C[Turn into embeddings]
  C --> D[(Vector Store)]
  Q([User Query]) --> E[Query embedding]
  E --> D
  D --> F[Retriever]
  F --> G[LLM uses retrieved info]
  G --> H([Answer])
```

---

## 构建模块

| 组件 | 作用 |
|------|------|
| Document Loaders | 从外部来源摄取数据 |
| Text Splitters | 分割大文档为小块 |
| Embedding Models | 文本 → 数字向量 |
| Vector Stores | 存储和搜索嵌入 |
| Retrievers | 查询接口，返回文档 |

---

## 三种 RAG 架构

### 2-Step RAG

```mermaid
graph LR
    A[User Question] --> B[Retrieve]
    B --> C[Generate]
    C --> D[Answer]
```

- 检索总是在生成之前
- 简单、可预测、快速
- 适用：FAQ、文档机器人

### Agentic RAG

```mermaid
graph LR
    A[User] --> B[Agent]
    B --> C{Need info?}
    C -- Yes --> D[Search]
    D --> H{Enough?}
    H -- No --> B
    H -- Yes --> I[Answer]
    C -- No --> I
```

- Agent 决定何时和如何检索
- 灵活、可变延迟
- 适用：研究助手、多工具场景

### Hybrid RAG

```mermaid
graph LR
    A[Question] --> B[Enhance Query]
    B --> C[Retrieve]
    C --> D{Enough?}
    D -- No --> E[Refine]
    E --> C
    D -- Yes --> F[Generate]
    F --> G{Quality OK?}
    G -- No --> H{Retry?}
    H -- Yes --> E
    H -- No --> I[Return]
    G -- Yes --> I
```

- 查询增强 + 检索验证 + 回答验证
- 支持迭代优化
- 适用：模糊查询、质量要求高

---

## 架构对比

| 架构 | 控制力 | 灵活性 | 延迟 |
|------|:------:|:------:|:----:|
| 2-Step RAG | 高 | 低 | 快 |
| Agentic RAG | 低 | 高 | 可变 |
| Hybrid RAG | 中 | 中 | 可变 |
