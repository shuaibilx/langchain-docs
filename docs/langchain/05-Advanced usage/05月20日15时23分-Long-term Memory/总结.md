# Long-term Memory 文档总结

## 一句话概述

长期记忆通过 LangGraph Store 跨对话和会话持久化数据，工具通过 `runtime.store` 读写。

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **Store** | 存储引擎（InMemoryStore / PostgresStore） |
| **Namespace** | 命名空间，类似文件夹（如 `("users",)`） |
| **Key** | 键，类似文件名（如 `"user_123"`） |
| **Value** | JSON 文档（任意字典） |

---

## 存储结构

```
Store
├── Namespace: ("users",)
│   ├── Key: "user_123" → {"name": "John", "language": "English"}
│   └── Key: "user_456" → {"name": "Jane", "language": "中文"}
└── Namespace: ("preferences",)
    └── Key: "user_123" → {"theme": "dark", "lang": "zh"}
```

---

## API 操作

| 操作 | 代码 |
|------|------|
| 写入 | `store.put(namespace, key, data)` |
| 读取 | `store.get(namespace, key)` → StoreValue |
| 搜索 | `store.search(namespace, filter={...}, query="...")` |

---

## 工具中的读写

```python
# 读取
@tool
def get_info(runtime: ToolRuntime[Context]) -> str:
    item = runtime.store.get(("users",), runtime.context.user_id)
    return str(item.value)

# 写入
@tool
def save_info(data: dict, runtime: ToolRuntime[Context]) -> str:
    runtime.store.put(("users",), runtime.context.user_id, data)
    return "已保存"
```

---

## 两种 Store 后端

| 后端 | 特点 | 适用 |
|------|------|------|
| InMemoryStore | 内存字典 | 开发/测试 |
| PostgresStore | PostgreSQL | 生产环境 |

---

## 与短期记忆的区别

| 维度 | 短期记忆 | 长期记忆 |
|------|---------|---------|
| 作用域 | 单个线程 | 跨线程 |
| 持久化 | checkpointer | Store |
| 数据 | 对话消息 | 任意 JSON |
| 访问 | 自动 | 工具显式读写 |
