# Human-in-the-Loop 文档总结

## 一句话概述

HITL 中间件通过可配置的策略在工具调用前暂停执行，等待人类的批准、编辑、拒绝或回复。

---

## 四种决策类型

| 类型 | 行为 | 工具是否执行 |
|------|------|------------|
| ✅ `approve` | 原样批准 | 执行原始调用 |
| ✏️ `edit` | 修改后批准 | 执行修改后的调用 |
| ❌ `reject` | 拒绝并反馈 | 不执行，返回反馈 |
| 💬 `respond` | 直接回复 | 不执行，回复作为结果 |

---

## 配置方式

```python
HumanInTheLoopMiddleware(
    interrupt_on={
        "tool_name": True,                                    # 全部决策
        "tool_name": {"allowed_decisions": ["approve", "reject"]},  # 限制决策
        "tool_name": False,                                   # 自动批准
    },
    description_prefix="Tool execution pending approval",
)
```

### 必需条件

- `checkpointer` — 持久化图状态（生产用 `AsyncPostgresSaver`，测试用 `InMemorySaver`）
- `thread_id` — 关联执行与对话线程
- `version="v2"` — 使用 v2 格式获取中断信息

---

## 响应中断的流程

```
1. agent.invoke(...) → 运行到中断
2. result.interrupts → 获取待审查操作
3. 展示给审查者
4. Command(resume={"decisions": [...]}) → 恢复执行
```

---

## 中断返回的数据结构

```python
Interrupt(
    value={
        'action_requests': [
            {
                'name': 'tool_name',
                'arguments': {...},
                'description': '...'
            }
        ],
        'review_configs': [
            {
                'action_name': 'tool_name',
                'allowed_decisions': ['approve', 'reject']
            }
        ]
    }
)
```

## 执行生命周期

```
模型响应
  ↓
中间件检查工具调用
  ↓
需要人类输入？ ──否──→ 执行工具
  │
  是
  ↓
构建 HITLRequest → interrupt()
  ↓
等待人类决策
  ↓
根据决策：
  approve → 执行原始调用
  edit    → 执行修改后的调用
  reject  → 合成 ToolMessage（反馈）
  respond → 返回人类回复作为 ToolMessage
  ↓
恢复执行
```

```python
for chunk in agent.stream(..., stream_mode=["updates", "messages"], version="v2"):

```

## 重点与难点

### 重点
- 四种决策类型（approve/edit/reject/respond）
- `interrupt_on` 配置映射
- 必须配置 checkpointer + thread_id

### 难点
- 多个中断时决策的顺序必须匹配
- edit 时保守修改参数
- 流式处理中断的检测方式
- 执行生命周期的 5 个步骤

---

## 学习建议

1. **先用简单配置**——`interrupt_on={"tool": True}` 即可
2. **理解四种决策**——approve/edit/reject/respond 各有用途
3. **生产环境用持久化检查点**——`AsyncPostgresSaver`
4. **edit 要保守**——大幅修改可能导致意外行为
5. **respond 用于交互式工具**——如 `ask_user` 占位符
