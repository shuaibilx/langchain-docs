# Runtime 文档总结

## 一句话概述

Runtime 是 LangGraph 提供的运行时对象，为工具和中间件提供依赖注入（上下文、存储、流写入器、执行信息、服务器信息）。

---

## Runtime 的 5 大信息

| 信息 | 属性 | 说明 |
|------|------|------|
| **Context** | `runtime.context` | 静态信息：用户 ID、DB 连接、配置 |
| **Store** | `runtime.store` | 长期记忆存储（BaseStore 实例） |
| **Stream writer** | `runtime.writer` | 自定义流模式写入器 |
| **Execution info** | `runtime.execution_info` | 线程 ID、运行 ID、尝试次数 |
| **Server info** | `runtime.server_info` | LangGraph Server 专属元数据 |

---

## 访问方式

### 在工具中

```python
@tool
def my_tool(runtime: ToolRuntime[Context]) -> str:
    user_id = runtime.context.user_id       # 上下文
    runtime.store.get(...)                  # 长期记忆
    info = runtime.execution_info           # 执行信息
    server = runtime.server_info            # 服务器信息
```

### 在中间件中

```python
# 节点风格：通过 Runtime 参数
@before_model
def hook(state, runtime: Runtime[Context]):
    runtime.context.user_name

# 包装风格：通过 ModelRequest 参数
@wrap_model_call
def hook(request, handler):
    request.runtime.context.user_name
```

---

## 核心设计理念：依赖注入

```
传统方式：硬编码 / 全局变量
         ↓ 问题：不可测试、不可重用
Runtime 方式：通过 context 注入
         ↓ 优势：可测试、可重用、灵活
```

## 重点与难点

### 重点
- `context_schema` 定义结构，`context` 传入实例
- 工具用 `ToolRuntime`，中间件用 `Runtime` 或 `request.runtime`
- `server_info` 仅在 LangGraph Server 上可用

### 难点
- 节点风格 vs 包装风格访问方式不同
- `server_info` 为 `None` 的判断
- `execution_info` 需要 `deepagents>=0.5.0`

---

## 学习建议

1. **理解依赖注入的价值**——不要在工具里硬编码用户信息
2. **dataclass 定义 Context**——类型安全、IDE 补全
3. **Store 是可选的**——本地开发可能没有配置 store
4. **先用 context，再用 store**——大多数场景 context 够用
