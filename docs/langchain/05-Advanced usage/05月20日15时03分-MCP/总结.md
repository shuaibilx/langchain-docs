# MCP 文档总结

## 一句话概述

MCP 是一个开放协议，标准化了 LLM 获取工具和上下文的方式。LangChain 通过 `langchain-mcp-adapters` 库连接 MCP 服务器。

---

## MCP 的三大核心概念

| 概念 | 说明 | LangChain 对应 |
|------|------|---------------|
| **Tools** | 可执行函数 | → LangChain Tool |
| **Resources** | 可读数据（文件、DB等） | → Blob 对象 |
| **Prompts** | 可重用提示模板 | → Messages |

---

## 传输机制

| 传输 | 特点 | 适用场景 |
|------|------|---------|
| **HTTP (streamable-http)** | HTTP 请求，支持远程 | 远程服务器、生产环境 |
| **stdio** | 子进程，标准IO | 本地工具、简单设置 |

关键区别：stdio 天然有状态，HTTP 默认无状态。

---

## 有状态 vs 无状态

| 模式 | 行为 |
|------|------|
| **无状态（默认）** | 每次工具调用创建新会话 → 执行 → 清理 |
| **有状态** | `client.session()` 创建持久会话 |

---

## 工具拦截器（核心高级功能）

拦截器 = MCP 版的中间件，解决 MCP 服务器无法访问 LangGraph 运行时的问题。

### 四种运行时访问

| 数据源 | 用途 |
|--------|------|
| `request.runtime.context` | 用户 ID、API 密钥、权限 |
| `request.runtime.store` | 长期记忆（用户偏好） |
| `request.runtime.state` | 对话状态（认证状态） |
| `request.runtime.tool_call_id` | 工具调用跟踪 |

### 拦截器模式

| 模式 | 说明 |
|------|------|
| 修改请求 | `request.override(args=...)` |
| 修改请求头 | `request.override(headers=...)` |
| 短路执行 | 直接返回 `ToolMessage` |
| 状态更新 | 返回 `Command(update=..., goto=...)` |
| 重试 | 循环调用 `handler(request)` |
| 组合 | 洋葱模式，先外后内 |

---

## 回调系统

| 回调 | 用途 |
|------|------|
| `on_progress` | 进度更新（长时间运行的工具） |
| `on_logging_message` | 服务器日志 |
| `on_elicitation` | 引导请求（服务器请求用户输入） |

---

## 引导（Elicitation）

服务器在工具执行期间请求用户输入：

| 操作 | 含义 |
|------|------|
| `accept` | 用户提供了数据 |
| `decline` | 用户拒绝提供 |
| `cancel` | 用户取消操作 |

---

## 与已学知识的关系

| 已学知识 | MCP 的角色 |
|---------|-----------|
| Tools（第 4 篇） | MCP 工具转换为 LangChain 工具 |
| Middleware（第 9-11 篇） | 拦截器 = MCP 版中间件 |
| Runtime（第 13 篇） | 拦截器通过 runtime 访问上下文 |
| Context Engineering（第 14 篇） | MCP 是工具上下文的外部来源 |

---

## 重点与难点

### 重点
- MCP 的三大概念：Tools/Resources/Prompts
- 传输机制的选择（HTTP vs stdio）
- 无状态 vs 有状态会话

### 难点
- 拦截器的洋葱模式执行顺序
- 拦截器中访问运行时上下文的方式
- 引导（Elicitation）的交互流程
- 结构化内容（structuredContent）的处理

---

## 学习建议

1. **先跑通 Quickstart**——理解 MultiServerMCPClient + get_tools()
2. **自定义服务器用 FastMCP**——`@mcp.tool()` 装饰器很简单
3. **拦截器是高级特性**——需要时再深入
4. **生产环境用 HTTP 传输**——stdio 适合本地开发
