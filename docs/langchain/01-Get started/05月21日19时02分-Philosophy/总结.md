# Philosophy - 总结

## 核心概念

Philosophy 是 LangChain 的**设计哲学和历史**——阐述核心信念、两个设计焦点，以及从 2022 年至今的完整演变时间线。

## 核心信念

1. LLM 是强大新技术
2. LLM + 外部数据源 = 更好
3. 未来应用将越来越 agent 化
4. 还处于变革早期
5. 构建原型容易，构建可靠生产 agent 很难

## 两个核心焦点

| 焦点 | 说明 |
|------|------|
| **标准化模型接口** | 统一不同提供商的 API，避免锁定 |
| **编排复杂流程** | 让 LLM 不只是生成文本，还能调用工具、处理数据 |

## 两种构建方式

| 方式 | 特点 | 适合场景 |
|------|------|---------|
| **LangChain** | 最大灵活性和控制 | 自定义架构 |
| **Deep Agents** | 内置规划/文件系统/子 agent | 快速启动 |

两者都构建在 **LangGraph** 之上。

## 演变时间线

```
2022-10  v0.0.1 发布（LLM 抽象 + Chains）
2022-12  第一批通用 agent（ReAct）
2023-01  Chat Completion API + JavaScript 版本
2023-02  LangChain Inc. 公司成立
2023-03  Function Calling 支持
2023-06  LangSmith 发布（可观测性 + 评估）
2024-01  v0.1.0（关注稳定性）
2024-02  LangGraph 发布（低级编排层）
2024-06  700+ 集成
2024-10  LangGraph 成为首选构建方式
2025-04  多模态输入支持
2025-10  v1.0.0（agent 抽象统一 + 标准消息格式）
2026-03  Deep Agents 发布（v0.5.3）
```

## 关键设计决策

| 时间 | 决策 | 影响 |
|------|------|------|
| 2024-02 | LangGraph 独立发布 | 提供低级编排控制 |
| 2024-10 | Chains/Agents 标记弃用 | 迁移到 LangGraph |
| 2025-10 | v1.0 统一 agent 抽象 | 只保留一个高级抽象 |
| 2026-03 | Deep Agents 发布 | 开箱即用的 agent harness |

## 关键注意事项

1. **LangChain = 标准化 + 编排** — 核心价值
2. **LangGraph = 低级控制** — 2024 年独立
3. **Deep Agents = 开箱即用** — 2026 年发布
4. **v1.0 统一了 agent 抽象** — 所有 chains/agents 迁移到 LangGraph
5. **700+ 集成** — 生态系统庞大
